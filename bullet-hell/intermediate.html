🧮 The big idea — a velocity is a polar coordinate

In the Beginner tier a bullet's velocity was fromAngle(θ, speed) — an angle and a speed. That's a polar coordinate, and it turns out every bullet pattern is just a rule for choosing θ (and sometimes speed) for a batch of bullets:

  • Pick θ evenly around the circle → a ring.
  • Pick θ across a small arc → a fan.
  • Rotate that arc's center a little each volley → a spiral.
  • Center the arc on atan2(player − emitter) → an aimed shot.
  • Modulate speed by cos(kθ) → a rose curve.

So we write the firing machinery once — an emitter — and then every pattern is a few lines of trig handed to it. This is why people call danmaku "the most math-beautiful genre": you are literally watching polar equations animate.

💡 Angle convention. On a canvas, +y points down, so θ = 0 aims right, θ = π/2 (90°) aims down (toward the player), and angles increase clockwise. Vector2D.fromAngle(θ, speed) returns (cos θ · speed, sin θ · speed) — a brand-new vector each call, so it's always safe to read its .x/.y.

⚙️ The emitter — what vs. when

An BHEmitter holds a pattern (a fire callback) and a cadence (an interval), plus a base angle it can spin over time. Decoupling those two is the whole trick: change what fires without touching when, and vice-versa. Below, the pattern is a fixed ring — drag the interval and bullet count, and pause it.

class BHEmitter {
  constructor(x, y, opts = {}) {
    this.pos = new Vector2D(x, y);
    this.interval = opts.interval ?? 0.25;  // seconds between volleys
    this.angle = opts.angle ?? 0;           // base aim
    this.spin = opts.spin ?? 0;             // rad/s added to angle (spirals)
    this.fire = opts.fire || (() => {});    // (emitter, field) => void
    this.timer = 0;
  }
  step(dt, field) {
    this.angle += this.spin * dt;           // rotate the base aim
    const iv = Math.max(0.001, this.interval);
    this.timer -= dt;
    while (this.timer <= 0) {                // fire everything we owe this step
      this.timer += iv;                      // carry remainder → exact cadence
      this.fire(this, field);
    }
  }
}

// usage: a pattern is just a `fire` callback handed to the emitter
const em = new BHEmitter(cx, cy, { interval: 0.4, fire: (e, f) => ring(f, e, 12, 130) });
type Fire = (em: BHEmitter, field: BHField) => void;
class BHEmitter {
  pos: Vector2D; interval: number; angle: number; spin: number; fire: Fire; timer = 0;
  constructor(x: number, y: number, opts: Partial<BHEmitter> = {}) { /* …as JS… */ }

    this.angle += this.spin * dt;
    const iv = Math.max(0.001, this.interval);
    this.timer -= dt;

One machine, every pattern. Everything below is the same emitter with a different fire callback (and maybe a non-zero spin). When the Advanced tier builds a boss, this exact class — promoted into the engine — runs its attacks.

⭕ Rings & n-way fans

A ring spaces n bullets evenly around the full circle: θ = i/n · 2π. A fan spreads them across a smaller arc, centered on the aim. Drag the arc from a tight fan all the way up to full ring, and change the count.

// RING — evenly around the circle. i/n (not i/(n-1)) so 0 and 2π aren't doubled.
function ring(field, em, n, speed) {
  for (let i = 0; i < n; i++) {
    const a = em.angle + (i / n) * Math.PI * 2;
    const v = Vector2D.fromAngle(a, speed);
    field.spawn(em.pos.x, em.pos.y, v.x, v.y, { radius: 4 });
  }
}

// FAN — spread across `arc` radians, centered on `center`. i/(n-1) so the
// bullets reach BOTH ends of the arc.
function fan(field, em, n, arc, center, speed) {
  for (let i = 0; i < n; i++) {
    const t = n === 1 ? 0.5 : i / (n - 1);
    const a = center + (t - 0.5) * arc;
    const v = Vector2D.fromAngle(a, speed);
    field.spawn(em.pos.x, em.pos.y, v.x, v.y, { radius: 4 });
  }
}
function ring(field: BHField, em: BHEmitter, n: number, speed: number): void {
  for (let i = 0; i < n; i++) {
    const v = Vector2D.fromAngle(em.angle + (i / n) * Math.PI * 2, speed);
    field.spawn(em.pos.x, em.pos.y, v.x, v.y, { radius: 4 });
  }
}
function fan(field: BHField, em: BHEmitter, n: number, arc: number, center: number, speed: number): void {
  for (let i = 0; i < n; i++) {
    const t = n === 1 ? 0.5 : i / (n - 1);
    const v = Vector2D.fromAngle(center + (t - 0.5) * arc, speed);
    field.spawn(em.pos.x, em.pos.y, v.x, v.y, { radius: 4 });
  }
}
The off-by-one that matters. Rings use i/n (so the bullet at θ=0 and the one at θ=2π don't land on top of each other); fans use i/(n-1) (so a bullet sits at both ends of the arc). Mixing them up is the classic "why is my ring lopsided?" bug.

🌀 The spiral — spin the base angle

Here's the magic trick: fire the same ring every volley, but rotate the base angle a little each step (spin). Because consecutive volleys are slightly turned, the bullets trace smooth spiral arms. More arms make flowers; faster spin tightens the coil. Flip the spin negative to reverse it.

a spiral is a ring that rotates…
📝 Click to see: spin + a multi-arm ring 
const em = new BHEmitter(cx, cy, {
  interval: 0.05,                         // fast → a continuous coil
  spin: 90 * Math.PI / 180,               // rad/s; BHEmitter.step adds spin·dt to angle
  fire: (e, f) => {
    const arms = 3;
    for (let k = 0; k < arms; k++) {       // `arms` evenly spaced spiral starts
      const a = e.angle + (k / arms) * Math.PI * 2;
      const v = Vector2D.fromAngle(a, 140);
      f.spawn(e.pos.x, e.pos.y, v.x, v.y, { radius: 4 });
    }
  },
});
const em = new BHEmitter(cx, cy, {
  interval: 0.05,
  spin: 90 * Math.PI / 180,
  fire: (e: BHEmitter, f: BHField): void => {
    const arms = 3;
    for (let k = 0; k < arms; k++) {
      const v = Vector2D.fromAngle(e.angle + (k / arms) * Math.PI * 2, 140);
      f.spawn(e.pos.x, e.pos.y, v.x, v.y, { radius: 4 });
    }
  },
});
Spin lives on the emitter, not the bullet. The bullets still fly dead straight — it's the emission angle that rotates between volleys. That separation (a slowly turning source firing straight shots) is what produces the hypnotic, readable spiral instead of a mess.

🎯 Aimed shots — a fan that hunts you

Until now patterns ignored you. An aimed shot points at the player: the angle from the emitter to your ship is atan2(player.y − emitter.y, player.x − emitter.x). Center a fan on that and the barrage follows you around. Click the canvas and move — notice you can't just sit still anymore.

const em = new BHEmitter(cx, topY, {
  interval: 0.3,
  fire: (e, f) => {
    // the angle pointing straight at the player, right now
    const aim = Math.atan2(player.pos.y - e.pos.y, player.pos.x - e.pos.x);
    fan(f, e, 5, 40 * Math.PI / 180, aim, 175);   // a 5-shot fan, centered on the aim
  },
});
const em = new BHEmitter(cx, topY, {
  interval: 0.3,
  fire: (e: BHEmitter, f: BHField): void => {
    const aim = Math.atan2(player.pos.y - e.pos.y, player.pos.x - e.pos.x);
    fan(f, e, 5, 40 * Math.PI / 180, aim, 175);
  },
});
atan2, not atan. atan2(dy, dx) takes the two components separately, so it returns the correct angle in all four quadrants (a plain atan(dy/dx) can't tell left from right and divides by zero straight up). It is the single most-used function in enemy AI and aiming.

🌹 Rose curves — r = cos(kθ)

The rose curve r = cos(kθ) is a classic polar equation that draws flower petals. We borrow it as a speed rule: fire a full ring, but set each bullet's speed from |cos(kθ)|. Slow bullets bunch up where the curve pinches to zero, fast ones race out where it peaks — so the expanding ring blooms into petals. Change k; toggle a second counter-rotating layer for interference.

r = |cos(k·θ)|…

const N = 44;
fire: (e, f) => {
  for (let i = 0; i < N; i++) {
    const a = e.angle + (i / N) * Math.PI * 2;
    const r = Math.abs(Math.cos(k * a));     // the rose-curve radius at this angle
    const speed = 50 + 170 * r;              // map r → speed: pinch = slow, peak = fast
    const v = Vector2D.fromAngle(a, speed);
    f.spawn(e.pos.x, e.pos.y, v.x, v.y, { radius: 3.5 });
  }
}
// odd k → k petals; even k → 2k petals (the same rule a pencil-drawn rose follows).

Patterns are equations you can read. Swap cos(kθ) for 0.5 + 0.5·sin(kθ), or feed θ through a noise function, and you've invented a new attack. Once you see a pattern as "a function from angle to (speed, radius, colour)", authoring danmaku becomes playing with equations.

↪️ Curved paths — bullets that turn & accelerate

So far every bullet flew straight. But back in the engine, BHBullet has two knobs that have been sitting at zero this whole time: turn (spin the heading, rad/s) and accel (change speed along it, px/s²). Switch them on and a straight stream bends into whips and arcs. Click and dodge.

📝 Click to see: the knobs were already in the engine 
// Just spawn with non-zero turn / accel — BHField.step does the rest:
const v = Vector2D.fromAngle(em.angle, 150);
field.spawn(em.pos.x, em.pos.y, v.x, v.y, {
  turn:  60 * Math.PI / 180,   // rad/s: rotate the heading every step
  accel: 0,                    // px/s²: change speed along the heading
});

// …inside engine/field.js, per bullet, per step:
//   if (b.turn)  b.vel.rotate(b.turn * dt);
//   if (b.accel) b.vel.multiply((speed + b.accel*dt) / speed);
//   b.pos.add(b.vel.copy().multiply(dt));
const v = Vector2D.fromAngle(em.angle, 150);
field.spawn(em.pos.x, em.pos.y, v.x, v.y, {
  turn:  60 * Math.PI / 180,
  accel: 0,
});
// BHField.step applies turn (rotate vel) then accel (rescale vel) before moving.
Design seams pay off later. Those knobs were added to BHBullet in the very first scaffold and left inert — exactly so this lesson could switch them on without touching the engine. A small, well-placed extension point turns a "rewrite" into a one-line opt-in. (Full scripted Bézier paths are an Advanced-tier follow-up.)

Click and survive; graze for +100, eat a bullet for −300. This little timeline — "play pattern A, then B, then C" — is precisely what the next tier turns into a boss's spell card.
