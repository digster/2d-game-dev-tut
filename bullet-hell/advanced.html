🧩 From a pattern to a fight

The Intermediate tier ended with an enemy cycling three patterns on a hand-written timeline. That timeline was a hint: a boss is just a richer version of it. In this tier we add the four things that turn "a pretty pattern" into "a fight you can win or lose":

  • An entity with HP that moves and can be shot down — so there's a goal.
  • Attack scripts as data — a "spell card" is a list of timed instructions, not a tangle of ifs. Data is editable, swappable, and (later) authorable in a visual editor.
  • Phases — cross an HP threshold and the boss changes its whole attack.
  • The player's bomb — a panic button that clears the screen at a cost, the risk/reward dial the whole genre balances around.
💡 The emitter graduated to the engine. The BHEmitter you built in the Intermediate tier now lives in engine/emitter.js — promoted the moment this tier became its second consumer. Spell cards drive it by swapping its fire callback and interval on a schedule.

👾 The boss entity — HP, movement, a health bar

A boss is a position that sweeps the top of the field (a Lissajous figure — two sines at different rates), plus HP and an emitter riding along with it. Click, dodge its fan, and hold Z to shoot it down — the health bar drains as your shots land.

class BHBoss {
  constructor(x, y, opts = {}) {
    this.pos = new Vector2D(x, y);
    this.radius = opts.radius ?? 22;
    this.maxHp = opts.maxHp ?? 800; this.hp = this.maxHp;
    this.t = 0;
  }
  move(dt, bounds) {                       // a Lissajous sweep: two sines, different rates
    this.t += dt;
    const cx = bounds.x + bounds.w / 2;
    this.pos.x = cx + Math.sin(this.t * 0.8) * (bounds.w * 0.30);
    this.pos.y = bounds.y + 74 + Math.sin(this.t * 1.6) * 20;
  }
  get alive() { return this.hp > 0; }
  get hpFrac() { return Math.max(0, this.hp) / this.maxHp; }
}
// the emitter rides the boss: em.pos.set(boss.pos.x, boss.pos.y) each frame.
class BHBoss {
  pos: Vector2D; radius: number; maxHp: number; hp: number; t = 0;
  constructor(x: number, y: number, opts: { radius?: number; maxHp?: number } = {}) {
    this.pos = new Vector2D(x, y);
    this.radius = opts.radius ?? 22;
    this.maxHp = opts.maxHp ?? 800; this.hp = this.maxHp;
  }
  move(dt: number, bounds: Rect): void { /* …two-sine sweep… */ }
  get alive(): boolean { return this.hp > 0; }
  get hpFrac(): number { return Math.max(0, this.hp) / this.maxHp; }
}
Two sines make organic movement. One sine on X and a faster one on Y traces a figure-eight-ish path that never sits still — far more alive than a straight slide, and trivial to author. Boss movement is mostly "pick two frequencies."

📜 Attack scripts — a spell card as data

An attack shouldn't be a pile of timers and ifs. A spell card is a list of timed steps; each step reconfigures the emitter (its fire callback, interval, spin) when the card's clock reaches it. The card below runs a ring burst, then a spiral, then an aimed fan, then loops — and it's pure data. Click and dodge while it plays itself.

class BHSpellCard {
  constructor(def) { this.steps = def.steps; this.duration = def.duration ?? Infinity; this.reset(); }
  reset() { this.t = 0; this.idx = 0; }
  step(dt, em, field) {                     // apply every step that's now due
    this.t += dt;
    while (this.idx < this.steps.length && this.t >= this.steps[this.idx].at) {
      const s = this.steps[this.idx++];
      if (s.set) Object.assign(em, s.set);   // swap fire / interval / spin
      if (s.set && 'fire' in s.set) em.timer = 0;
      if (s.burst) s.burst(em, field);        // one-shot volley
    }
    return this.t >= this.duration;          // true → card finished
  }
}

// THE ATTACK, as data:
const card = new BHSpellCard({ name: 'Three-Act Opener', duration: 8, steps: [
  { at: 0,   set: { spin: 0,   interval: 0.5,  fire: (e,f) => ring(f, e, 22, 110) } },
  { at: 2.6, set: { spin: 2.2, interval: 0.05, fire: (e,f) => spiral(f, e, 3) } },
  { at: 5.2, set: { spin: 0,   interval: 0.22, fire: (e,f) => aimedFan(f, e, player) } },
]});
interface Step { at: number; set?: Partial<BHEmitter>; burst?: (em: BHEmitter, f: BHField) => void; }
interface CardDef { name: string; duration?: number; steps: Step[]; }
class BHSpellCard {
  steps: Step[]; duration: number; t = 0; idx = 0;
  constructor(def: CardDef) { this.steps = def.steps; this.duration = def.duration ?? Infinity; }
  step(dt: number, em: BHEmitter, field: BHField): boolean { /* …apply due steps… */ return this.t >= this.duration; }
}
Why data beats code. Because a card is a plain list, you can store it, diff it, hot-swap it, generate it — and, in the Simulations tier, author it in a visual editor that emits exactly this structure. "Attacks are data" is the single decision that makes a real bullet-hell content pipeline possible.

🔀 Phases — HP thresholds & screen-clear transitions

A boss with one attack is a target; a boss with phases is a fight. When its HP crosses a threshold, we swap the active spell card and clear the screen so the new pattern starts clean (and the player gets a breath). Shoot the bar down past 50% to trigger phase two.

click the playfield · Z to shoot…
📝 Click to see: a threshold-driven phase swap 
const cards = [phase1Card, phase2Card];
let phase = 0;

// in update(): swap card + screen-clear when HP crosses 50%
if (phase === 0 && boss.hp <= boss.maxHp * 0.5) {
  phase = 1;
  enemy.clear();      // wipe the screen so phase 2 starts fresh
  flash = 0.4;        // a white flash sells the transition
}
cards[phase].step(dt, em, enemy);
em.step(dt, enemy);
const cards: BHSpellCard[] = [phase1Card, phase2Card];
let phase = 0;
if (phase === 0 && boss.hp <= boss.maxHp * 0.5) { phase = 1; enemy.clear(); flash = 0.4; }
cards[phase].step(dt, em, enemy);
em.step(dt, enemy);
The screen-clear is a free design tool. Wiping bullets at a phase change isn't just mercy — it's punctuation. It tells the player "that was a movement; here comes the next," and it stops leftover bullets from the old pattern making the new one unfair.

🎆 Sub-emitters — bullets that spawn bullets

The richest patterns come from nesting. A carrier bullet drifts out, and after a short fuse airbursts into a ring at wherever it happens to be — a bullet that spawns bullets. Add a second emitter orbiting the boss (a turret) and one source produces a layered, composite barrage. Click and weave.

click the playfield…
📝 Click to see: the airburst (a fuse + a ring) 
// carriers are tagged + big; BHBullet.age (from the engine) is the fuse clock.
const FUSE = 0.85;
for (let i = field.bullets.length - 1; i >= 0; i--) {
  const b = field.bullets[i];
  if (b.tag === 'carrier' && b.age >= FUSE) {
    // a one-off emitter at the carrier's position → a ring
    ring(field, { pos: b.pos, angle: Math.random() * Math.PI * 2 }, 10, 130);
    field.bullets.splice(i, 1);    // the carrier is consumed by its burst
  }
}
const FUSE = 0.85;
for (let i = field.bullets.length - 1; i >= 0; i--) {
  const b = field.bullets[i];
  if (b.tag === 'carrier' && b.age >= FUSE) {
    bhFireRing(field, { pos: b.pos, angle: Math.random() * Math.PI * 2 } as BHEmitter, 10, 130);
    field.bullets.splice(i, 1);
  }
}
bhFireRing only needs { pos, angle }. Because the pattern helpers read just those two fields, you can fire a ring from anything — a boss, an orbiting turret, or a dying bullet — by handing them an ad-hoc object. Small, duck-typed interfaces make composition cheap.

💣 The bomb — clear the screen, on a stock

Every danmaku gives the player a panic button. The bomb clears every bullet on screen, grants a second of invulnerability, and costs one of your limited stocks. It's the risk/reward dial of the genre: bomb too early and you waste it; too late and you're already dead. Click, let the spiral fill the screen, then press X.

click the playfield · X to bomb…
📝 Click to see: the bomb is a field operation 
function bomb() {
  if (bombs <= 0) return;
  bombs--;                                   // costs a stock
  invuln = 1.6;                              // i-frames: ignore hits for a moment
  enemy.clear();                             // the screen-clear — one field op
  fx = { x: player.pos.x, y: player.pos.y, r: 0 };  // an expanding shockwave to sell it
}

// in update(): hits only count while not invulnerable
if (invuln <= 0 && bhHitTest(player, b)) { /* you got hit */ }
invuln = Math.max(0, invuln - dt);
function bomb(): void {
  if (bombs <= 0) return;
  bombs--; invuln = 1.6; enemy.clear();
  fx = { x: player.pos.x, y: player.pos.y, r: 0 };
}
Invulnerability windows are everywhere. The same "ignore collisions for N seconds" trick powers the bomb, the moment after you respawn, and the dash in other genres. It's a tiny timer guarding a collision check — but it's most of what makes a game feel fair.

🏆 Mini-Project: "Spell Card Duel"

Everything at once: a two-phase boss, each phase its own spell card, against the full player kit — move, focus, shoot (Z), bomb (X) — with three lives. Graze for points; capture a phase (beat it without being hit and without bombing) for a big bonus. Defeat both phases to win. This is a complete, if compact, bullet-hell boss fight.

click the playfield to begin…
📝 Click to see: the capture bonus 
let faulted = false;                 // got hit OR bombed during this phase?

// a hit or a bomb "faults" the current phase
function onHit()  { lives--; faulted = true; invuln = 1.5; enemy.clear(); }
function bomb()   { bombs--; faulted = true; invuln = 1.6; enemy.clear(); }

// clearing a phase awards the capture bonus only if you stayed clean
function clearPhase() { if (!faulted) { score += 10000; captures++; } }

// reset `faulted` at the start of each phase; graze still scores throughout
if (!b.grazed && bhGrazeTest(player, b)) { b.grazed = true; score += 100; }
let faulted = false;
function clearPhase(): void { if (!faulted) { score += 10000; captures++; } }
// faulted = true on hit or bomb; reset to false when a new phase begins.

🚀 Make it yours

  • Add a third phase with your rose-curve pattern and a faster sweep.
  • Make the capture bonus scale with how little time the phase took.
  • Give the boss a brief "desperation" attack in the last 10% of each phase's HP.
