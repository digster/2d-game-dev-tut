🧭 From one room to a whole dungeon

In the Beginner tier you typed a room out by hand. That doesn't scale, and it kills the genre's main draw: every run is a new dungeon. This tier is the procedural generator. We'll build it in layers, each one a demo you can re-roll with a seed:

  • Rooms — scatter rectangles, rejecting ones that overlap.
  • Corridors — dig tunnels so the rooms form one connected space.
  • BSP — a smarter, tidier way to lay out rooms with no overlap by construction.
  • Caves — a "drunkard's walk" for organic, non-rectangular layouts.
  • Population & connectivity — drop the player, stairs, monsters and items, and use a flood fill to prove the player can actually reach everything.
💡 Same seed, same dungeon. Every generator here draws from RogueRng, so a given seed always carves the exact same map. Type a seed, hit Regenerate, and you can hand that number to a friend and you'll both explore an identical dungeon — the foundation of the "daily challenge" you'll finish in the Simulations tier.

🟫 Placing rooms (and rejecting overlaps)

The simplest generator: try to drop N random rectangles. For each candidate, if it overlaps an already-placed room (plus a one-tile margin so walls don't fuse), throw it away; otherwise carve its interior to floor. Crank the attempts slider — past a point, new rooms can't find space and get rejected, so "attempts" and "rooms placed" diverge.

// Two rooms overlap if their rectangles touch within `margin` tiles.
function roomsOverlap(a, b, margin) {
    return a.x - margin < b.x + b.w && a.x + a.w + margin > b.x
        && a.y - margin < b.y + b.h && a.y + a.h + margin > b.y;
}

function placeRooms(level, rng, { attempts, minSize, maxSize }) {
    const rooms = [];
    for (let i = 0; i < attempts; i++) {
        const w = rng.between(minSize, maxSize);
        const h = rng.between(minSize, maxSize);
        const x = rng.between(1, level.width - w - 1);   // keep a 1-tile wall border
        const y = rng.between(1, level.height - h - 1);
        const room = { x, y, w, h, cx: x + (w >> 1), cy: y + (h >> 1) };

        if (rooms.some(r => roomsOverlap(r, room, 1))) continue;  // reject overlaps
        for (let yy = y; yy < y + h; yy++)
            for (let xx = x; xx < x + w; xx++)
                level.set(xx, yy, Tile.FLOOR);                    // carve interior
        rooms.push(room);
    }
    return rooms;
}
interface Room { x: number; y: number; w: number; h: number; cx: number; cy: number; }

function roomsOverlap(a: Room, b: Room, margin: number): boolean {
    return a.x - margin < b.x + b.w && a.x + a.w + margin > b.x
        && a.y - margin < b.y + b.h && a.y + a.h + margin > b.y;
}
💡 Rooms alone are islands. Notice there are no doorways yet — each room is sealed. A pile of disconnected rooms isn't a dungeon you can walk; the next demo digs the corridors that stitch them together.

🚪 Connecting rooms with corridors

Sort the rooms (here, just placement order) and dig an L-shaped tunnel between each consecutive pair's centers — one horizontal run, one vertical run. That alone guarantees the whole dungeon is connected (every room links to the chain). Toggle the connection graph to see which centers got wired, and flip the elbow direction to vary the tunnel shapes.

📝 Click to see: L-shaped corridor carving 
function carveH(level, x1, x2, y) {            // horizontal run
    for (let x = Math.min(x1, x2); x <= Math.max(x1, x2); x++) level.set(x, y, Tile.FLOOR);
}
function carveV(level, y1, y2, x) {            // vertical run
    for (let y = Math.min(y1, y2); y <= Math.max(y1, y2); y++) level.set(x, y, Tile.FLOOR);
}

function connectRooms(level, rng, rooms, randomElbow) {
    for (let i = 1; i < rooms.length; i++) {
        const a = rooms[i - 1], b = rooms[i];
        // Two ways to bend the L. Picking randomly varies the layout.
        if (randomElbow && rng.chance(0.5)) {
            carveH(level, a.cx, b.cx, a.cy);   // go across first,
            carveV(level, a.cy, b.cy, b.cx);   // then down
        } else {
            carveV(level, a.cy, b.cy, a.cx);   // go down first,
            carveH(level, a.cx, b.cx, b.cy);   // then across
        }
    }
}
function connectRooms(level: Level, rng: RogueRng, rooms: Room[], randomElbow: boolean): void {
    for (let i = 1; i < rooms.length; i++) {
        const a = rooms[i - 1], b = rooms[i];
        if (randomElbow && rng.chance(0.5)) { carveH(level, a.cx, b.cx, a.cy); carveV(level, a.cy, b.cy, b.cx); }
        else { carveV(level, a.cy, b.cy, a.cx); carveH(level, a.cx, b.cx, b.cy); }
    }
}
💡 Connecting consecutive rooms is the cheapest connectivity guarantee. Because room i always links to room i−1, the rooms form one unbroken chain — so the player can always reach every room. Fancier generators connect a minimum spanning tree or add loops for tactical choices, but the chain never disconnects.

🌲 BSP partitioning — the structured classic

Random placement wastes space and clusters awkwardly. Binary Space Partitioning fixes both: recursively split the whole map into two halves (along its longer axis, at a random line), stop when the pieces are small enough, then drop one room into each leaf. Rooms can't overlap because each lives in its own partition. Connect siblings as you unwind the recursion and the dungeon is connected by construction. Toggle the partition overlay to see the tree of splits.

// Recursively split a rectangle into a binary tree of sub-rectangles.
function bspBuild(x, y, w, h, depth, rng, opts) {
    const node = { x, y, w, h };
    const canV = w >= opts.minLeaf * 2;   // room to split into left/right?
    const canH = h >= opts.minLeaf * 2;   // ...into top/bottom?
    if (depth >= opts.maxDepth || (!canV && !canH)) { node.leaf = true; return node; }

    const splitVert = (canV && canH) ? (w >= h) : canV;   // cut the LONGER axis
    if (splitVert) {
        const sx = rng.between(x + opts.minLeaf, x + w - opts.minLeaf);
        node.left  = bspBuild(x, y, sx - x, h, depth + 1, rng, opts);
        node.right = bspBuild(sx, y, x + w - sx, h, depth + 1, rng, opts);
    } else {
        const sy = rng.between(y + opts.minLeaf, y + h - opts.minLeaf);
        node.left  = bspBuild(x, y, w, sy - y, depth + 1, rng, opts);
        node.right = bspBuild(x, sy, w, y + h - sy, depth + 1, rng, opts);
    }
    return node;
}

// Connect on the way back up: link a room from the left subtree to one from
// the right. Returns a representative room so the parent can connect too.
function bspConnect(node, level, rng) {
    if (node.leaf) return node.room || null;
    const a = bspConnect(node.left, level, rng);
    const b = bspConnect(node.right, level, rng);
    if (a && b) { carveV(level, a.cy, b.cy, a.cx); carveH(level, a.cx, b.cx, b.cy); }
    return a || b;
}
interface BspNode { x:number; y:number; w:number; h:number; leaf?:boolean; left?:BspNode; right?:BspNode; room?:Room; }

function bspConnect(node: BspNode, level: Level, rng: RogueRng): Room | null {
    if (node.leaf) return node.room ?? null;
    const a = bspConnect(node.left!, level, rng);
    const b = bspConnect(node.right!, level, rng);
    if (a && b) { carveV(level, a.cy, b.cy, a.cx); carveH(level, a.cx, b.cx, b.cy); }
    return a ?? b;
}
💡 No overlap is structural, not lucky. Because every room is born inside its own leaf rectangle, two rooms can never collide — there's no reject-and-retry like the scatter method. Lower the max-depth slider for a few big rooms; raise it for many small ones. That single knob controls the whole feel of the floor.

🍺 Drunkard's walk — the organic alternative

Rooms and corridors look built. For natural caves, let a "drunkard" stumble around: start somewhere, carve the current tile, step in a random cardinal direction, repeat until enough floor is dug. The wandering path leaves a connected, blobby cavern. Watch it carve live; add walkers for bigger, branchier caves (but note — extra walkers start at random spots, so they can leave disconnected pockets, which the next demo catches).

📝 Click to see: the drunkard's walk carver 
const STEPS = [{dx:1,dy:0},{dx:-1,dy:0},{dx:0,dy:1},{dx:0,dy:-1}];

function drunkardsWalk(level, rng, { targetFloor }) {
    let x = level.width >> 1, y = level.height >> 1;   // start in the middle
    let carved = 0;
    while (carved < targetFloor) {
        if (level.get(x, y) !== Tile.FLOOR) { level.set(x, y, Tile.FLOOR); carved++; }
        const s = rng.pick(STEPS);
        // Clamp inside the 1-tile wall border so we never carve the edge.
        x = Math.max(1, Math.min(level.width - 2, x + s.dx));
        y = Math.max(1, Math.min(level.height - 2, y + s.dy));
    }
}
function drunkardsWalk(level: Level, rng: RogueRng, opts: { targetFloor: number }): void {
    let x = level.width >> 1, y = level.height >> 1, carved = 0;
    const STEPS = [{dx:1,dy:0},{dx:-1,dy:0},{dx:0,dy:1},{dx:0,dy:-1}];
    while (carved < opts.targetFloor) {
        if (level.get(x, y) !== Tile.FLOOR) { level.set(x, y, Tile.FLOOR); carved++; }
        const s = rng.pick(STEPS);
        x = Math.max(1, Math.min(level.width - 2, x + s.dx));
        y = Math.max(1, Math.min(level.height - 2, y + s.dy));
    }
}
💡 One walker is always connected (it never teleports — every carved tile touches the previous one). The moment you add walkers with independent random starts, you risk separate caverns that never meet. That's not a bug to avoid blindly — it's exactly why the next demo's flood fill exists.

🎯 Populating & the connectivity guarantee (flood fill)

A generated map needs a spawn, a down-stairs, and some monsters and items — all dropped on random floor tiles from the seed. But first a hard question: can the player actually reach the stairs? A flood fill from the spawn answers it — it marks every tile reachable on foot. Switch the generator to "cave (multi-walker)" with the reachability overlay on, and watch isolated pockets light up red. "Keep largest region" walls those pockets off so the floor is guaranteed solvable.

// Breadth-first flood fill: which floor tiles can you walk to from (sx, sy)?
function floodFill(level, sx, sy) {
    const seen = new Uint8Array(level.width * level.height);
    const queue = [[sx, sy]];
    seen[level.idx(sx, sy)] = 1;
    let count = 0;
    while (queue.length) {
        const [x, y] = queue.pop();
        count++;
        for (const [dx, dy] of [[1,0],[-1,0],[0,1],[0,-1]]) {
            const nx = x + dx, ny = y + dy, i = level.idx(nx, ny);
            if (level.isWalkable(nx, ny) && !seen[i]) { seen[i] = 1; queue.push([nx, ny]); }
        }
    }
    return { seen, count };   // count === total floor  ⇒  fully connected
}
function floodFill(level: Level, sx: number, sy: number): { seen: Uint8Array; count: number } {
    const seen = new Uint8Array(level.width * level.height);
    const queue: [number, number][] = [[sx, sy]];
    seen[level.idx(sx, sy)] = 1;
    let count = 0;
    while (queue.length) {
        const [x, y] = queue.pop()!;
        count++;
        for (const [dx, dy] of [[1,0],[-1,0],[0,1],[0,-1]]) {
            const nx = x + dx, ny = y + dy, i = level.idx(nx, ny);
            if (level.isWalkable(nx, ny) && !seen[i]) { seen[i] = 1; queue.push([nx, ny]); }
        }
    }
    return { seen, count };
}
💡 Flood fill is the roguelike Swiss-army knife. The same BFS answers "is the level solvable?", "how far is each tile from the player?" (Dijkstra maps, Advanced tier), and "which region is biggest?" (connectivity repair). If reachable-tile-count equals total-floor-count, the map is fully connected — a one-line sanity check worth running on every generated level.
