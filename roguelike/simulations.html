🪨 Cellular-automata caves

Rooms-and-corridors look architected; the drunkard's walk is connected-but-blobby. For natural-looking caverns, start from random noise (≈45% wall) and repeatedly apply one rule: a cell becomes wall if 5+ of the nine cells in its 3×3 neighbourhood (itself included) are walls, else floor — a 5-of-9 majority vote. A few iterations and the noise condenses into smooth caves and pillars. Watch it settle, then "keep largest region" flood-fills away the disconnected pockets.

// Count wall neighbours in the 8-cell ring. Out-of-bounds counts as wall,
// so the cave naturally seals itself against the map border.
function wallNeighbours(level, x, y) {
    let n = 0;
    for (let dy = -1; dy <= 1; dy++)
        for (let dx = -1; dx <= 1; dx++)
            if (!(dx === 0 && dy === 0) && level.get(x + dx, y + dy) === Tile.WALL) n++;
    return n;
}

// One smoothing pass — the 5-of-9 majority rule (count the 3x3 INCLUDING the
// centre). Read from `level`, write a new grid (never read your own
// half-written output) and swap. The 8-neighbour-only variant erodes walls to
// nothing; including the centre makes solid regions self-stabilise into caves.
function caStep(level) {
    const next = level.clone();
    for (let y = 1; y < level.height - 1; y++)
        for (let x = 1; x < level.width - 1; x++) {
            const walls = wallNeighbours(level, x, y) + (level.get(x, y) === Tile.WALL ? 1 : 0);
            next.set(x, y, walls >= 5 ? Tile.WALL : Tile.FLOOR);
        }
    return next;
}

// Full generator: seed noise, smooth N times, then guarantee connectivity.
function caCave(w, h, rng, { fill, iterations }) {
    let level = new Level(w, h, Tile.WALL);
    for (let y = 1; y < h - 1; y++)
        for (let x = 1; x < w - 1; x++)
            level.set(x, y, rng.next() < fill ? Tile.WALL : Tile.FLOOR);
    for (let i = 0; i < iterations; i++) level = caStep(level);
    dgKeepLargest(level);     // flood-fill away disconnected pockets (Intermediate tier)
    return level;
}
function caStep(level: Level): Level {
    const next = level.clone();
    for (let y = 1; y < level.height - 1; y++)
        for (let x = 1; x < level.width - 1; x++)
            next.set(x, y, wallNeighbours(level, x, y) >= 5 ? Tile.WALL : Tile.FLOOR);
    return next;
}
💡 Double-buffer or it lies. If caStep wrote into the same grid it's reading, later cells in the pass would see already-updated neighbours and the rule would smear. Reading level and writing next (then swapping) is the same discipline as Conway's Life and the Noita sand sim — the iteration is only correct on a consistent snapshot.

🎨 Level themes by depth

A great dungeon changes as you go down. A level recipe ties depth to a generator, a palette, and a monster/item table: the upper halls are tidy rooms; the mid caverns switch to cellular caves; the deep is darker and meaner. Drag the depth slider to roll the recipe for that floor — the generator, the wall tint, and the spawn table all change.

const themeForDepth = (d) => THEMES.find(t => d <= t.maxDepth);

// One entry point picks the generator from the recipe — callers don't branch.
function generateThemed(w, h, rng, depth) {
    const theme = themeForDepth(depth);
    const level = theme.gen === 'cave'
        ? caCave(w, h, rng, { fill: 0.45, iterations: 5 })
        : generateDungeon(w, h, rng, { attempts: 30 }).level;
    return { level, theme };
}

🎯 Deterministic seeds & record/replay

The promise the whole track relied on: same seed + same actions ⇒ same outcome, exactly. That's what lets players share a "daily challenge" seed, and what lets you file a bug as "seed 1337, then ↓↓→, the snake should be dead". Click to focus and play a few moves in this small dungeon — every keypress is recorded. Then hit Verify replay: a fresh game is built from the same seed and fed the same inputs, and its final state is compared to yours. It lands bit-identical, every time.

🧬 What "deterministic" really demands

Two things: a seeded RNG (RogueRng — no Math.random() anywhere) and a fixed update order (the energy scheduler processes actors in a stable sequence). Get those right and the entire game becomes a pure function of (seed, inputs) — which is exactly the property the netcode track's rollback and replay systems also depend on. Determinism isn't a feature you add; it's a discipline you keep.
