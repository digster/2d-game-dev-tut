👁️ Why sight changes everything

Until now you could see the whole map. That's not a roguelike — it's a board game. The dread of a dungeon crawl comes from not knowing what's around the corner. This tier builds that, in two halves:

  • What YOU can see. Line of sight answers "is there a clear line to that tile?"; field of view answers the bigger question "which of all tiles can I currently see?"; fog of war remembers where you've been but greys out what you can't see right now.
  • What the MONSTERS can see — and how they chase. The same line-of-sight check decides when a monster notices you; then A* and Dijkstra maps get it to you across the dungeon you can't fully see.
💡 Symmetry you can exploit: sight is mutual. If you can't see the monster, it (usually) can't see you — so breaking line of sight is how you escape. The capstone turns that into a stealth game: duck behind a wall and the hunt goes cold.

📏 Line of sight (Bresenham)

The atom of vision: given two tiles, is the straight line between them clear of walls? Bresenham's line algorithm — the same one used to draw pixel lines — walks the grid cells along that line. If any cell between the endpoints is a wall, sight is blocked. Move your mouse over the map; the line from the @ turns green when clear and red when a wall is in the way.

// Walk the grid cells along the line (x0,y0)->(x1,y1). Sight is clear unless
// a wall sits strictly BETWEEN the endpoints (you can still SEE a wall you bump).
function losLine(level, x0, y0, x1, y1) {
    const cells = [];
    let dx = Math.abs(x1 - x0), dy = Math.abs(y1 - y0);
    let sx = x0 < x1 ? 1 : -1, sy = y0 < y1 ? 1 : -1;
    let err = dx - dy, x = x0, y = y0;
    let clear = true, blockedAt = null;
    while (true) {
        cells.push({ x, y });
        if (x === x1 && y === y1) break;
        // A wall between the endpoints blocks the line.
        if (!(x === x0 && y === y0) && level.isOpaque(x, y)) { clear = false; blockedAt = { x, y }; break; }
        const e2 = 2 * err;
        if (e2 > -dy) { err -= dy; x += sx; }
        if (e2 < dx)  { err += dx; y += sy; }
    }
    return { cells, clear, blockedAt };
}
interface LosResult { cells: {x:number;y:number}[]; clear: boolean; blockedAt: {x:number;y:number} | null; }

}
💡 LOS is cheap but not symmetric-perfect. A single Bresenham line is great for "does this monster see the player?" but if you used it for the player's whole view you'd get ugly artefacts (some tiles visible, their neighbours not). For the player you want a proper field-of-view — the next demo.

🔦 Recursive shadowcasting field-of-view

Field of view answers "which tiles can I see at all?" — and it has to be fast, artefact-free, and symmetric. The classic answer is recursive shadowcasting: split the area around the player into 8 octants, scan each one row by row, and track the slopes of the shadows that walls cast. Tiles inside a shadow are skipped; walls start new shadows. Click to focus, walk the @ around, and watch the lit area flow around corners. The dimmed tiles are out of view.

📝 Click to see: recursive shadowcasting (the 8-octant classic) 
// Eight octant transforms (Björn Bergström's classic shadowcasting).
const FOV_MULT = [
    [1, 0, 0, -1, -1, 0, 0, 1],   // xx
    [0, 1, -1, 0, 0, -1, 1, 0],   // xy
    [0, 1, 1, 0, 0, -1, -1, 0],   // yx
    [1, 0, 0, 1, -1, 0, 0, -1],   // yy
];

function computeFOV(level, ox, oy, radius) {
    const vis = new Uint8Array(level.width * level.height);
    const mark = (x, y) => { if (level.inBounds(x, y)) vis[level.idx(x, y)] = 1; };
    mark(ox, oy);                                   // you always see your own tile
    for (let oct = 0; oct < 8; oct++)
        castLight(level, ox, oy, 1, 1.0, 0.0, radius,
                  FOV_MULT[0][oct], FOV_MULT[1][oct], FOV_MULT[2][oct], FOV_MULT[3][oct], mark);
    return vis;
}

function castLight(level, cx, cy, row, start, end, radius, xx, xy, yx, yy, mark) {
    if (start < end) return;
    const r2 = radius * radius;
    let newStart = 0;
    for (let j = row; j <= radius; j++) {
        let dx = -j - 1, dy = -j, blocked = false;
        while (dx <= 0) {
            dx++;
            const X = cx + dx * xx + dy * xy, Y = cy + dx * yx + dy * yy;
            const lSlope = (dx - 0.5) / (dy + 0.5), rSlope = (dx + 0.5) / (dy - 0.5);
            if (start < rSlope) continue;
            if (end > lSlope) break;
            if (dx * dx + dy * dy < r2) mark(X, Y);     // within the sight circle
            if (blocked) {                               // we're scanning a run of walls
                if (level.isOpaque(X, Y)) { newStart = rSlope; continue; }
                blocked = false; start = newStart;
            } else if (level.isOpaque(X, Y) && j < radius) {
                blocked = true;                          // a wall starts a shadow:
                castLight(level, cx, cy, j + 1, start, lSlope, radius, xx, xy, yx, yy, mark);
                newStart = rSlope;                       // ...recurse for the lit part beside it
            }
        }
        if (blocked) break;
    }
}
type Mark = (x: number, y: number) => void;

function computeFOV(level: Level, ox: number, oy: number, radius: number): Uint8Array {
    const vis = new Uint8Array(level.width * level.height);
    const mark: Mark = (x, y) => { if (level.inBounds(x, y)) vis[level.idx(x, y)] = 1; };
    mark(ox, oy);
    for (let oct = 0; oct < 8; oct++)
        castLight(level, ox, oy, 1, 1.0, 0.0, radius,
                  FOV_MULT[0][oct], FOV_MULT[1][oct], FOV_MULT[2][oct], FOV_MULT[3][oct], mark);
    return vis;
}
// castLight signature identical to the JS version above.
💡 Why slopes, not rays? Casting a ray to every tile re-walks the same near cells over and over and leaves gaps. Shadowcasting instead tracks the angular interval each wall blocks and skips whole shadowed spans at once — it touches every visible tile exactly once, which is why it's the genre standard. Don't be discouraged if the slope bookkeeping looks fiddly; it's the kind of algorithm you implement once, test hard, and never touch again.

🌫️ Fog of war & the remembered map

Real dungeon crawling has three tile states, not two: unseen (pitch black — you've never been there), remembered (you've seen it before; the terrain is drawn dim, but you can't see what's there now), and visible (lit by your current FOV). Monsters and items only render when actually visible — step away and they vanish back into the dark. Click to focus and explore; the map fills in as you go.

click, then explore…

🧠 Two bitmaps, recomputed each turn

visible is thrown away and recomputed (one computeFOV call) every time the player moves. explored is permanent: each turn you OR the new visible set into it. Rendering then just asks each tile which of the three buckets it's in. Monsters check visible before drawing — that single check is the whole "they disappear when you look away" effect.

🧭 Monster pathing with A*

A monster that sees you needs to get to you — around walls, not through them. That's A* pathfinding (from the Fundamentals), now on the dungeon grid. The rat here computes a fresh A* path to the @ each turn only while it has line of sight; lose its line and it stops chasing. Click to focus, move around, and watch its planned path (the dotted trail) bend around corners — a huge upgrade over the Beginner tier's "walk straight at you and get stuck".

💡 A* is perfect for ONE chaser. It finds the shortest path and you can see it planning. But run A* for twenty monsters every turn and you're recomputing twenty overlapping searches toward the same target. For the many-monsters case there's a cheaper trick — the next demo.

🗺️ Dijkstra "scent" maps

Brogue's signature trick: instead of each monster pathing to the player, compute one distance field — the number of steps from every floor tile to the player — and let every monster just step to its lowest-numbered neighbour. One flood fill serves any number of monsters. It's the Intermediate tier's flood fill, but recording distance instead of just "reached". Toggle the arrows to see the gradient every monster rolls down; toggle "flee map" to negate it (cowards run uphill, away from you). Click to focus and move — the two chasers follow the scent.

📝 Click to see: the Dijkstra distance map (multi-source BFS) 
// Distance (in steps) from every floor tile to the nearest source.
// For the "scent" map the only source is the player's tile.
function dijkstraFrom(level, sources) {
    const dist = new Float32Array(level.width * level.height).fill(Infinity);
    const queue = [];
    for (const s of sources) { dist[level.idx(s.x, s.y)] = 0; queue.push([s.x, s.y]); }
    for (let head = 0; head < queue.length; head++) {     // BFS (uniform step cost)
        const [x, y] = queue[head], d = dist[level.idx(x, y)];
        for (const [dx, dy] of [[1,0],[-1,0],[0,1],[0,-1]]) {
            const nx = x + dx, ny = y + dy;
            if (!level.isWalkable(nx, ny)) continue;
            if (d + 1 < dist[level.idx(nx, ny)]) { dist[level.idx(nx, ny)] = d + 1; queue.push([nx, ny]); }
        }
    }
    return dist;
}

// A monster chases by stepping to the neighbour with the SMALLEST distance.
function stepDownhill(level, m, dist) {
    let best = null, bestD = dist[level.idx(m.x, m.y)];
    for (const [dx, dy] of [[1,0],[-1,0],[0,1],[0,-1]]) {
        const nx = m.x + dx, ny = m.y + dy;
        if (level.isWalkable(nx, ny) && dist[level.idx(nx, ny)] < bestD) { bestD = dist[level.idx(nx, ny)]; best = { dx, dy }; }
    }
    return best;   // null if already adjacent / no downhill step
}
function dijkstraFrom(level: Level, sources: {x:number;y:number}[]): Float32Array {
    const dist = new Float32Array(level.width * level.height).fill(Infinity);
    const queue: [number, number][] = [];
    for (const s of sources) { dist[level.idx(s.x, s.y)] = 0; queue.push([s.x, s.y]); }
    for (let head = 0; head < queue.length; head++) {
        const [x, y] = queue[head], d = dist[level.idx(x, y)];
        for (const [dx, dy] of [[1,0],[-1,0],[0,1],[0,-1]]) {
            const nx = x + dx, ny = y + dy;
            if (level.isWalkable(nx, ny) && d + 1 < dist[level.idx(nx, ny)]) {
                dist[level.idx(nx, ny)] = d + 1; queue.push([nx, ny]);
            }
        }
    }
    return dist;
}
💡 One map, N monsters, and flee for free. Recompute the field once per turn (the player is the only thing that moved). Chase = roll downhill. Flee = multiply the whole field by −1.2 and re-roll downhill, and cowards flow away while naturally avoiding dead ends. Brogue runs its entire bestiary on a handful of these maps.
