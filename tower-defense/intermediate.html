🎯 From one tower to a toolkit

The Beginner tier had one tower and one rule: shoot the creep furthest along. That's enough to be a game, but not enough to be a puzzle. A real TD asks you to choose — which kind of tower, aimed by which rule, placed where. This tier builds that toolkit, and every piece is a small, sharp idea: a tower is a row of data; a targeting mode is one line that picks a different creep; leading a target is one quadratic; splash and slow are one extra effect on a hit.

💡 The entity model moved to the engine. TDEnemy, TDTower and TDProjectile were taught inline in the Beginner tier; now that a second tier uses them, they live in engine/entities.js (loaded by both pages). They gained optional powers here — a timed slow, lead aiming, splash — all behind defaults, so the Beginner code that built a plain gun still works untouched.

🧰 A tower is just data

A "tower kind" isn't a subclass — it's a spec object: a range, a fire-rate, a damage, a projectile speed, a cost. Hand one to new TDTower(x, y, spec) and you have that kind. Below, one tower object takes three different specs — a balanced Gun, a long-range hard-hitting slow Sniper, and a short-range spray Rapid. Nothing about the code changes; only the data does.

📝 Show the code: towers are a table 
const TD_TOWER_TYPES = {
  gun:    { range: 115, fireRate: 1.8, damage: 2, cost: 40 },
  sniper: { range: 205, fireRate: 0.6, damage: 8, cost: 70 },  // far, hard, slow
  rapid:  { range: 95,  fireRate: 5.0, damage: 1, cost: 50 },  // near, weak, fast
};
const tower = new TDTower(x, y, TD_TOWER_TYPES.sniper);  // behaviour = data

🎯 Targeting modes — the strategy slot

Which creep should a tower shoot? There's no single right answer — it's a strategy you pick. First (furthest along) races leaks to the goal; last softens newcomers; closest maximizes uptime; strongest and weakest trade focus-fire for finishing blows. Each mode is just a different number we maximize — the whole "strategy slot" is one switch. The tower below doesn't shoot; the reticle shows the creep each mode would pick (HP bars vary so strongest/weakest differ).

📝 Show the code: tdPickTarget 
function tdPickTarget(tower, enemies, mode) {
  let best = null, score = -Infinity;
  for (const e of enemies) {
    if (!e.alive || !tower.inRange(e)) continue;
    let s;
    switch (mode) {
      case 'last':      s = -e.dist; break;
      case 'closest':   s = -tower.pos.distanceSquared(e.pos); break;
      case 'strongest': s = e.hp; break;
      case 'weakest':   s = -e.hp; break;
      default:          s = e.dist; break;   // 'first' (furthest along)
    }
    if (s > score) { score = s; best = e; }   // maximize the chosen score
  }
  return best;
}

🧮 Lead-the-target — the intercept quadratic

A projectile takes time to arrive, so aiming at where a creep is means hitting where it was. To hit a moving target you must aim where it will be. Given the creep at P moving with velocity V, and a shot of speed s from the tower at T, we need the time t when |P + V·t − T| = s·t. Squaring gives a quadratic in t — solve it, aim at P + V·t, and the shot connects. Toggle lead off and drop the shot speed: watch the shots trail behind, and the hit-rate collapse.

📝 Show the code: the intercept solve 
// solve (|V|² − s²)·t² + 2(D·V)·t + |D|² = 0  for the intercept time t,
// where D = P − T (tower→target). The sign of (|V|²−s²) is the physics:
// faster shot ⇒ a solution always exists; slower ⇒ it can be uncatchable.
function tdInterceptTime(D, V, s) {
  const a = V.lengthSquared() - s * s;
  const b = 2 * (D.x * V.x + D.y * V.y);
  const c = D.lengthSquared();
  const disc = b * b - 4 * a * c;
  if (disc < 0) return null;                 // too slow to ever catch it
  const t = (-b - Math.sqrt(disc)) / (2 * a);
  return t > 0 ? t : null;
}
// aim at the predicted spot (build a NEW vector — don't mutate target.pos):
const aim = new Vector2D(target.x + V.x * t, target.y + V.y * t);

💣 Splash towers — one shot, many creeps

Single-target towers stall against a crowd. A splash tower's shot deals its damage to every creep within a blast radius on impact — the answer to clustered waves. It's a tiny change to the hit code: instead of damaging one target, loop the creeps and damage all within range. Toggle splash off and watch the tight cluster sail through.

📝 Show the code: area damage on hit 
_hit(primary, enemies) {
  if (this.splash > 0) {                       // AREA damage
    for (const e of enemies)
      if (e.alive && this.pos.distanceSquared(e.pos) <= this.splash * this.splash)
        e.damage(this.damage);
  } else {
    primary.damage(this.damage);               // single target
  }
}

❄️ Frost towers — slow is damage

A frost tower barely scratches a creep, but it applies a timed slow — and a slowed creep spends longer in everyone else's range, eating more shots. That's the genre's first real synergy: the frost tower's value is the extra DPS it hands your gun towers. Below, the teal frost tower chills creeps (they tint and crawl) and the gun downstream cleans up. Tune the slow's strength and duration.

📝 Show the code: a timed slow status 
// on the creep: keep the STRONGEST slow and the LONGEST remaining time
applySlow(factor, duration) {
  this.slowFactor = Math.min(this.slowFactor, factor);
  this.slowTimer  = Math.max(this.slowTimer, duration);
}
get curSpeed() { return this.speed * this.slowFactor; }  // crawls while chilled
// (the slowTimer ticks down each step; at 0 the factor resets to 1)

🛣️ Smooth spline lanes you can drag

A jagged waypoint lane works, but a curved one reads better and plays differently. A Catmull-Rom spline passes through its control points (unlike a Bézier, whose handles only pull toward them), so you drop points and get a smooth lane through them. Because the lane is still arc-length parameterized, creeps follow the curve with the exact same "advance one scalar" trick. Drag the gold handles — the lane re-smooths live and the creeps follow.

📝 Show the code: a smooth lane creeps follow 
// a Catmull-Rom spline passes THROUGH its control points
const path = new TDPath(controlPoints, { smooth: true });

// drag a handle → rebuild → point the creeps at the new curve. Arc-length
// parameterization means "dist += speed*dt" still works on the new shape.
function rebuild() {
  path = new TDPath(controlPoints, { smooth: true });
  for (const creep of creeps) creep.path = path;
}
