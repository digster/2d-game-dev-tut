📊 From "it works" to "it's balanced"

A tower defense is two curves in tension: the threat the waves throw at you, and the power your gold can buy. Fun lives in the narrow band where they nearly meet. This finale builds the tools to see that band — generate the threat curve, model the economy, price upgrades, and chart which tower is efficient against which enemy — and the one guarantee they all rest on: determinism. A seeded run reproduces exactly, so a measurement means something. Then we put it all into one complete game.

💡 Determinism is the foundation of balancing. If a wave plays out differently each time, "can I beat it with this build?" has no answer. A seeded tdRng (mulberry32 — the same PRNG the bullet-hell and roguelike tracks use) makes a run a pure function of its seed, so two runs match bit-for-bit. The threat heatmap below proves it.

🌊 Wave generation — difficulty as a curve

A wave isn't hand-placed; it's generated from the wave number. Count and HP grow each wave, fast creeps appear partway in, and every fifth wave ends with a tanky boss. The generator is pure — wave N is always the same — so the difficulty curve is the design you tune, not a list you write. Drag the growth rates and watch the threat curve bend.

📝 Show the code: a pure wave generator 
function tdGenerateWave(n) {
  const count = Math.round(6 * (1 + 0.20 * (n - 1)));       // grows each wave
  const hp = 5 * Math.pow(1.22, n - 1);                      // compounding HP
  const q = [];
  for (let i = 0; i < count; i++) {
    if (n % 5 === 0 && i === count - 1) q.push(boss(hp));   // boss every 5th wave
    else if (n >= 3 && i % 4 === 0)     q.push(fast(hp));   // fast creeps mixed in
    else                                 q.push(normal(hp));
  }
  return q;   // pure: wave N is always identical → the CURVE is the design
}

💰 Economy — income, interest & ROI

Gold is the real resource, and the deepest decision in many TDs is when to spend it. Add interest (a percentage of your banked gold each wave) and saving compounds — but you have to survive while under-defended to reap it. That tension between spending now (safety) and banking (power later) is the whole eco game. The curve shows banked-with-interest pulling away.

⬆️ Upgrades & the slow→DPS synergy

Spending gold on a tower you own is often better than a new one — upgrades scale its damage and range, and the dashboard tracks the DPS you're buying per gold. The deepest payoff is synergy: a frost tower barely scratches anything, but a creep slowed to half speed spends twice as long in every other tower's range — so frost doubles their effective DPS. Toggle it and watch the effective number jump.

📝 Show the code: the synergy math 
// a creep slowed to `slowFactor` speed spends 1/slowFactor longer in range,
// so it eats 1/slowFactor as many shots → effective DPS is multiplied:
const dps = damage * fireRate;
const effectiveDps = slowed ? dps / slowFactor : dps;   // 0.5 speed → 2× DPS

// that's why a low-damage Frost tower is a force multiplier, not a damage dealer.

📈 Balancing dashboard — DPS-per-gold & time-to-kill

Balance is a table, not a feeling. DPS-per-gold ranks towers by cost-efficiency; time-to-kill (enemy HP ÷ DPS) ranks them against a specific enemy. Pick an enemy and the picture flips — the cheap spray that shreds a swarm is hopeless against a tank, where the expensive sniper wins. This is the lens a designer tunes the roster through.

📝 Show the code: the balancing metrics 
const dps = tower.damage * tower.fireRate * (tower.splash ? 3 : 1); // ~3 hit by splash
const dpsPerGold = dps / tower.cost;          // cost-efficiency
const timeToKill = enemy.hp / dps;            // seconds to drop this enemy
// rank the roster by these to balance it — no playtesting required to spot outliers

🔥 Threat heatmap & deterministic replay

Where should you build? A threat heatmap answers it: simulate one wave and color each cell by the creep-HP-seconds that pass through it — the hottest cell is the highest-value tower spot (the white box). And because the sim is driven by a seeded tdRng, it's deterministic: re-run the same seed and you get a bit-for-bit identical heatmap (the badge proves it). Same seed, same run — the bedrock of every measurement on this page.

📝 Show the code: determinism = reproducibility 
// drive the whole sim from a seeded PRNG → a run is a pure function of its seed
const a = simulateWave(seed);
const b = simulateWave(seed);
// a and b are bit-for-bit identical — which is the ONLY reason a balancing
// measurement (or a "watch replay") means anything at all.
