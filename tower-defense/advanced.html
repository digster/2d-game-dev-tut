🧠 Two ways to think about "the way to the goal"

Until now the lane was fixed — the creeps walked a path you didn't control. This tier hands navigation to the creeps, and there are two classic answers, costed for two different swarms:

  • A* pathfinding computes one shortest route per creep from where it is. Drop a tower and it re-plans around it — so your towers become a maze. Great for a few smart creeps (Kingdom Rush).
  • Flow fields run a single sweep from the goal and tag every cell with a "downhill" arrow. A thousand creeps each do one lookup to steer — so a whole crowd flows across open ground (Bloons). Cheap at scale, shared by everyone.

Plus the tower's own version of "can I reach it?": a raycast for line-of-sight, so a wall is real cover. These three — A*, flow fields, line-of-sight — are this tier's lessons, written as top-level functions you can call straight from the console.

💡 The same Dijkstra, two outputs. A flow field's first half is an integration field — the distance from every cell to the goal (a BFS, since each step costs 1). Its second half turns those distances into vectors: point each cell at its lowest-distance neighbour. The first is "how far"; the second is "which way".

🧭 A* — the shortest route on a grid

A* explores cells in order of f = g + h: g is the steps taken so far, h is an optimistic guess of the steps remaining (here the Manhattan distance, which can never overestimate on a 4-connected grid — so the first route A* completes is guaranteed shortest). Drag to paint walls and watch the route bend around them; wall it off completely and A* returns null.

📝 Show the code: A* on a grid 
function tdAStar(grid, start, goal) {
  const h = (c, r) => Math.abs(c - goal.col) + Math.abs(r - goal.row); // Manhattan
  const open = [{ ...start, g: 0, f: h(start.col, start.row) }];

🧱 Maze TD — towers are the walls

If a tower blocks a cell, the creeps must go around it — so where you build is how you shape the route. That's "mazing": stretch the path so creeps spend longer in your fire. But there's a catch — you must never be allowed to wall the goal off completely. Before each placement we run the connectivity guard: tentatively occupy the cell, ask A* if a route still exists, and refuse the build if it doesn't. Click to place; try to seal the goal and watch it reject you.

📝 Show the code: the connectivity guard 
// would dropping a tower here seal the goal off? (run A* on a hypothetical)
function tdBlocksPath(grid, col, row, start, goal) {
  grid.occupy(col, row, true);                  // pretend we built here
  const stillReachable = tdAStar(grid, start, goal) !== null;
  grid.occupy(col, row, false);                 // undo the hypothetical
  return !stillReachable;                        // true ⇒ refuse the placement
}
// on a real placement we re-plan every creep from its CURRENT cell to the goal

👁️ Line-of-sight — a wall blocks the shot

A tower with a creep in range still shouldn't fire through a wall. Line-of-sight is a raycast: march the segment from tower to target and fail the instant it crosses a wall cell. Below, the sight line glows green when clear and red when blocked — and the tower only fires on green. Drag a wall between the tower and the lane.

📝 Show the code: tdLineOfSight 
// march the segment in sub-cell steps; a wall anywhere on it blocks the shot
function tdLineOfSight(grid, ax, ay, bx, by) {
  const steps = Math.ceil(Math.hypot(bx - ax, by - ay) / (grid.tile * 0.33));
  for (let i = 1; i < steps; i++) {                  // skip the endpoints
    const t = i / steps;
    const c = grid.worldToCell(ax + (bx - ax) * t, ay + (by - ay) * t);
    if (c && grid.isBlocked(c.col, c.row)) return false;  // wall in the way
  }
  return true;                                         // clear shot
}

🌊 Flow fields — one sweep, every cell knows the way

A flow field is computed once, from the goal. A BFS sweep fills every cell with its distance (the heatmap — bright near the goal, dim far away); then each cell points at its lowest-distance neighbour (the arrows). Now any creep, anywhere, reads its cell's arrow and walks — no per-creep search at all. Drag to paint walls and watch the whole field re-flow around them.

📝 Show the code: the integration field → flow field 
// 1) integration field: ONE BFS from the goal → distance for every cell
cost[goal] = 0;
for (let frontier = [goal]; frontier.length; frontier = next) {
  const next = [];
  for (const cur of frontier)
    for (const nb of walkableNeighbours(grid, cur))
      if (cost[nb] === Infinity) { cost[nb] = cost[cur] + 1; next.push(nb); }
}
// 2) flow field: each cell → a unit vector toward its lowest-cost neighbour
//    (a creep reads ONE vector and steers — any number of creeps, O(1) each)

👥 The open field — one field, a whole crowd

This is why flow fields exist. There's no lane and no per-creep pathfinding here — just one field and a crowd of creeps, each reading the arrow under its feet. Crank the spawn rate to hundreds and it stays smooth, because adding a creep costs a single array lookup per step. Paint walls to split the stream into channels.

📝 Show the code: open-field steering 
// no path — read the field at your cell and move along its vector
function tdStepFlowCreep(c, field, grid, dt) {
  const cell = grid.worldToCell(c.x, c.y);
  const v = field.flow[cell.row * grid.cols + cell.col];   // precomputed
  c.x += v.x * c.speed * dt;
  c.y += v.y * c.speed * dt;
}
