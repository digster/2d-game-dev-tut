🖼️ Render-to-Texture (FBOs)

A framebuffer object (FBO) is an off-screen render target backed by a texture. Bind it, draw the scene → the result lands in a texture instead of the screen. Then draw that texture to the screen. The screen is just another texture — that realisation is the whole of post-processing.

The GL1 ↔ GL2 fragment header (the only real dialect diff)

WebGL2 needs #version 300 es as the literal first line (a leading space or comment is a hard compile error), and replaces gl_FragColor/texture2D with a declared out + texture():

// WebGL1 — GLSL ES 1.00
precision mediump float;
uniform sampler2D u_prev;
void main() {
    gl_FragColor = texture2D(u_prev, gl_FragCoord.xy / vec2(800.0, 450.0));
}
#version 300 es
precision mediump float;
uniform sampler2D u_prev;
out vec4 fragColor;
void main() {
    fragColor = texture(u_prev, gl_FragCoord.xy / vec2(800.0, 450.0));
}

🏓 Ping-Pong Buffers

A pass can't read and write the same texture — that's a framebuffer feedback loop (undefined). For iterative effects you need two FBOs: read A → write B, then swap. Each extra ping-pong pass here blurs a little more; watch the buffers do the work.

📝 Click to see: N ping-pong passes (WebGL1) 
// chainFor('ping', {iters}) → [BOX, BOX, …]. makeFXChain ping-pongs
// two scratch FBOs between every pass automatically.
const st = { iters: 4 };
const toy = makeFXChainGL1(canvas, SCENE_GL1, chainFor1('ping', st), { info: info });
// more iterations → blurrier; each is a separate buffer↔buffer pass.
const st = { iters: 4 };
const toy = makeFXChainGL1(canvas, SCENE_GL1, chainFor1('ping', st), { info: info });
Two FBOs swapped each pass.

// identical control flow on a webgl2 context + GLSL ES 3.00 passes.
const st = { iters: 4 };
const toy = makeFXChainGL2(canvas, SCENE_GL2, chainFor2('ping', st), { info: info });

🌁 Separable Blur

A 2D Gaussian is the product of two 1D ones, so an N×N blur becomes a cheap horizontal pass then a vertical pass (2N taps, not N²). This is the workhorse behind soft shadows, depth-of-field, and the next section's bloom.

📝 Click to see: separable Gaussian (WebGL1) 
// 9-tap weights, unrolled (constant — the GLSL ES 1.00 loop rule):
const BLUR_H = PHEAD_GL1 + `void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec2 d = vec2(u_param, 0.0) / u_resolution;
  vec4 s = texture2D(u_prev, uv) * 0.227;
  s += (texture2D(u_prev, uv+d) + texture2D(u_prev, uv-d)) * 0.194;
  // … 2d,3d,4d taps … then a vertical twin BLUR_V.
  gl_FragColor = s;
}`;
const toy = makeFXChainGL1(canvas, SCENE_GL1, [BLUR_H, BLUR_V], { info: info, param: 2.0 });
const toy = makeFXChainGL1(canvas, SCENE_GL1, [BLUR_H, BLUR_V], { info: info, param: 2.0 });
// u_param = blur radius; setParam() changes it with no recompile.
Separable Gaussian — H then V pass.
📝 Click to see: separable Gaussian (WebGL2) 
const BLUR_H = PHEAD_GL2 + `void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec2 d = vec2(u_param, 0.0) / u_resolution;
  vec4 s = texture(u_prev, uv) * 0.227;
  s += (texture(u_prev, uv+d) + texture(u_prev, uv-d)) * 0.194;
  fragColor = s;
}`;
const toy = makeFXChainGL2(canvas, SCENE_GL2, [BLUR_H, BLUR_V], { info: info, param: 2.0 });

✨ Bloom

The glow around bright things: bright-pass (keep only the brightest pixels) → blur them → add back onto the original scene. Four passes — the canonical multi-pass effect; the composite reads both the blurred bright-pass and the original scene texture.

💡 8-bit caveat: these FBOs are RGBA8, not float — so bright-pass is an approximation (no true HDR). Real engines use half-float targets; the technique is identical.
📝 Click to see: bright → blur → composite (WebGL1) 
// 4 passes: bright-pass, blur H, blur V, additive composite.
const COMPOSITE = PHEAD_GL1 + `void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec3 base  = texture2D(u_scene, uv).rgb;   // original scene
  vec3 bloom = texture2D(u_prev,  uv).rgb;   // blurred bright-pass
  gl_FragColor = vec4(base + bloom * u_param, 1.0);
}`;
const toy = makeFXChainGL1(canvas, SCENE_GL1,
    [BRIGHT, BBLUR_H, BBLUR_V, COMPOSITE], { info: info, param: 1.4 });
const toy = makeFXChainGL1(canvas, SCENE_GL1,
    [BRIGHT, BBLUR_H, BBLUR_V, COMPOSITE], { info: info, param: 1.4 });
// u_scene is always bound to the original scene FBO for the composite.
Bright-pass → blur → additive composite.
📝 Click to see: bright → blur → composite (WebGL2) 
const COMPOSITE = PHEAD_GL2 + `void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec3 base  = texture(u_scene, uv).rgb;
  vec3 bloom = texture(u_prev,  uv).rgb;
  fragColor = vec4(base + bloom * u_param, 1.0);
}`;

🎞️ Colour Grade + CRT

The final full-screen pass gives a game its "look": exposure, contrast, saturation, a tint — plus the retro screen FX (scanline, vignette, chromatic aberration). One pass, a fistful of one-liners; presets just swap the constants.

📝 Click to see: the grade pass (WebGL1) 
// Chromatic aberration = sample R/G/B at slightly offset UVs.
//   c *= exposure; c = (c-0.5)*contrast + 0.5;
//   c = mix(vec3(luma), c, saturation) * tint;
//   c *= 0.85 + 0.15*sin(gl_FragCoord.y * scan);   // scanline
//   c *= 1.0 - dot(q,q) * vignette;                 // vignette

🎬 Scene Transitions

A level wipe, a death iris-out, a dissolve into the next room — every scene transition is a single full-screen post pass that masks the rendered scene by a time-driven progress value. Same chain as bloom/grade; only the mask shape changes.

One pass, four masks

  • Wipe: smoothstep(t, t+ε, uv.x) — a moving edge.
  • Iris: a growing/shrinking circle (radius = t).
  • Dissolve: per-pixel hash thresholded by t.
  • Bars: step(fract(uv.y·n), t) — venetian blinds.

Progress here ping-pongs via 0.5 - 0.5·cos(u_time) so it plays in and out.

📝 Click to see: scene transition (WebGL1) 
// chainFor('transition', st) → [one post pass]. The mask multiplies the
// scene; progress t comes from u_time so it loops in/out automatically.
const st = { mode: 'wipe' };
const toy = makeFXChainGL1(canvas, SCENE_GL1, chainFor('transition', st), { info: info });
// btnTransGL1Iris → st.mode='iris'; toy.rebuild(SCENE_GL1, chainFor('transition', st));
const st = { mode: 'wipe' };
const toy = makeFXChainGL1(canvas, SCENE_GL1, chainFor('transition', st), { info: info });
A scene-change wipe as one post pass, driven by u_time.

☀️ Radial Blur & God Rays

Sample the scene repeatedly along the line toward a centre point and average: that's a radial (zoom) blur — speed lines, hit-impact punch, warp. Add a decay weight and a brighten and the same loop becomes god rays / light shafts.

🟪 Pixelation / Mosaic

Snapping uv to a coarse grid before sampling downscales the whole scene into chunky pixels — retro pixelation, a censorship mosaic, or a stylised low-res look. A bevel term makes a tile mosaic; a per-row sine makes a CRT-ish low-res screen.

📝 Click to see: pixelation (WebGL1) 
// q = (floor(uv * n) + 0.5) / n  → every pixel in a cell reads one texel.

🌈 Chromatic Aberration & RGB-Shift

Real lenses focus red, green and blue at slightly different points — sampling each channel at a small offset fakes that chromatic aberration. Offset radially (zero at the centre, growing to the edges) for a lens look; offset with a constant lateral shift + scanline jitter for the digital RGB-shift glitch. One post pass, three texture reads.

📝 Click to see: chromatic aberration (WebGL1) 
// c.r = TEX(u_prev, uv + d); c.g = TEX(u_prev, uv); c.b = TEX(u_prev, uv - d);
// 'ca': d = (uv-0.5)*k (radial)   'shift': d = (k+jitter, 0) (lateral)
const st = { mode: 'ca' };
const toy = makeFXChainGL1(canvas, SCENE_GL1, chainFor('chroma', st), { info: info, param: 1.0 });
const st = { mode: 'ca' };

📺 Datamosh / Block Glitch

The corrupted-video aesthetic. Quantise time into "keyframes", hash a coarse block grid, and on a fraction of blocks displace the sample and tear the colour channels. It's a single pass — no cross-frame feedback — yet reads convincingly as tape datamosh, perfect for a damage / hack / teleport stinger.

📝 Click to see: datamosh glitch (WebGL1) 
// seg = floor(u_time*2.0); block hash > threshold → displace + channel tear.
// 'mosh' = sparse blocks; 'heavy' = lower threshold + larger amplitude.
const st = { mode: 'mosh' };
const toy = makeFXChainGL1(canvas, SCENE_GL1, chainFor('glitch', st), { info: info, param: 1.0 });
const st = { mode: 'mosh' };

🎛️ Mini-Project: Post-Process Stack

Compose the tier into a real engine post chain over the live scene: scene → [bloom] → colour-grade/CRT. Toggle the bloom stage and swap grade presets — the chain rebuilds, the FBO plumbing never changes.

📝 Click to see: the composed stack (WebGL1) 
function build(st) {
  const post = [];
  if (st.bloom) post.push(BRIGHT, BBLUR_H, BBLUR_V, COMPOSITE);
  post.push(GRADE(PRESETS[st.preset]));
  return post;
}
const toy = makeFXChainGL1(canvas, SCENE_GL1, build(st), { info: info, param: 1.4 });
// a button: st.bloom = !st.bloom; toy.rebuild(SCENE_GL1, build(st));
const toy = makeFXChainGL1(canvas, SCENE_GL1, build(st), { info: info, param: 1.4 });
scene → [bloom] → grade, composed.
