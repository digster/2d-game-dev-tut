The first six demos warp one shared dscene(uv) (palette + grid + ring) so the distortion is unmistakable. DISTORTION

🌀 Swirl / Pinch / Bulge

Convert uv to polar coordinates around a centre (the mouse), tweak the radius or angle, convert back. A radius-decaying angle gives a swirl; shrinking the radius is a pinch; pow(r, <1) is a bulge magnifier.

Polar remap

d = uv - centre; r = length(d); a = atan(d.y, d.x);
a += swirlAmount * exp(-r * k);   // decays outward
uv' = centre + vec2(cos(a), sin(a)) * r;
📝 Click to see: the swirl shader 
const WARP = {
  swirl: 'a += (0.6 + 1.8 * u_param) * exp(-r * 4.0);',
  pinch: 'r *= mix(1.0, smoothstep(0.0, 0.5, r), 0.5 + 0.9 * u_param);',
  bulge: 'r = pow(r, 1.0 - 0.55 * clamp(u_param, 0.0, 0.95));'
};  // centre = u_mouse / u_resolution
// resample dscene() at the remapped uv → the image appears to bend.
Polar remap around the cursor — move the mouse over the canvas.

🔭 Fisheye / Barrel / Pincushion

Lens distortion is a radial scale that grows with r²: uv' = c + (uv-c)·(1 + k·r²). Negative k bows the edges out (barrel / fisheye); positive pinches them in (pincushion).

📝 Click to see: the lens shader 
vec2 c = uv - 0.5;
float r2 = dot(c, c);
float k = sign * (0.35 + 1.4 * u_param);
vec2 luv = 0.5 + c * (1.0 + k * r2);   // r² → corners distort most
// sign = -1 barrel/fisheye, +1 pincushion.
A radial scale that grows with r² — fisheye vs pincushion.

🔆 Kaleidoscope / Mirror Symmetry

Fold the polar angle into N wedges with mod, then mirror inside each wedge — instant radial symmetry. A staple of menu backdrops, portals and music visualisers.

Fold & mirror the angle

wedge = 2π / N;
a = mod(a, wedge);
a = abs(a - wedge * 0.5);   // reflect within the wedge

Segment count is a shader constant → the buttons rebuild via setFrag.

📝 Click to see: the kaleidoscope shader 
float wedge = 6.28318 / ${seg}.0;     // ${seg} is a constant → rebuild
a = mod(a, wedge);
a = abs(a - wedge * 0.5);
vec2 kuv = 0.5 + vec2(cos(a), sin(a)) * r;
// a slow u_time term makes the symmetry rotate.
Fold the angle into N mirrored wedges → radial symmetry.

🌈 Chromatic Aberration & RGB-Shift

Sample R, G and B at slightly different UVs. Offset radially for a real-lens chromatic aberration; offset with a wobbling lateral vector for the digital RGB-shift glitch.

📝 Click to see: the RGB-shift shader 
col.r = dscene(uv + dir * k).r;
col.g = dscene(uv).g;
col.b = dscene(uv - dir * k).b;   // dir = (uv-0.5) radial, or lateral
// k = 0.004 + u_param * 0.03 — channel separation amount.
Sample each channel at a different uv — lens CA vs digital shift.

📼 VHS / Tape Distortion

Stack the analogue artefacts: per-scanline horizontal jitter, a slow tape wobble, chroma bleed, scanlines and grain — the retro found-footage look.

📝 Click to see: the VHS shader 
float jit = (h11(band + floor(u_time*12.0)) - 0.5) * 0.03 * A;
float wob = sin(uv.y * 8.0 + u_time * 2.0) * 0.004 * A;
vec2 suv = uv + vec2(jit + wob, 0.0);
// + chroma bleed, scanlines, grain
// A is the severity (normal vs heavy chewed-up tape).
Jitter + wobble + chroma bleed + scanlines + grain.

📺 Datamosh / Block Slice Glitch

Quantise time into "keyframes", hash horizontal slices and a coarse block grid, and on a fraction of them displace the sample and tear the channels — the corrupted-codec aesthetic, a perfect damage / hack / teleport stinger.

📝 Click to see: the datamosh shader 
float seg = floor(u_time * 3.0);          // keyframes
float slice = floor(uv.y * 14.0);
float on = step(thr, fract(h2(vec2(slice, seg)) * 7.0 + u_time));
g.x += (hs - 0.5) * 0.35 * on;            // slice tear
// + a coarse block displacement on top, + channel tear.
Time-quantised slice + block displacement with channel tear.

🎏 Mini-Project: Vertex FX — Jelly / Wind / Flag

Every other demo in the whole track is a full-screen fragment quad. This one is different: a real triangulated mesh whose vertices are moved in the vertex shader. That's how games do flags, foliage sway, jelly sprites, water surfaces and squash-and-stretch — geometry, not pixels.

A new (tiny) harness: makeMeshToy

makeShaderToy only ever draws two triangles, so it physically can't do this. makeMeshToy builds a 56×56 grid mesh with a_position + a_uv, runs your vertex + fragment shaders, and mirrors the same handle shape (plus rebuild({vert,frag})) so lazyToy drives it unchanged — zero edits to the shared runtime.

💡 Displacement = animation: the vertex shader offsets each vertex by a sin wave of its UV and u_time, and passes the displacement to the fragment shader as a shading term so the folds catch light. Drag on the canvas to grab the mesh.
📝 Click to see: the vertex-FX demo 
// VERTEX shader (not fragment!) — it moves geometry:
const DISP = {
  flag:  'float dz = sin(a_uv.x*9.0 - u_time*6.0)*0.09*a_uv.x*(0.5+u_param);',
  jelly: 'float dz = sin(a_uv.x*6.0+u_time*5.0)*sin(a_uv.y*6.0+u_time*4.0)*0.10*(0.4+u_param);'
};
// p.y += dz;  v_shade = 0.5 + dz*3.0;  gl_Position = vec4(p,0,1);
const toy = makeMeshToy(canvas, buildVert('flag'), FRAG, { info });
type Mode = 'calm' | 'jelly' | 'wind' | 'flag';
// makeMeshToy returns { stop, setParam, setPaused, rebuild, destroy }.
A real mesh deformed in the vertex stage. Drag to grab it.
