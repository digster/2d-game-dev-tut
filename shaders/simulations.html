💡 New this tier — persistent state & WebGL2 float textures. Until now a shader's output vanished after one frame. A simulation keeps its state in a texture and reads it back next frame: state(t+1) = f(state(t)). That's ping-pong feedback across frames (not Advanced's in-frame scratch). The values (chemical concentrations, velocities) need real precision, so this tier is WebGL2 + RGBA16F float render targets — exactly where the Advanced note said float would earn its keep. The makeSim helper hides the double-buffer plumbing; you write the step shader.
⚠️ Requires EXT_color_buffer_float. If your browser can't render to float textures the demos show a clear message instead of breaking — that's the graceful-degradation path, not a bug.

🔁 State in a Texture (the sim loop)

The whole tier in one idea: two float FBOs. Each frame, read the previous state texture, compute the next one, write it to the other FBO, swap, and draw a colourised view. Here the "state" is a brightness field that simply decays — drag to splat light into it and watch it fade. On 8-bit that decay would visibly stair-step; on float it's smooth. 

#version 300 es
precision highp float;
uniform sampler2D u_state;          // last frame's buffer
uniform vec2 u_resolution;
uniform vec2 u_mouse;               // 0..1, Y-flipped
uniform float u_mouseDown;
out vec4 outColor;

void main() {
    vec2 uv = gl_FragCoord.xy / u_resolution;
    // texelFetch = exact texel, no filtering — the right read for sims.
    float v = texelFetch(u_state, ivec2(gl_FragCoord.xy), 0).r * 0.985;
    if (u_mouseDown > 0.5 && distance(uv, u_mouse) < 0.03) v = 1.0;
    outColor = vec4(v, 0.0, 0.0, 1.0);   // → becomes next frame's u_state
}
📝 Click to see: the sim-loop scaffold 
// makeSim seeds both float buffers, then each frame: step (ping-pong)
// → colourised display. SIM_HEAD / SIM_LIB / DISP_HEAD are shared kit.
makeSim(canvas, {
  seed:    SIM_HEAD + `void main() { outColor = vec4(0.0); }`,
  step:    SIM_HEAD + SIM_LIB + `void main() {
    vec2 uv = gl_FragCoord.xy / u_resolution;
    float v = cell(ivec2(0)).r * 0.985;          // cell() = texelFetch helper
    if (u_mouseDown > 0.5 && distance(uv, u_mouse) < 0.03) v = 1.0;
    outColor = vec4(v, 0.0, 0.0, 1.0);
  }`,
  display: DISP_HEAD + `void main() {
    float v = texture(u_state, gl_FragCoord.xy / u_resolution).r;
    outColor = vec4(mix(vec3(0.03,0.04,0.08), vec3(1.0,0.6,0.15), v), 1.0);
  }`
}, { info: info });
// Identical — the helper + GLSL are dialect-agnostic JS strings.

  display: DISP_HEAD + `…` }, { info: info });

🦠 Conway's Game of Life

Cellular automata: every cell looks at its 8 neighbours and lives or dies by the B3/S23 rule. One texelFetch per neighbour, a count, a branch — the entire universe in a step shader. Reseed for fresh soup; drag to draw cells.

📝 Click to see: the Life rule 
step: SIM_HEAD + SIM_LIB + `void main() {
  float a = cell(ivec2(0)).r;
  float n = cell(ivec2(-1,-1)).r + cell(ivec2(0,-1)).r + cell(ivec2(1,-1)).r
          + cell(ivec2(-1,0)).r                        + cell(ivec2(1,0)).r
          + cell(ivec2(-1,1)).r + cell(ivec2(0,1)).r  + cell(ivec2(1,1)).r;
  float alive = (n == 3.0 || (a > 0.5 && n == 2.0)) ? 1.0 : 0.0;
  outColor = vec4(alive, 0.0, 0.0, 1.0);
}`
// cell(offset) clamps + texelFetches a neighbour (shared SIM_LIB helper).
step: SIM_HEAD + SIM_LIB + `…8-neighbour count → B3/S23…`

🧫 Reaction-Diffusion (Gray-Scott)

Two chemicals A and B diffuse and react: A + 2B → 3B. A Laplacian (9-tap neighbour sum) diffuses each; feed adds A, kill removes B. Tiny parameter changes flip it between coral, mitotic blobs, and writhing worms. This is the demo float precision is for — at 8-bit the slow accumulation just bands into mush.

💡 Sub-steps: RD converges slowly, so makeSim runs the step shader 12× per frame (substeps: 12) — 12 ping-pong swaps every rAF.
📝 Click to see: Gray-Scott step 
// feed/kill are baked into the shader per preset, then rebuilt:
step: SIM_HEAD + SIM_LIB + `const float Da=1.0, Db=0.5, dt=1.0;
const float feed=${p.f}, kill=${p.k};
void main() {
  vec2 s = cell(ivec2(0)).rg;                 // A=r, B=g
  vec2 lap = /* 9-tap Laplacian of A,B */ ;
  float r = s.r * s.g * s.g;
  float nA = s.r + (Da*lap.r - r + feed*(1.0 - s.r)) * dt;
  float nB = s.g + (Db*lap.g + r - (feed+kill)*s.g) * dt;
  outColor = vec4(clamp(nA,0.0,1.0), clamp(nB,0.0,1.0), 0.0, 1.0);
}`
// Presets {coral, mitosis, worms} just swap feed/kill, then sim.rebuild().
function spec(p) { return { substeps: 12, seed, step: /*…${p.f}/${p.k}…*/, display }; }
Two chemicals, a Laplacian, feed/kill. Why float matters.

✨ GPU Particles

65,536 particles, one per texel of a 256×256 float texture: .xy = position, .zw = velocity. The step shader integrates motion for every particle in parallel. The clever bit is the display: a vertex shader that is the particle — gl_VertexID → texel → position (a WebGL2/GLSL ES 3.00 trick). Drag to attract the swarm.

📝 Click to see: the particle vertex fetch 
// POINT_VERT: no vertex buffer — the texture IS the geometry.
const POINT_VERT = `#version 300 es
uniform sampler2D u_state;
out float v_spd;
void main() {
  int W = textureSize(u_state, 0).x;
  ivec2 t = ivec2(gl_VertexID % W, gl_VertexID / W);
  vec4 s = texelFetch(u_state, t, 0);
  v_spd = length(s.zw);
  gl_Position = vec4(s.xy * 2.0 - 1.0, 0.0, 1.0);
  gl_PointSize = 2.0;
}`;
// makeSim(..., { points: { vert: POINT_VERT, frag: POINT_FRAG, count: 256*256 } })
// Drawn with gl.POINTS, count = particle texels; additive-blended glow.
65k particles; the texture IS the particle buffer. Drag to attract.

🌊 2D Fluid / Ink

A compact Stam-style fluid: advect the velocity & dye fields (sample backward along the flow), add force where you drag, fade. Velocity in .xy, dye in .zw of the float state. Stir it with the mouse.

💡 Honest scope: this is a lite, non-projected fluid (no pressure solve) sampled with NEAREST — believable ink, not physically incompressible Navier-Stokes. The technique (advect a float field in a ping-pong buffer) is exactly how the real thing is built; the capstone adds controls.
📝 Click to see: the advection step 
step: SIM_HEAD + SIM_LIB + `void main() {
  vec2 uv  = gl_FragCoord.xy / u_resolution;
  vec2 vel = cell(ivec2(0)).xy;
  vec4 a   = texture(u_state, uv - vel * 0.012);   // trace back along flow
  vel = a.xy * 0.998;
  vec2 dye = a.zw * 0.99;
  float md = distance(uv, u_mouse);
  if (u_mouseDown > 0.5 && md < 0.07) {
    vel += normalize(uv - u_mouse + 1e-4) * (0.07 - md) * 2.2;
    dye += vec2(0.6, 1.0) * (0.07 - md) * 6.0;
  }
  outColor = vec4(clamp(vel,-3.0,3.0), clamp(dye,0.0,4.0));
}`
// substeps: 2 — two advection iterations per frame for stability.
Advect velocity + dye, mouse adds force. Drag to stir.

🎮 Mini-Project: Interactive Fluid Playground

The capstone: the fluid sim wrapped in controls — dye-colour presets, a viscosity/damping knob (u_param), and Clear. The step shader is rebuilt per dye preset; everything else is the same persistent-buffer machinery.

📝 Click to see: the playground wiring 
const DYES = { aqua:'vec2(0.7,1.0)', fire:'vec2(1.0,0.25)', toxic:'vec2(0.3,0.9)' };
const sim = makeSim(canvas, { substeps: 2, seed, step: stepFor(DYES.aqua),
                              display: disp }, { info: info, param: 0.5 });
// preset button → sim.rebuild({ …, step: stepFor(DYES[name]) });
