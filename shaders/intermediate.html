🔲 Tiling & Repetition

One shader, run per pixel, can fill the screen with a repeating pattern for free — no loop, no instancing. The trick is fract(): multiply uv by n and take fract, and space wraps into n identical cells, each with its own fresh 0..1 coordinate.

The two halves of a tiled coordinate

vec2 g = uv * n;
vec2 cell = floor(g);   // which tile (integer)
vec2 f = fract(g);    // position inside it (0..1)

cell lets you vary tiles (a checkerboard via mod(cell.x + cell.y, 2.0)); f draws the same motif in every tile. This is the backbone of brick walls, grids, sprite sheets, and noise.

💡 GLSL ES 1.00: the tile count is baked into the shader as a constant, so the buttons rebuild the shader (toy.setFrag(...)) rather than push a uniform — the same pattern you'll need for fbm octaves below.
📝 Click to see: the tiling shader 
// Cell count is a shader constant → a button rebuilds the shader.
function buildFrag(n) {
    return `precision mediump float;
uniform vec2 u_resolution;
uniform float u_time;
void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  uv.x *= u_resolution.x / u_resolution.y;
  vec2 g = uv * ${n}.0;
  vec2 cell = floor(g);
  vec2 f = fract(g);
  float checker = mod(cell.x + cell.y, 2.0);
  float d = distance(f, vec2(0.5));
  float dot = 1.0 - smoothstep(0.30, 0.34 + 0.05 * sin(u_time * 2.0), d);
  vec3 bg = mix(vec3(0.07,0.10,0.18), vec3(0.12,0.16,0.28), checker);
  gl_FragColor = vec4(mix(bg, vec3(0.31,0.76,0.97), dot), 1.0);
}`;
}

🎲 Randomness: the hash

GLSL has no rand(). Shaders must be deterministic — the same pixel must produce the same value every frame. So we fake randomness with a hash: a function that scrambles an input into a chaotic-looking but repeatable number. The classic one-liner: 

// The "canonical" GLSL hash. Not real randomness — a deterministic
// scramble. sin() goes wild, the big multiply amplifies it, fract()
// keeps the chaotic low bits in 0..1.
float hash(vec2 p) {
    return fract(sin(dot(p, vec2(127.1, 311.7))) * 43758.5453);
}

Why this works (intuitively)

  • dot(p, k) — collapse the 2D coord to one number, mixing x and y.
  • sin(...) — a wildly non-linear curve; tiny input changes swing the output.
  • * 43758.5453 — amplify so the unpredictable fractional bits dominate.
  • fract(...) — keep just those bits, in [0, 1).

Feed it a per-cell floor(uv * n) and every cell gets its own value → TV static.

💡 mediump caveat: we keep precision mediump float; for consistency and broad device support. The big multiply can band slightly on low-precision GPUs — fine for a tutorial; production code sometimes uses highp or a bit-hash instead.
📝 Click to see: per-cell random 
const STATIC = `precision mediump float;
uniform vec2 u_resolution;
uniform float u_time;

  uv.x *= u_resolution.x / u_resolution.y;

🌫️ Value Noise

Raw per-cell hash is too harsh for natural effects. Value noise tames it: hash the four corners of each cell, then mix between them by the fractional position — with a smooth fade so cell boundaries vanish. The result is the smooth, organic randomness clouds and terrain are made of.

The fade curve matters

A plain linear mix leaves visible diamond creases at cell edges. The fix is the Beginner-tier smoothstep idea applied to the blend factor: f * f * (3.0 - 2.0 * f) (zero slope at 0 and 1) so neighbouring cells join seamlessly. Same maths as easing curves.

📝 Click to see: bilinear value noise 
// inside the fragment shader:
float valueNoise(vec2 p) {
  vec2 i = floor(p);
  vec2 f = fract(p);
  float a = hash(i);
  float b = hash(i + vec2(1.0, 0.0));
  float c = hash(i + vec2(0.0, 1.0));
  float d = hash(i + vec2(1.0, 1.0));
  vec2 u = f * f * (3.0 - 2.0 * f);          // smooth fade
  return mix(mix(a, b, u.x), mix(c, d, u.x), u.y);
}
// inside the fragment shader (GLSL is identical across JS/TS):
float valueNoise(vec2 p) {
  vec2 i = floor(p);
  vec2 f = fract(p);
  float a = hash(i);
  float b = hash(i + vec2(1.0, 0.0));
  float c = hash(i + vec2(0.0, 1.0));
  float d = hash(i + vec2(1.0, 1.0));
  vec2 u = f * f * (3.0 - 2.0 * f);
  return mix(mix(a, b, u.x), mix(c, d, u.x), u.y);
}

🪵 fbm (Fractal Brownian Motion)

One layer of value noise is soft but featureless. fbm stacks several octaves: each adds noise at double the frequency and half the amplitude. Big shapes from the low octaves, fine grain from the high ones — the texture of clouds, marble, wood, terrain, smoke.

The octave loop

for each octave: sum += amp * noise(p * freq);
  freq *= 2.0;   amp *= 0.5;

GLSL ES 1.00 constraint: a for loop's bound must be a constant — you can't drive octave count with a uniform. So changing octaves means rebuilding the shader with the number baked in (the buttons do exactly this via toy.setFrag). Continuous knobs still use u_param; structural ones rebuild.

📝 Click to see: fbm with a constant octave count 
function buildFrag(octaves) {
    return `precision mediump float;
uniform vec2 u_resolution;
uniform float u_time;
// ... hash() and valueNoise() here ...
float fbm(vec2 p) {
  float sum = 0.0, amp = 0.5, freq = 1.0;
  for (int i = 0; i < ${octaves}; i++) {   // bound MUST be constant
    sum += amp * valueNoise(p * freq);
    freq *= 2.0;
    amp *= 0.5;
  }
  return sum;
}
void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  uv.x *= u_resolution.x / u_resolution.y;
  float v = fbm(uv * 4.0 + vec2(u_time * 0.15, 0.0));
  gl_FragColor = vec4(vec3(v), 1.0);
}`;
}

🌀 Domain Warping

Here's the trick that makes procedural art look alive: instead of evaluating noise at p, evaluate it at p + noise(p) — feed noise into its own input. The pattern drags itself around, producing the swirling, flowing, marbled looks behind fire, nebulae, oil, and magic effects.

One line, big payoff

vec2 q = vec2(fbm(p), fbm(p + offset));
float v = fbm(p + strength * q);

u_param is the warp strength (continuous → a uniform, no rebuild needed). u_mouse nudges the warp source, so dragging the cursor stirs the field. Off (strength 0) it's plain fbm; turn it up and it churns.

📝 Click to see: the domain warp 
// ... hash(), valueNoise(), fbm() (5 octaves) in the shader ...
void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  uv.x *= u_resolution.x / u_resolution.y;
  vec2 p = uv * 3.0;
  vec2 mo = (u_mouse / u_resolution - 0.5);
  vec2 q = vec2(fbm(p + u_time * 0.10),
                fbm(p + vec2(5.2, 1.3) - u_time * 0.10));
  float v = fbm(p + u_param * (q + mo));   // <- the domain warp
  vec3 col = mix(vec3(0.04,0.10,0.22), vec3(0.45,0.85,0.95), v);
  gl_FragColor = vec4(col, 1.0);
}

♨️ UV Distortion

Warping noise is pretty; warping the UV you sample a scene with is a real game effect. Offset the lookup coordinate by a small field and the whole image wobbles — that's heat-haze over lava, refraction through water, a shockwave, a portal. Here a procedural "scene" (gradient + stripes, so the distortion is obvious) is displaced two ways.

Same idea, two offset fields

  • Heat haze: a vertical sin(uv.y * f + u_time) shimmer on x only.
  • Water refraction: a 2D value-noise field offsetting both axes.

u_param scales the strength — subtle for in-game realism, cranked to make the mechanism visible.

📝 Click to see: scene + UV offset 
// A "scene" we can see the distortion against:
vec3 scene(vec2 uv) {
  float stripes = step(0.5, fract(uv.x * 10.0));
  vec3 grad = mix(vec3(0.10,0.20,0.45), vec3(0.95,0.65,0.30), uv.y);
  return mix(grad, grad * 0.45, stripes);
}
// Heat haze: shimmer the sampling UV, then read the scene there.
void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec2 off = vec2(sin(uv.y * 40.0 + u_time * 6.0), 0.0) * 0.02;
  gl_FragColor = vec4(scene(uv + off * u_param), 1.0);
}

🟣 Gradient (Perlin) Noise

Value noise interpolates random values at cell corners — cheap, but it has visible axis-aligned blockiness. Gradient (Perlin) noise instead interpolates the dot product of a random gradient with the offset to each corner. The result is smoother and more organic — the noise most marble, cloud, and terrain shaders use.

Value vs gradient noise

  • Value: lerp random heights → can look grid-aligned.
  • Gradient: lerp dot(grad, f - corner) → zero at lattice points, slopes between → fewer artifacts.
  • Stack it in fbm() exactly like before; warp it for marble veins.
📝 Click to see: single octave / fbm / marble / warp 
// Gradient noise: a random gradient per lattice point, dotted with the
// offset to the pixel, then smooth-interpolated. Lib stays fixed; the
// final value expression is injected per button.
const LIB = `precision mediump float;
uniform vec2 u_resolution;
uniform float u_time;
vec2 hash2(vec2 p) {
  p = vec2(dot(p, vec2(127.1, 311.7)), dot(p, vec2(269.5, 183.3)));
  return -1.0 + 2.0 * fract(sin(p) * 43758.5453);
}
float perlin(vec2 p) {
  vec2 i = floor(p), f = fract(p);
  vec2 u = f * f * (3.0 - 2.0 * f);
  float a = dot(hash2(i),                f);
  float b = dot(hash2(i + vec2(1,0)), f - vec2(1.0, 0.0));
  float c = dot(hash2(i + vec2(0,1)), f - vec2(0.0, 1.0));
  float d = dot(hash2(i + vec2(1,1)), f - vec2(1.0, 1.0));
  return mix(mix(a, b, u.x), mix(c, d, u.x), u.y);
}
float fbm(vec2 p) {
  float s = 0.0, a = 0.5;
  for (int i = 0; i < 5; i++) { s += a * perlin(p); p *= 2.0; a *= 0.5; }
  return s;
}`;

function buildFrag(expr) {
    return LIB + `
void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  uv.x *= u_resolution.x / u_resolution.y;
  float v = ${expr};
  gl_FragColor = vec4(mix(vec3(0.04,0.06,0.12), vec3(0.31,0.76,0.97), v), 1.0);
}`;
}

🔷 Voronoi / Cellular Noise

Scatter one random point per grid cell, then for each pixel find the nearest point (F1) and the second-nearest (F2). That single idea gives cracked ground, reptile scales, stained glass, water caustics, and cell-shaded blobs — depending only on what you display.

What F1 / F2 give you

  • F1 — distance to nearest point → classic Voronoi blobs.
  • cell id → flat colored cells (scales, mosaic, stained glass).
  • F2 - F1 — small near borders → free cell edges (cracked earth).
📝 Click to see: cells / distance / edges / cracked 
// vor() fills F1, F2 and the nearest cell id via an out-parameter. The
// 3x3 neighbour loop has a CONSTANT bound (GLSL ES 1.00 requirement).
const LIB = `precision mediump float;
uniform vec2 u_resolution;
uniform float u_time;
vec2 hash2(vec2 p) {
  p = vec2(dot(p, vec2(127.1, 311.7)), dot(p, vec2(269.5, 183.3)));
  return fract(sin(p) * 43758.5453);
}
void vor(vec2 p, out float f1, out float f2, out vec2 cid) {
  vec2 g = floor(p), f = fract(p);
  f1 = 8.0; f2 = 8.0; cid = g;
  for (int j = -1; j <= 1; j++)
  for (int i = -1; i <= 1; i++) {
    vec2 o = vec2(float(i), float(j));
    vec2 pt = o + 0.5 + 0.45 * sin(u_time + 6.2831 * hash2(g + o));
    float d = length(pt - f);
    if (d < f1) { f2 = f1; f1 = d; cid = g + o; }
    else if (d < f2) { f2 = d; }
  }
}`;

function buildFrag(out) {
    return LIB + `
void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  uv.x *= u_resolution.x / u_resolution.y;
  float f1, f2; vec2 cid;
  vor(uv * 6.0, f1, f2, cid);
  vec3 col = ${out};
  gl_FragColor = vec4(col, 1.0);
}`;
}

🌊 Flow & Scrolling

Animating a texture by moving the UVs over time is how rivers, lava, conveyor belts, waterfalls, and force fields are done. Scroll uv for a straight conveyor; bend the scroll direction with a vector field for flowing water.

  • Scroll: uv -= dir * u_time — the pattern slides.
  • Flow field: a per-pixel direction; advect noise along it for curling water/lava.
  • Layer two scroll speeds → parallax depth (classic for lava & waterfalls).
📝 Click to see: conveyor / river / lava / flow field 
const LIB = `precision mediump float;
uniform vec2 u_resolution;
uniform float u_time;
float hash(vec2 p) { return fract(sin(dot(p, vec2(127.1, 311.7))) * 43758.5453); }
float vnoise(vec2 p) {
  vec2 i = floor(p), f = fract(p);
  f = f * f * (3.0 - 2.0 * f);
  return mix(mix(hash(i), hash(i + vec2(1,0)), f.x),
             mix(hash(i + vec2(0,1)), hash(i + vec2(1,1)), f.x), f.y);
}
float fbm(vec2 p) {
  float s = 0.0, a = 0.5;
  for (int i = 0; i < 5; i++) { s += a * vnoise(p); p *= 2.0; a *= 0.5; }
  return s;
}`;

    return LIB + `
void main() {

  uv.x *= u_resolution.x / u_resolution.y;
  vec3 col;
  ${body}
  gl_FragColor = vec4(col, 1.0);
}`;
}

🌈 Cosine Palettes & Colour Spaces

Hand-picking colours with mix() gets old fast. Inigo Quilez's cosine palette is one tiny function that produces an entire smooth gradient from four vec3 knobs — bias, amplitude, frequency, phase:

The whole palette in one line

vec3 pal(float t, vec3 a, vec3 b, vec3 c, vec3 d) {
  return a + b * cos(6.2831 * (c * t + d));
}

a is the midpoint colour, b the contrast, c how many colour cycles fit in 0..1, d the per-channel phase (offsetting R/G/B is what makes it a rainbow instead of greyscale). It's used everywhere — heatmaps, fire, UI accents, this whole track's blue ramp.

💡 Colour spaces: RGB is for the screen, not for picking colours. A straight mix() of green→magenta in RGB muddies through grey; sweeping the hue angle (HSV) or mixing in a perceptual space (OKLab) stays vivid. The HSV preset below walks the hue ring — same "colour from one number" idea, different space. The palette function is structural, so the buttons rebuild via setFrag (same rule as tiling/fbm).
📝 Click to see: the cosine-palette shader 
// Only `colorize()` changes per preset → inject it & rebuild.
const COLORFN = {
    rainbow: 'vec3 colorize(float t){ return pal(t, vec3(0.5), vec3(0.5), vec3(1.0), vec3(0.0,0.33,0.67)); }',
    hsv:     'vec3 colorize(float t){ return hsv2rgb(vec3(t, 0.8, 1.0)); }'
    // ... sunset, neon ...
};
function buildFrag(key) {
    return `precision mediump float;
uniform vec2 u_resolution; uniform float u_time;
vec3 pal(float t, vec3 a, vec3 b, vec3 c, vec3 d){ return a + b*cos(6.28318*(c*t+d)); }
vec3 hsv2rgb(vec3 c){ /* hue ring */ }
${COLORFN[key]}
void main(){
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec3 col = uv.y < 0.20
    ? colorize(uv.x)                                   // raw 1-D ramp
    : colorize(fract(length(uv-0.5)*1.7 - u_time*0.12)); // animated field
  gl_FragColor = vec4(col, 1.0);
}`;
}

🧩 Truchet Tiles & Hex Grids

Tiling so far used a square grid with the same motif in every cell. Two upgrades unlock a huge amount of free pattern: Truchet tiles (let a per-cell hash pick the tile's orientation) and hex grids (pack cells like a honeycomb instead of a checkerboard).

Truchet: one random bit per cell

vec2 id = floor(uv * n);
if (hash(id) < 0.5) f.x = 1.0 - f.x;   // flip this tile
// draw two quarter-circle arcs → they connect across cells

Each cell holds the same two arcs; flipping half of them at random makes the arcs join up into an endless, organic maze — zero authoring, infinite variety. It's how games fake circuit boards, pipes, mazes, and woven backgrounds.

💡 Hex without trig: a hex grid is just "the nearer of two square lattices offset by half a cell." That hexFold is branchless (one ternary on squared distance) — no loop, GLSL ES 1.00-safe. Mode & density are structural → setFrag.
📝 Click to see: the Truchet / hex shader 
// Truchet arcs: flip the cell on a hashed bit, then draw two
// corner arcs — they line up across the flip.
const arc = (n) => `vec3 tile(vec2 uv){
  vec2 g = uv * ${n}.0; vec2 id = floor(g); vec2 f = fract(g);
  if (hash(id) < 0.5) f.x = 1.0 - f.x;
  float d = min(abs(length(f)-0.5), abs(length(f-vec2(1.0))-0.5));
  return mix(vec3(0.06,0.08,0.16), vec3(0.31,0.76,0.97),
             smoothstep(0.07, 0.035, d));
}`;

♾️ Seamless & Looping Patterns

A background texture has two ways to betray itself: a visible seam where copies meet, and a visible jump when the animation restarts. Both have one-line fixes, and this demo shows the broken vs fixed side by side (a 2×2 grid of the same tile, with red guides on the seams).

Tile in space, loop in time

// seamless: wrap the noise lattice by the period
p = mod(p, vec2(period));   // corner `period` == corner 0

// perfect loop: drive motion ONLY by a periodic function
mix(a, b, 0.5 + 0.5 * sin(6.2831 * t / T)); // frame 0 == frame T

If every time-dependent term is a sin/cos of 2πt/T, then t = 0 and t = T are bit-identical — a flawless GIF-able loop, no fade-cut needed.

💡 Why the seam happens: value noise interpolates between lattice corners. At the tile edge it reaches corner period, but the next tile starts at corner 0 — different hashes unless you mod the lattice. Period & loop length are constants → setFrag.
📝 Click to see: the seamless / looping shader

        return fract(sin(dot(p, vec2(127.1,311.7)))*43758.5453); }`

🔥 Mini-Project: Animated Fire / Smoke

Everything composes into one of the most-shipped 2D shader effects. Recipe:

  • fbm for the turbulent texture
  • a rising flow — scroll the noise domain downward over u_time
  • a vertical falloff — (1.0 - uv.y) so it's dense at the base, wispy on top
  • a palette — mix dark-red → orange → yellow for fire; a grey ramp for smoke
💡 Same building blocks: turbulence just swaps the fbm octave count (structural → rebuild via state, exactly like Beginner's water buildFrag); intensity is continuous → u_param. No new concepts — just composition.
📝 Click to see: the fire shader (state-built) 
function buildFrag(s) {
    const octaves = s.turbulent ? '6' : '3';
    const palette = s.smoke
        ? 'vec3 col = mix(vec3(0.02,0.02,0.03), vec3(0.55,0.55,0.58), v);'
        : 'vec3 col = mix(vec3(0.40,0.02,0.0), vec3(1.0,0.85,0.25), v);\n' +
          '  col = mix(col, vec3(1.0,0.35,0.05), smoothstep(0.45,0.05,v));';
    return `precision mediump float;
uniform vec2 u_resolution;
uniform float u_time;
uniform float u_param;
// ... hash(), valueNoise(), fbm() with ${octaves} octaves ...
void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec2 p = vec2(uv.x * 3.0, uv.y * 4.0 - u_time * 2.0);  // rising flow
  float n = fbm(p);
  float flame = n * (1.0 - uv.y) * 2.2 * u_param;        // base-heavy
  float v = smoothstep(0.0, 0.7, flame);
  ${palette}
  gl_FragColor = vec4(col, 1.0);
}`;
}
