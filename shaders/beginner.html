🧠 What Is a Shader?

A shader is a tiny program that runs on your GPU instead of your CPU. The kind we'll write all tier — a fragment shader — has one job: given a pixel, decide its color. The twist is that it runs for every pixel at the same time, in parallel, thousands at once.

🕹️ What shaders do in games you've played

  • Water & lava — animated surfaces in Zelda, Terraria, Hollow Knight
  • Hit flashes — the white blink when an enemy takes damage
  • Screen transitions — dissolves, pixelate-outs, circular wipes
  • CRT / scanline filters — the retro look in Shovel Knight, Celeste
  • Lighting & fog — 2D dynamic lights and darkness

CPU loop vs GPU parallel

On the CPU you draw a scene with a loop: "draw shape 1, then shape 2, …". A shader flips that around — there's no loop over objects; instead the same little color program is evaluated for every pixel of the screen, every frame. Toggle the two mental models:

Click a button to switch the mental model
💡 Key insight: We render shaders onto a full-screen quad — two triangles covering the whole canvas. The vertex shader just stretches that quad edge to edge; all the art is the fragment shader coloring each pixel. That's the entire mental model for 2D shader effects.

🟥 Your First Fragment Shader

Here is a complete, valid GLSL fragment shader. It colors every pixel the same solid red: 

precision mediump float;     // required: float precision for this shader

void main() {
    // gl_FragColor is the OUTPUT: this pixel's color.
    // Channels are 0.0–1.0, NOT 0–255. (r, g, b, a)
    gl_FragColor = vec4(1.0, 0.0, 0.0, 1.0);
}

Read it line by line

  • precision mediump float; — GLSL ES requires you to state float precision. mediump is the safe default.
  • void main() — the entry point, run once per pixel.
  • gl_FragColor — a built-in output variable. Whatever you assign becomes the pixel's color.
  • vec4(r, g, b, a) — a 4-component vector. Colors are normalized: 1.0 = full, 0.0 = none.

You never set up WebGL by hand in this tier — a small helper called makeShaderToy builds the quad, compiles your shader, wires the standard uniforms, and runs the render loop. You just supply main(). (The Export button ships the helper inline so the file runs anywhere.)

🗺️ UV Coordinates: Where Am I?

A solid color is boring because the shader doesn't know which pixel it is. The built-in gl_FragCoord.xy gives the pixel's screen position in pixels. Divide it by the canvas size (passed in via the u_resolution uniform) and you get uv — a normalized 0→1 coordinate. That's the foundation of everything.

The one line you'll write forever

vec2 uv = gl_FragCoord.xy / u_resolution;

uv.x goes 0 (left) → 1 (right). uv.y goes 0 (bottom) → 1 (top) — WebGL's origin is bottom-left, not top-left like the 2D canvas. Mapping uv straight to color (vec4(uv, 0.0, 1.0)) gives the classic red-green gradient: red grows rightward, green grows upward.

💡 Aspect ratio: uv is 0–1 on both axes even when the canvas is wider than it is tall — so circles come out as ellipses. The fix: multiply uv.x by u_resolution.x / u_resolution.y. The "Aspect-corrected" button shows a truly round circle.
📝 Click to see: the uv visualizer 
const RAW = `precision mediump float;
uniform vec2 u_resolution;
void main() {

}`;

📐 Shaping Functions

Shapes in shaders aren't drawn — they're computed. You turn uv into a number per pixel (a "mask": 1 inside, 0 outside) using a handful of built-in functions:

  • length(v) / distance(a, b) — a distance field: how far this pixel is from a point.
  • step(edge, x) — a hard 0/1 switch at edge.
  • smoothstep(a, b, x) — like step but with a soft, anti-aliased ramp between a and b.
  • fract(x) — the fractional part; instantly tiles/repeats space.
  • clamp(x, lo, hi) — keep a value in range.

A soft circle is just 1.0 - smoothstep(r, r + softness, distance(uv, center)). Use the Softer/Sharper buttons — they feed a number into the u_param uniform that widens or narrows the smoothstep band live, with no recompile.

📝 Click to see: distance-field circle & stripes 
const CIRCLE = `precision mediump float;
uniform vec2 u_resolution;
uniform float u_param;                 // edge softness, set from JS
void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  uv.x *= u_resolution.x / u_resolution.y;
  float c = u_resolution.x / u_resolution.y * 0.5;
  float d = distance(uv, vec2(c, 0.5));
  float m = 1.0 - smoothstep(0.28, 0.28 + u_param, d);
  gl_FragColor = vec4(vec3(m) * vec3(0.31, 0.76, 0.97), 1.0);
}`;

🌈 Color & Gradients

Colors are just vec3s, and the single most useful color function is mix(a, b, t) — a per-pixel linear interpolation between two colors as t goes 0→1. (If you did the Fundamentals Lerp topic: mix is lerp, running on the GPU.)

Three gradients, one idea

  • Linear: t = uv.x → left-to-right blend.
  • Radial: t = distance(uv, center) → vignette / glow.
  • Banded: t = floor(t * 6.0) / 6.0 → posterized, retro palette look.

Color math is taught right here in this track — there's no separate Fundamentals page for it yet.

📝 Click to see: linear / radial / banded gradients 
const LINEAR = `precision mediump float;
uniform vec2 u_resolution;
vec3 A = vec3(0.05, 0.16, 0.40);   // deep blue
vec3 B = vec3(1.00, 0.65, 0.30);   // warm amber
void main() {
  float t = gl_FragCoord.x / u_resolution.x;

}`;

⏱️ Animation with u_time

Nothing here is "animated" by storing state — the shader is recomputed from scratch every frame. To move things you feed in time as a uniform (u_time, in seconds) and run it through sin/cos. Because it's real seconds, the motion is frame-rate independent — same speed at 30 or 144 fps.

💡 The animation trick: 0.5 + 0.5 * sin(u_time) is the workhorse — it oscillates smoothly between 0 and 1. Drop it into a radius, a color mix factor, or a position and it pulses, fades, or bobs.
📝 Click to see: the pulsing / wave shader 
const FRAG = `precision mediump float;
uniform vec2 u_resolution;
uniform float u_time;
void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  uv.x *= u_resolution.x / u_resolution.y;
  float c = u_resolution.x / u_resolution.y * 0.5;
  float r = 0.22 + 0.06 * sin(u_time * 3.0);   // breathing radius
  float d = distance(uv, vec2(c, 0.5));
  float circle = 1.0 - smoothstep(r, r + 0.02, d);
  float wave = 0.5 + 0.5 * sin(uv.x * 12.0 - u_time * 2.0);
  vec3 bg = mix(vec3(0.05,0.07,0.15), vec3(0.10,0.20,0.35), wave);
  gl_FragColor = vec4(mix(bg, vec3(1.0,0.65,0.30), circle), 1.0);
}`;

const toy = makeShaderToy(canvas, FRAG, { info: info });

🖱️ Interactivity with u_mouse

The last standard uniform is u_mouse — the cursor position in pixels. Combine it with a distance field and you get a flashlight, a ripple origin, a reveal mask… the basis of lots of 2D game UI and effects.

⚠️ The #1 beginner bug: mouse Y is flipped

The 2D canvas measures Y from the top; gl_FragCoord measures it from the bottom. If your spotlight chases the mouse upside-down, that's why. This track's helper fixes it once — it passes u_mouse = (mouseX, canvasHeight - mouseY) so u_mouse lives in the same space as gl_FragCoord. You never have to think about it again.

📝 Click to see: cursor spotlight & ripple 
const SPOT = `precision mediump float;
uniform vec2 u_resolution;
uniform vec2 u_mouse;
void main() {
  float d = distance(gl_FragCoord.xy, u_mouse) / u_resolution.y;
  float light = 1.0 - smoothstep(0.0, 0.45, d);
  vec3 col = mix(vec3(0.04,0.05,0.10), vec3(0.31,0.76,0.97), light);
  gl_FragColor = vec4(col, 1.0);
}`;

🌊 Mini-Project: Animated Water Tint

Everything composes. This capstone is a full-screen water effect — the kind of layer a 2D game drops behind a tilemap (an ocean background) or over the scene (underwater tint). It uses only what this tier taught:

  • uv from gl_FragCoord / u_resolution
  • sin waves driven by u_time
  • a distance-based ripple around u_mouse
  • mix for a depth tint, smoothstep for fake caustics
  • an optional sin(gl_FragCoord.y) scanline — a real CRT post-FX trick
📝 Click to see: the full water shader (state-built) 
// buildFrag() assembles the shader from a small state object so the
// buttons can swap amplitude / tint / scanline without WebGL plumbing.
function buildFrag(s) {
    const shallow = s.murky ? 'vec3(0.18, 0.38, 0.30)' : 'vec3(0.35, 0.78, 0.92)';
    const deep    = s.murky ? 'vec3(0.02, 0.10, 0.09)' : 'vec3(0.03, 0.18, 0.38)';
    const scan = s.scanline
        ? '  col *= 0.86 + 0.14 * sin(gl_FragCoord.y * 3.14159);\n'
        : '';
    return `precision mediump float;
uniform vec2 u_resolution;
uniform vec2 u_mouse;
uniform float u_time;
void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec2 m = u_mouse / u_resolution;
  float md = distance(uv, m);
  float ripple = sin(md * 38.0 - u_time * 4.0) * exp(-md * 6.0) * 0.025;
  float wave = sin(uv.x * 11.0 + u_time * 1.4) * ${s.amp}
             + sin(uv.y * 17.0 - u_time * 2.0) * ${s.amp} * 0.6
             + ripple;
  float depth = clamp(uv.y + wave, 0.0, 1.0);
  vec3 col = mix(${deep}, ${shallow}, depth);
  float caustic = smoothstep(0.55, 1.0,
      sin(uv.x * 26.0 + u_time * 2.0 + wave * 22.0)
    * sin(uv.y * 20.0 - u_time * 1.6));
  col += caustic * 0.10;
${scan}  gl_FragColor = vec4(col, 1.0);
}`;
}

const toy = makeShaderToy(canvas, buildFrag(state), { info: info });

    const deep    = s.murky ? 'vec3(0.02, 0.10, 0.09)' : 'vec3(0.03, 0.18, 0.38)';
    const scan = s.scanline
        ? '  col *= 0.86 + 0.14 * sin(gl_FragCoord.y * 3.14159);\n'
        : '';
    return `precision mediump float;
uniform vec2 u_resolution;
uniform vec2 u_mouse;
uniform float u_time;
void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec2 m = u_mouse / u_resolution;
  float md = distance(uv, m);
  float ripple = sin(md * 38.0 - u_time * 4.0) * exp(-md * 6.0) * 0.025;
  float wave = sin(uv.x * 11.0 + u_time * 1.4) * ${s.amp}
             + sin(uv.y * 17.0 - u_time * 2.0) * ${s.amp} * 0.6
             + ripple;
