🖼️ Sampling a Texture

A texture is an image the GPU can read per pixel. Declare a uniform sampler2D u_tex; and read it with texture2D(u_tex, uv) where uv is 0..1 across the image. That's the whole API — here's a complete minimal texture shader: 

precision mediump float;
uniform vec2 u_resolution;
uniform sampler2D u_tex;            // the sprite, on texture unit 0

void main() {
    vec2 uv = gl_FragCoord.xy / u_resolution;
    gl_FragColor = texture2D(u_tex, uv);   // rgba straight from the image
}

Two gotchas, both fixed once in the runner

  • Y is flipped. A 2D canvas is Y-down; gl_FragCoord is Y-up. The runner sets UNPACK_FLIP_Y_WEBGL so the sprite samples upright (mirrors the Beginner u_mouse Y-flip fix).
  • Aspect. A square 256² sprite in an 800×450 canvas would squash. toSpriteUV() maps a centred, aspect-correct square box; outside it a checkerboard backdrop() shows through the sprite's transparent margin. Try "Stretched" to see why the fit matters.
📝 Click to see: sampling the sprite 
// GLSL_HEAD declares the uniforms + toSpriteUV()/backdrop() helpers.
const FIT = GLSL_HEAD + `void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec4 tex = texture2D(u_tex, toSpriteUV(uv));
  gl_FragColor = vec4(mix(backdrop(uv), tex.rgb, tex.a), 1.0);
}`;

🎨 Tinting & Multiply

The simplest sprite shader, and one of the most used: multiply the sampled colour by a tint. Because it's a multiply, the sprite's own shading and shadows survive — a flat overlay would erase them. Keep tex.a so the silhouette stays crisp.

vec3 tinted = tex.rgb * tint; — that's the entire effect. It's the team-colour in an RTS, the red damage flash, the gold of a power-up, the grey of a disabled button.

📝 Click to see: tint multiply 
function buildFrag(tint) {
    return GLSL_HEAD + `void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec4 tex = texture2D(u_tex, toSpriteUV(uv));
  vec3 tinted = tex.rgb * ${tint};        // multiply keeps shading
  gl_FragColor = vec4(mix(backdrop(uv), tinted, tex.a), 1.0);
}`;
}
// Damage tint:
toy.setFrag(buildFrag('vec3(1.0, 0.35, 0.35)'));

🌈 Palette Swap

Ship one sprite, recolour it a dozen ways. Read the sample's luminance (perceived brightness) and remap it through a small colour ramp. The four luminance bands the gem is drawn in become four stops of any palette — fire, ice, poison, gold-rare.

float l = dot(tex.rgb, vec3(0.299, 0.587, 0.114));
vec3 c = l < 0.5 ? mix(A, B, l*2.0) : mix(B, C, (l-0.5)*2.0);

This is how NES-era games (and modern pixel-art ones) get a whole palette of enemy variants from a single sprite sheet.

📝 Click to see: luminance → ramp 
function buildFrag(a, b, c) {
    return GLSL_HEAD + `void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec4 tex = texture2D(u_tex, toSpriteUV(uv));
  float l = dot(tex.rgb, vec3(0.299, 0.587, 0.114));
  vec3 ramp = l < 0.5
    ? mix(${a}, ${b}, l * 2.0)
    : mix(${b}, ${c}, (l - 0.5) * 2.0);
  gl_FragColor = vec4(mix(backdrop(uv), ramp, tex.a), 1.0);
}`;
}

💨 Dissolve & Burn

Death, teleport-out, a disintegrating enemy. Sample the Intermediate-tier value noise per pixel and compare it to a progress value: where noise < progress, drop the pixel. A thin band right at the threshold glows — the burning edge.

💡 discard & thresholding: here we simply skip colouring dissolved pixels (leaving the backdrop). GLSL ES 1.00's discard does the same for true alpha sprites. progress is a continuous knob → the u_param uniform; the "Auto" button instead drives it from fract(u_time) for a looping effect.

📝 Click to see: noise-thresholded dissolve 
// GLSL_NOISE adds hash()/valueNoise() (from the Intermediate tier).
function buildFrag(progressExpr) {
    return GLSL_HEAD + GLSL_NOISE + `void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec2 s = toSpriteUV(uv);
  vec4 tex = texture2D(u_tex, s);
  float prog = ${progressExpr};
  float n = valueNoise(s * 14.0);
  vec3 col = backdrop(uv);
  if (tex.a > 0.5) {
    if (n >= prog) {
      col = tex.rgb;
      float edge = smoothstep(prog, prog + 0.08, n);
      col = mix(vec3(1.0, 0.6, 0.1), col, edge);   // burn edge
    }
  }
  gl_FragColor = vec4(col, 1.0);
}`;
}

⭕ Outline / Rim

The selection highlight in every strategy game. A pixel just outside the sprite should glow if one of its neighbours is inside. Sample the sprite's alpha at the eight neighbouring texels — one texel is 1.0 / u_texResolution — and if any is opaque while this pixel isn't, you're on the rim.

Why u_texResolution

To step exactly one texel you must know the texture's pixel size. The runner passes it as u_texResolution (256, 256). u_param scales the step → outline thickness, with zero recompiles.

📝 Click to see: 8-tap alpha dilation 
// 8 fixed taps (no loop) → GLSL ES 1.00-safe.
float silhouette(vec2 s) {
  vec2 px = u_param / u_texResolution;      // texel step * thickness
  float m = 0.0;
  m = max(m, texture2D(u_tex, s + vec2( px.x, 0.0)).a);
  m = max(m, texture2D(u_tex, s + vec2(-px.x, 0.0)).a);
  m = max(m, texture2D(u_tex, s + vec2(0.0,  px.y)).a);
  m = max(m, texture2D(u_tex, s + vec2(0.0, -px.y)).a);
  m = max(m, texture2D(u_tex, s + px).a);
  m = max(m, texture2D(u_tex, s - px).a);
  m = max(m, texture2D(u_tex, s + vec2( px.x, -px.y)).a);
  m = max(m, texture2D(u_tex, s + vec2(-px.x,  px.y)).a);
  return m;
}
// ring = neighbour-is-opaque but I'm not → draw the outline colour there.

⚡ Hit-Flash & Invincibility Blink

The two pieces of damage feedback every action game ships. Hit-flash: mix the sprite toward white on a fast pulse. Invincibility blink: toggle the sprite's alpha on and off during i-frames. Both are one line driven by u_time.

💡 Triggering it for real: in a game you'd start the pulse on the damage event and let it decay. Here, clicking the canvas re-arms the flash — the same idea, JS-side. (A one-shot decay needs a "hit time" uniform; that's an Advanced-tier topic.)
📝 Click to see: flash & blink 
const FLASH = GLSL_HEAD + `void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec4 tex = texture2D(u_tex, toSpriteUV(uv));
  float f = 0.5 + 0.5 * sin(u_time * 18.0);
  vec3 hit = mix(tex.rgb, vec3(1.0), f);          // white-out pulse
  gl_FragColor = vec4(mix(backdrop(uv), hit, tex.a), 1.0);
}`;
const BLINK = GLSL_HEAD + `void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec4 tex = texture2D(u_tex, toSpriteUV(uv));
  float on = step(0.5, fract(u_time * 6.0));      // i-frame blink
  gl_FragColor = vec4(mix(backdrop(uv), tex.rgb, tex.a * on), 1.0);
}`;
canvas.addEventListener('click', () => toy.setFrag(FLASH));

💎 Mini-Project: Fully Shaded Sprite

Compose the tier into one production-shaped sprite material:

  • palette-swap the base colour (original / fire / ice)
  • a toggleable selection outline
  • a hit-flash on click

It's state-built — the buttons rebuild the shader from a small state object, the same pattern as Beginner's water and Intermediate's fire.

📝 Click to see: the composed sprite material 
function buildFrag(s) {
    const pal = PALETTES[s.pal];
    const colorExpr = pal
        ? '(l < 0.5 ? mix(' + pal[0] + ', ' + pal[1] + ', l*2.0) : mix(' + pal[1] + ', ' + pal[2] + ', (l-0.5)*2.0))'
        : 'tex.rgb';
    const outlineBlock = s.outline ? '/* 8-tap dilation → col */' : '';
    const flashBlock = s.flash
        ? '  spr = mix(spr, vec3(1.0), 0.5 + 0.5 * sin(u_time * 18.0));' : '';
    return GLSL_HEAD + `void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec2 sUV = toSpriteUV(uv);
  vec4 tex = texture2D(u_tex, sUV);
  float l = dot(tex.rgb, vec3(0.299, 0.587, 0.114));
  vec3 spr = ${colorExpr};
${flashBlock}
  vec3 col = backdrop(uv);
${outlineBlock}
  col = mix(col, spr, tex.a);
  gl_FragColor = vec4(col, 1.0);
}`;
}

${flashBlock}
