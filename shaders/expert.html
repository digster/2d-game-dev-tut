🖼️ Sampling a Texture

A texture is an image the GPU can read per pixel. Declare a uniform sampler2D u_tex; and read it with texture2D(u_tex, uv) where uv is 0..1 across the image. That's the whole API — here's a complete minimal texture shader: 

precision mediump float;
uniform vec2 u_resolution;
uniform sampler2D u_tex;            // the sprite, on texture unit 0

void main() {
    vec2 uv = gl_FragCoord.xy / u_resolution;
    gl_FragColor = texture2D(u_tex, uv);   // rgba straight from the image
}

Two gotchas, both fixed once in the runner

  • Y is flipped. A 2D canvas is Y-down; gl_FragCoord is Y-up. The runner sets UNPACK_FLIP_Y_WEBGL so the sprite samples upright (mirrors the Beginner u_mouse Y-flip fix).
  • Aspect. A square 256² sprite in an 800×450 canvas would squash. toSpriteUV() maps a centred, aspect-correct square box; outside it a checkerboard backdrop() shows through the sprite's transparent margin. Try "Stretched" to see why the fit matters.
📝 Click to see: sampling the sprite 
// GLSL_HEAD declares the uniforms + toSpriteUV()/backdrop() helpers.
const FIT = GLSL_HEAD + `void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec4 tex = texture2D(u_tex, toSpriteUV(uv));
  gl_FragColor = vec4(mix(backdrop(uv), tex.rgb, tex.a), 1.0);
}`;

🎨 Tinting & Multiply

The simplest sprite shader, and one of the most used: multiply the sampled colour by a tint. Because it's a multiply, the sprite's own shading and shadows survive — a flat overlay would erase them. Keep tex.a so the silhouette stays crisp.

vec3 tinted = tex.rgb * tint; — that's the entire effect. It's the team-colour in an RTS, the red damage flash, the gold of a power-up, the grey of a disabled button.

📝 Click to see: tint multiply 
function buildFrag(tint) {
    return GLSL_HEAD + `void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec4 tex = texture2D(u_tex, toSpriteUV(uv));
  vec3 tinted = tex.rgb * ${tint};        // multiply keeps shading
  gl_FragColor = vec4(mix(backdrop(uv), tinted, tex.a), 1.0);
}`;
}
// Damage tint:
toy.setFrag(buildFrag('vec3(1.0, 0.35, 0.35)'));

🌈 Palette Swap

Ship one sprite, recolour it a dozen ways. Read the sample's luminance (perceived brightness) and remap it through a small colour ramp. The four luminance bands the gem is drawn in become four stops of any palette — fire, ice, poison, gold-rare.

float l = dot(tex.rgb, vec3(0.299, 0.587, 0.114));
vec3 c = l < 0.5 ? mix(A, B, l*2.0) : mix(B, C, (l-0.5)*2.0);

This is how NES-era games (and modern pixel-art ones) get a whole palette of enemy variants from a single sprite sheet.

📝 Click to see: luminance → ramp 
function buildFrag(a, b, c) {
    return GLSL_HEAD + `void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec4 tex = texture2D(u_tex, toSpriteUV(uv));
  float l = dot(tex.rgb, vec3(0.299, 0.587, 0.114));
  vec3 ramp = l < 0.5
    ? mix(${a}, ${b}, l * 2.0)
    : mix(${b}, ${c}, (l - 0.5) * 2.0);
  gl_FragColor = vec4(mix(backdrop(uv), ramp, tex.a), 1.0);
}`;
}

💨 Dissolve & Burn

Death, teleport-out, a disintegrating enemy. Sample the Intermediate-tier value noise per pixel and compare it to a progress value: where noise < progress, drop the pixel. A thin band right at the threshold glows — the burning edge.

💡 discard & thresholding: here we simply skip colouring dissolved pixels (leaving the backdrop). GLSL ES 1.00's discard does the same for true alpha sprites. progress is a continuous knob → the u_param uniform; the "Auto" button instead drives it from fract(u_time) for a looping effect.

📝 Click to see: noise-thresholded dissolve 
// GLSL_NOISE adds hash()/valueNoise() (from the Intermediate tier).
function buildFrag(progressExpr) {
    return GLSL_HEAD + GLSL_NOISE + `void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec2 s = toSpriteUV(uv);
  vec4 tex = texture2D(u_tex, s);
  float prog = ${progressExpr};
  float n = valueNoise(s * 14.0);
  vec3 col = backdrop(uv);
  if (tex.a > 0.5) {
    if (n >= prog) {
      col = tex.rgb;
      float edge = smoothstep(prog, prog + 0.08, n);
      col = mix(vec3(1.0, 0.6, 0.1), col, edge);   // burn edge
    }
  }
  gl_FragColor = vec4(col, 1.0);
}`;
}

⭕ Outline / Rim

The selection highlight in every strategy game. A pixel just outside the sprite should glow if one of its neighbours is inside. Sample the sprite's alpha at the eight neighbouring texels — one texel is 1.0 / u_texResolution — and if any is opaque while this pixel isn't, you're on the rim.

Why u_texResolution

To step exactly one texel you must know the texture's pixel size. The runner passes it as u_texResolution (256, 256). u_param scales the step → outline thickness, with zero recompiles.

📝 Click to see: 8-tap alpha dilation 
// 8 fixed taps (no loop) → GLSL ES 1.00-safe.
float silhouette(vec2 s) {
  vec2 px = u_param / u_texResolution;      // texel step * thickness
  float m = 0.0;
  m = max(m, texture2D(u_tex, s + vec2( px.x, 0.0)).a);
  m = max(m, texture2D(u_tex, s + vec2(-px.x, 0.0)).a);
  m = max(m, texture2D(u_tex, s + vec2(0.0,  px.y)).a);
  m = max(m, texture2D(u_tex, s + vec2(0.0, -px.y)).a);
  m = max(m, texture2D(u_tex, s + px).a);
  m = max(m, texture2D(u_tex, s - px).a);
  m = max(m, texture2D(u_tex, s + vec2( px.x, -px.y)).a);
  m = max(m, texture2D(u_tex, s + vec2(-px.x,  px.y)).a);
  return m;
}

⚡ Hit-Flash & Invincibility Blink

The two pieces of damage feedback every action game ships. Hit-flash: mix the sprite toward white on a fast pulse. Invincibility blink: toggle the sprite's alpha on and off during i-frames. Both are one line driven by u_time.

💡 Triggering it for real: in a game you'd start the pulse on the damage event and let it decay. Here, clicking the canvas re-arms the flash — the same idea, JS-side. (A one-shot decay needs a "hit time" uniform; that's an Advanced-tier topic.)
📝 Click to see: flash & blink 
const FLASH = GLSL_HEAD + `void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec4 tex = texture2D(u_tex, toSpriteUV(uv));
  float f = 0.5 + 0.5 * sin(u_time * 18.0);
  vec3 hit = mix(tex.rgb, vec3(1.0), f);          // white-out pulse
  gl_FragColor = vec4(mix(backdrop(uv), hit, tex.a), 1.0);
}`;
const BLINK = GLSL_HEAD + `void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec4 tex = texture2D(u_tex, toSpriteUV(uv));
  float on = step(0.5, fract(u_time * 6.0));      // i-frame blink
  gl_FragColor = vec4(mix(backdrop(uv), tex.rgb, tex.a * on), 1.0);
}`;
canvas.addEventListener('click', () => toy.setFrag(FLASH));

💡 2D Normal-Map Lighting

This is what makes 2D games look 3D: a moving light that actually wraps the art. You don't need a hand-painted normal map — for a single sprite you can treat its luminance as a height field, finite-difference it into a surface normal, and shade it with Lambert N·L from a point light. Move the mouse: it's the light.

Heightmap → normal → light

  • Height h = luminance(sprite) — bright = raised.
  • Normal from neighbour texels: N = normalize(vec3(hL-hR, hD-hU, k)).
  • Light diff = max(dot(N, L), 0.0) with L toward the mouse.
📝 Click to see: lit / normals / diffuse / flat 
// GLSL_HEAD gives u_tex / u_texResolution / toSpriteUV / backdrop.
// We add u_mouse ourselves (the runner fills it if the shader declares it).
function buildFrag(out) {
    return GLSL_HEAD + `
uniform vec2 u_mouse;
float lum(vec3 c) { return dot(c, vec3(0.299, 0.587, 0.114)); }
float H(vec2 uv) { vec4 t = texture2D(u_tex, uv); return lum(t.rgb) * t.a; }
void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec2 s  = toSpriteUV(uv);
  vec4 tex = texture2D(u_tex, s);
  vec2 px = 1.5 / u_texResolution;
  vec3 N = normalize(vec3(H(s - vec2(px.x,0.0)) - H(s + vec2(px.x,0.0)),
                          H(s - vec2(0.0,px.y)) - H(s + vec2(0.0,px.y)), 0.6));
  vec2 lp = u_mouse / u_resolution;
  vec3 L = normalize(vec3((lp - uv) * vec2(u_resolution.x/u_resolution.y, 1.0), 0.35));
  float diff = max(dot(N, L), 0.0);
  float spec = pow(max(dot(reflect(-L, N), vec3(0.0,0.0,1.0)), 0.0), 24.0);
  vec3 c = ${out};
  gl_FragColor = vec4(mix(backdrop(uv), c, tex.a), 1.0);
}`;
}

const toy = makeShaderToy(canvas,
    buildFrag('tex.rgb * (0.15 + diff) + spec * 0.6'),
    { info: info, sprite: drawSprite });

🎞️ Sprite-Sheet Animation

Every animated 2D character is one image — a sprite sheet — sampled one cell at a time. The shader picks the current frame from a time-driven index and remaps uv into that cell. Here the "art" is a procedurally painted 4×4 sheet of a spinning gem.

The atlas UV remap

frame = mod(floor(time · fps), 16);
cell = vec2(mod(frame,4), floor(frame/4));
uv   = (localUV + cell) / 4;

One texture, a tiny bit of mod/floor math, zero extra draw calls.

📝 Click to see: the sheet painter + frame-picker shader 
// Custom sprite painter: a 4x4 grid of frames (gem rotating per cell).
function drawSheet(ctx, n) {
    ctx.clearRect(0, 0, n, n);
    const cs = n / 4;
    for (let f = 0; f < 16; f++) {
        const cx = (f % 4) * cs + cs / 2;
        const cy = Math.floor(f / 4) * cs + cs / 2;
        const ang = f / 16 * Math.PI * 2, R = cs * 0.32;
        ctx.save(); ctx.translate(cx, cy); ctx.rotate(ang);
        ctx.beginPath();
        ctx.moveTo(0, -R); ctx.lineTo(R * 0.7, 0);
        ctx.lineTo(0, R);  ctx.lineTo(-R * 0.7, 0);
        ctx.closePath(); ctx.fillStyle = '#4fc3f7'; ctx.fill();
        ctx.restore();
    }
}

// fps injected as a GLSL constant; UNPACK_FLIP_Y means row 0 is the TOP.
function buildFrag(fps) {
    return GLSL_HEAD + `
void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec2 s  = clamp(toSpriteUV(uv), 0.0, 1.0);
  float frame = mod(floor(u_time * ${fps}), 16.0);
  vec2 cell = vec2(mod(frame, 4.0), floor(frame / 4.0));
  vec2 fuv = (s + vec2(cell.x, 3.0 - cell.y)) / 4.0;
  vec4 tex = texture2D(u_tex, fuv);
  gl_FragColor = vec4(mix(backdrop(uv), tex.rgb, tex.a), 1.0);
}`;
}

🟦 Dither / Posterize / 1-bit

The retro look — Game Boy, Obra Dinn, 1-bit jams — is color quantization. Snap colors to a few levels (posterize), threshold to two (1-bit), and add an ordered Bayer dither so the eye reads more shades than are really there.

Quantize, then dither

  • Posterize: floor(col * n) / n → visible bands.
  • Dither: add a 4×4 Bayer threshold before the floor → bands dissolve into a pattern.
  • 1-bit: compare luminance against the Bayer value → pure two-tone.
📝 Click to see: posterize / dither / 1-bit / palette 
// 4x4 Bayer built recursively from the 2x2 base [[0,2],[3,1]].
const LIB = GLSL_HEAD + `
float lum(vec3 c) { return dot(c, vec3(0.299, 0.587, 0.114)); }
float B2(vec2 c) { return mix(mix(0.0,2.0,c.x), mix(3.0,1.0,c.x), c.y); }
float bayer4(vec2 P) {
  vec2 hi = mod(floor(P * 0.5), 2.0);
  vec2 lo = mod(P, 2.0);
  return (4.0 * B2(hi) + B2(lo)) / 16.0;
}`;

function buildFrag(mode) {
    return LIB + `
void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec4 tx = texture2D(u_tex, toSpriteUV(uv));
  vec3 col = mix(backdrop(uv), tx.rgb, tx.a);
  vec2 P = floor(gl_FragCoord.xy);
  vec3 outc;
  ${mode}
  gl_FragColor = vec4(outc, 1.0);
}`;
}

const toy = makeShaderToy(canvas,
    buildFrag('outc = floor(col * 4.0 + bayer4(P)) / 4.0;'),

✒️ Cross-Hatch & Halftone Shading

Dithering quantised colour. The next step is non-photorealistic tonal shading: throw the colour away, keep only luminance, and re-draw it as ink — engraved cross-hatching, a newspaper halftone dot screen, or a 1-bit ordered dither. Same idea every time: darker pixel → more ink.

Tone → ink, three ways

L = dot(rgb, vec3(0.299, 0.587, 0.114));   // perceived brightness
hatch  : unlock another rotated line layer as L drops
halftone: dot radius = sqrt(1.0 - L)
dither  : step(interleavedGradientNoise(px), L)

The hatch lines and halftone dots are computed in screen space (gl_FragCoord.xy) so the "ink" stays a constant physical size regardless of the sprite — exactly how a real engraving behaves.

💡 Structural vs continuous: the screen kind (hatch / halftone / dither) changes the code path → setFrag; the screen frequency is just a scale → the u_param uniform ("Coarser").
📝 Click to see: the cross-hatch / halftone shader 
// Each mode injects a `float ink(vec2 px, float L)` (0 = ink, 1 = paper):
const INK = {
    hatch: `float ink(vec2 px, float L){
  float F = mix(0.06, 0.16, u_param);
  float k = 1.0;
  if (L < 0.85) k = min(k, step(0.5, fract((px.x + px.y) * F)));
  if (L < 0.40) k = min(k, step(0.5, fract( px.x        * F * 1.4)));
  return k; // ... more layers as L drops ...
}`,
    dither: `float ign(vec2 p){ return fract(52.9829189*fract(dot(p,vec2(0.0671,0.0058)))); }
float ink(vec2 px, float L){ return step(ign(px), L); }`
    // ... halftone: rotated dot grid, radius = sqrt(1.0 - L) ...
};

🎨 Stylized Sprite: ASCII & Oil Paint

Two more stylizers that change the structure, not just the tone. ASCII collapses a block of pixels to one luminance value and draws a brightness-keyed glyph (the terminal-art look). Oil paint is a compact Kuwahara filter: sample four overlapping quadrants and keep the flattest one — it smooths interiors while keeping edges crisp, like a palette knife.

Kuwahara in one breath

for each of 4 corner quadrants: compute mean & variance
output = mean of the quadrant with the lowest variance

Low variance = "this side is a smooth region" — so a pixel on an edge copies the calm side instead of blurring across the boundary. That edge-preserving property is why it reads as paint rather than blur.

💡 ES 1.00 reminder: the Kuwahara window K and the ASCII cell count are for-loop bounds / grid constants — structural, so "Detail +" rebuilds the shader (setFrag), the same rule as fbm octaves.

// Oil = compact Kuwahara: 4 quadrant means, keep the lowest-variance one.
const oil = (k) => `const int K = ${k};
void region(vec2 s, vec2 px, vec2 d, out vec3 mean, out float varr){
  vec3 sum=vec3(0.0), sum2=vec3(0.0); float n=0.0;
  for (int j=0;j<=K;j++) for (int i=0;i<=K;i++){
    vec3 c = texture2D(u_tex, s + d*vec2(float(i),float(j))*px).rgb;
    sum+=c; sum2+=c*c; n+=1.0;
  }
  mean = sum/n; vec3 v = sum2/n - mean*mean; varr = v.r+v.g+v.b;
}`;

function buildFrag(s) {
    const pal = PALETTES[s.pal];
    const colorExpr = pal
        ? '(l < 0.5 ? mix(' + pal[0] + ', ' + pal[1] + ', l*2.0) : mix(' + pal[1] + ', ' + pal[2] + ', (l-0.5)*2.0))'
        : 'tex.rgb';
    const outlineBlock = s.outline ? '/* 8-tap dilation → col */' : '';
    const flashBlock = s.flash
        ? '  spr = mix(spr, vec3(1.0), 0.5 + 0.5 * sin(u_time * 18.0));' : '';
    return GLSL_HEAD + `void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  vec2 sUV = toSpriteUV(uv);
  vec4 tex = texture2D(u_tex, sUV);
  float l = dot(tex.rgb, vec3(0.299, 0.587, 0.114));
  vec3 spr = ${colorExpr};
${flashBlock}
  vec3 col = backdrop(uv);
${outlineBlock}
  col = mix(col, spr, tex.a);
  gl_FragColor = vec4(col, 1.0);
}`;
}
interface SpriteState { pal: number; outline: boolean; flash: boolean; }
