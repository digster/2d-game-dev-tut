🎯 What You'll Build

Shaders are how 2D games get their look — water, lava, lighting, fog, screen transitions, hit flashes, the retro CRT filter. They run on the GPU, in parallel, essentially for free. This track teaches them from gl_FragColor up, always tied to concrete game effects.

🎬 By the end of the track

  • A library of full-screen GLSL effects — water, fire, fog, vignette, CRT/scanline, color grading
  • A multi-pass post-processing chain — render-to-texture → blur → bloom
  • Sprite/material shaders — dissolve, outline, hit-flash, palette-swap
  • GPU compute-style effects — reaction-diffusion, fluid, particles on the GPU
  • Every demo is exportable — 📋 Export gives a standalone runnable HTML
💡 Why a shaders track: the Fundamentals and the Isometric Strategy track teach gameplay and math on the 2D canvas. Shaders are the other half of "game feel" — the visual layer — and they're a distinct skill (the GPU, GLSL, the per-pixel mindset). Hence a separate track.

🗺️ Track Roadmap

Five tiers, each in its own file so it stays focused and quick to load. Tiers build up: fundamentals first, then progressively more game-oriented effects. The track is shipped iteratively — only tiers marked Ready are written yet.

📚 Recommended Prerequisites

You can start cold, but a little intuition from the Fundamentals guide makes shaders click faster:

Strongly recommended before BEGINNER

  • Vectors: Position & Direction — vec2/vec3/vec4 are vectors; same algebra
  • Trigonometry for Game Developers — sin/cos drive every animated shader
  • Linear Interpolation (Lerp) — GLSL's mix() is lerp
  • Easing Functions & Smooth Motion — smoothstep is an easing curve

Helpful before INTERMEDIATE

  • Perlin Noise & Terrain Generation — we port noise to GLSL
  • Matrix Transformations — for rotating & warping UVs
💡 Note: color math (RGB, normalized 0–1 channels, palettes) is taught inline in this track — there's no separate Fundamentals page for it yet, so don't go looking for one.

💡 How to Use This Track

👶 If you're new to game dev

Skim the Fundamentals Vectors and Trig pages first, then start the Beginner tier here. You don't need WebGL experience — the track wraps all the boilerplate.

🛠️ If you've made 2D games but never a shader

Start at the Beginner tier. It's hands-on from the first section and ends with a water effect you can paste into a real game.

🎮 If you've written shaders before

Skim Beginner for the helper/Export conventions, then jump ahead as the later tiers land (noise, sprite shaders, multi-pass post-FX, GPU sims).
