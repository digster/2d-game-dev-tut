🎯 What You'll Build

This track teaches you how to build a top-down 2D racing game from first principles. We lean sim, not arcade: every demo reflects how real cars actually move — heading separate from velocity, dt-correct integration, grip thresholds, weight transfer. Think the overhead camera moments of Race Driver GRID, the indie sim REDLINE, or the original top-down Need For Speed demo modes — but with the math exposed.

🎬 The final game (by the end of the Expert tier)

  • A drivable car with throttle, brake, and steering — WASD/arrows or gamepad
  • Believable grip — lateral friction, slip onset, oversteer/understeer felt in the wheel
  • A closed race track built from a Bézier centerline, with wall collision and bounce
  • Lap counting + checkpoints and a start-lights → green → finish race state machine
  • An AI opponent that follows the racing line using steering behaviours
  • A smooth follow camera with lead anticipation and a telemetry HUD
  • A simplified Pacejka tire curve so high-speed cornering feels right
💡 Why a racing sim is the perfect capstone: a top-down racer is the smallest possible game that exercises continuous physics, player input, AI driving, collision response, spline math, state machines, and camera work — every one of the Fundamentals tiers shows up. It's also the easiest genre in which "feel" is the whole game, so you'll learn to tune.

📚 Recommended Prerequisites

You can start this track cold, but you'll move faster if you've already worked through these topics from the Fundamentals guide:

Strongly recommended before BEGINNER

  • Vectors: Position & Direction — a car's position and velocity are 2D vectors
  • Trigonometry for Game Developers — heading + cos/sin gives velocity
  • Matrix Transformations — translate+rotate draws the car sprite
  • Delta Time & Fixed Timestep — every demo here is dt-correct

Strongly recommended before INTERMEDIATE

  • Friction & Drag Systems — the grip model is friction with a threshold
  • Physics Systems with Forces — once a car has mass, throttle becomes a force

Strongly recommended before ADVANCED

  • Collision Detection — wall collision against track edges
  • Bouncing & Reflection — for the wall bounce response
  • Bézier Curves & Splines — track centerlines and racing lines are splines
  • Steering Behaviours (AI Movement) — the AI opponent runs on these

Strongly recommended before EXPERT

  • State Machines — for the race state machine (lights → green → finish)
  • Spring Physics & Damping — used for the camera lead and (in Sims) suspension

Strongly recommended before SIMULATIONS

  • Spring Physics & Damping — the suspension model is a spring + damper
  • Procedural Generation — for the procedural track generator

💡 How to Use This Track

👶 If you're new to game dev

Don't start here — start with the Beginner Fundamentals first, then come back. This track assumes you've seen vectors, basic trig, and a canvas before.

🛠️ If you've made small games before

Start at the Beginner tier of this track. The first tier locks in the car-state-plus-integration loop you'll lean on for the rest of the journey, and ends with a playable sandbox you can drive around.

🏎️ If you've shipped a racing game before

Skim the Beginner and Intermediate tiers, jump to Advanced for the race loop and AI, and use the Simulations tier as your reference for the deeper tire/aero/suspension math.
