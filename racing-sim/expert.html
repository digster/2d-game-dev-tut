✨ What "Sim Polish" Means

Up through Advanced we had a working racing game — track, walls, laps, AI. But every veteran of iRacing or Gran Turismo would smell a hobby project in five seconds: the cars snap from grip to slide with no warning, the camera is bolted to the world, and the race just starts with no countdown. This tier closes those gaps.

📐 The six polish steps

  1. Pacejka tire curve — replace the hard grip clamp with a smooth force-vs-slip-angle curve that peaks then drops. Suddenly slide is controllable.
  2. Weight transfer — when you brake, the front loads up (more grip); when you accelerate, the rear does. Visual indicator only at this tier; full two-axle physics lands in Simulations.
  3. Counter-steer — once Pacejka is in, the steering wheel can meaningfully oppose the slide. We show why and how.
  4. Race state machine — formal PRE_RACE → COUNTDOWN → RACING → FINISHED. Same pattern from Fundamentals.
  5. Follow camera with lead — the camera lerps toward the car plus a velocity-based lead offset. Plays like Forza, not like Atari.
  6. Gamepad input — the Gamepad API gives you analog steering and triggers for free. We fall back to WASD when no pad is connected.
💡 What stays the same: every previous tier's code keeps working. The Pacejka integrator is a strict superset of the grip-clamp integrator (set the right coefficients and you recover the Intermediate behaviour). 🛞 The Pacejka Tire Curve

The grip clamp from Intermediate (cancel up to gripLimit·dt of lateral velocity; anything beyond survives) is a step function. Real tires aren't step functions — they produce a force that depends on slip, with a clear peak. The most common model is Hans Pacejka's Magic Formula:

🧮 The Magic Formula (simplified)

Flateral = D · sin(C · arctan(B · α))

where:

  • α (alpha) — the slip angle, in radians: the angle between the velocity vector and the heading. Zero when gripping perfectly; positive or negative once you start sliding.
  • B — stiffness factor: how steep the curve climbs from zero. Higher B = sharper response, lower peak slip.
  • C — shape factor: controls how the curve falls past the peak. Values around 1.3–1.6 give the "rising then falling" shape that lets tires snap.
  • D — peak factor: the maximum force the tire can generate at the optimal slip angle. This is what surface preset adjusts in our model.

The peak sits at α_peak = tan(π/(2·C)) / B. With B=8, C=1.5 that's tan(π/3)/8 ≈ 12.4°. Real street tyres tend to peak around 6–10°; if you want a tighter peak, raise B (try B=15 → peak ≈ 6.6°).

// alpha in radians; returns a force in [-D, +D].
function pacejka(alpha, B = 8, C = 1.5, D = 1.0) {
    return D * Math.sin(C * Math.atan(B * alpha));
}

// In an integrator: convert lateral velocity to slip angle, then look up
// the force the tyre generates, then apply that force to v_lateral.
function applyPacejkaLateral(vForward, vLateral, gripLimit, dt, B, C, D) {
    // Slip angle relative to forward velocity. Guard against zero forward speed.
    const fwd = Math.max(Math.abs(vForward), 1e-3);
    const alpha = Math.atan2(vLateral, fwd);  // radians
    // Pacejka gives a normalised force in [-D, +D]; scale by gripLimit so the
    // peak matches the surface's grip budget.
    const force = pacejka(alpha, B, C, D) * gripLimit;
    // The force opposes lateral velocity, so we *subtract* (force has the same
    // sign as alpha, which has the same sign as vLateral).
    return vLateral - force * dt;
}
function pacejka(alpha: number, B: number = 8, C: number = 1.5, D: number = 1.0): number {
    return D * Math.sin(C * Math.atan(B * alpha));
}

function applyPacejkaLateral(
    vForward: number,
    vLateral: number,
    gripLimit: number,
    dt: number,
    B: number,
    C: number,
    D: number
): number {
    const fwd = Math.max(Math.abs(vForward), 1e-3);
    const alpha = Math.atan2(vLateral, fwd);
    const force = pacejka(alpha, B, C, D) * gripLimit;
    return vLateral - force * dt;
}

🤔 Why does the curve fall past the peak?

A tyre sliding past its peak slip angle is partially airborne — the contact patch starts to skim instead of grip. Less rubber on the road means less force. This falling slope is the entire physics of "losing the back end" — once you exceed peak, more steering makes things worse, not better. That's what makes real cars hard to catch, and what counter-steer (next section) tries to undo.

⚖️ Weight Transfer Under Brake & Throttle

A real car's weight is split (roughly) 50/50 front/rear when coasting. The moment you brake, that weight shifts forward — the front tyres press harder into the road and find more grip; the rears lighten and find less. Throttle does the opposite. Cornering shifts weight to the outside tyres. This is why pro drivers brake while turning in (trail braking) — the front grip surplus from braking helps the front tyre bite into the corner.

📐 The simplest model

Real weight transfer involves CG height, wheelbase, and load shift formulas. For a top-down sim that's overkill. A useful approximation:

frontLoad = 0.5 − k · alongitudinal / amax
rearLoad = 1 − frontLoad

where k is a single tunable in roughly [0.1, 0.3]. Positive a (accelerating) reduces front load; negative a (braking) increases it. Both frontLoad and rearLoad become multipliers on the relevant tyres' peak force D in the Pacejka curve.

↩️ Counter-Steer & the Drift Save

With the Pacejka curve in place, drift becomes a controllable state. If the back end starts to come around (slip angle past peak), the falling side of the curve means more steering INTO the original turn makes the slide worse. The fix is counter-steer — turn the wheels the other way, briefly, until slip drops back below the peak. Then you can steer normally again.

📐 What the Pacejka curve is telling you

The slip angle is atan2(v_lateral, v_forward). To bring it down, you need to either reduce v_lateral or increase v_forward. Counter-steer rotates the car so its heading lines up with where it's already going — which directly reduces the slip angle without needing the tyres to do impossible work.

In our hot-lap demo, the HUD shows live slip angle in degrees. Watch it during a deliberate drift: it climbs past 12°, the back swings out, you tap the opposite direction, and the slip angle drops back below peak as the heading catches up with the velocity vector.

🤔 Why this isn't taught in the Intermediate tier

The Intermediate grip clamp doesn't have a peak. Once you're past the threshold, the lateral velocity decays linearly back toward zero — the slide is "self-correcting" because nothing makes it worse. Without a peak, counter-steer is a no-op. The Pacejka curve creates the need for counter-steer, and that's what makes it the real sim model.

🚦 Race State Machine — Lights, Green, Finish

Up to now the race just was — reset, drive, win. A real race has phases. We model the lifecycle with a tiny state machine — formally the same pattern as State Machines in Fundamentals.

📐 The four states

  1. PRE_RACE — track laid out, car at the line, no input accepted.
  2. COUNTDOWN — red lights light up one per second, then all go green. Input still locked.
  3. RACING — timer started, input live, lap timer accumulating.
  4. FINISHED — final lap completed, timer frozen, banner up, input ignored.

Each state has an enter(), an update(dt), and an exit condition. The race machine drives everything — the input layer reads its state to know whether to forward throttle to the integrator, the HUD reads it to render lights vs lap timer, the camera reads it to decide whether to centre on the lights or follow the car.

📷 Smooth Follow Camera with Lead

A good follow camera does three things: follow the car (don't lock to the world), smooth the follow (don't be twitchy), and look ahead (anticipate where the car is going so the player sees the next corner, not the corner they just left).

📐 The three camera flavours

  • Rigid: camera.position = car.position. Twitchy, no anticipation. Used in 1980s Atari arcade games.
  • Lerp: camera.position = lerp(camera.position, car.position, smoothness · dt). Smooth follow; lags slightly behind so corners read well. Good for top-down racers.
  • Lerp + lead: target the car's position plus a velocity-based offset: target = car.position + car.velocity · leadTime. Then lerp toward that. Anticipates corners by pushing the camera in the direction of travel.

The lerp is mathematically a damped spring (cross-link: Spring Physics & Damping) without the overshoot — exactly what you want for a camera.

🎮 Gamepad API Input

The browser's Gamepad API gives you analog input — exactly what driving wants. Left stick X for steering, right trigger for throttle, left trigger for brake. We poll on every frame (navigator.getGamepads()) — there is no event API for sticks/triggers because they're sampled continuously.

📐 The poll loop, in five lines

Plug a controller in, then press anything on it — the browser only exposes pads that have produced input recently.

// Returns { throttle, steer, connected, name } for the first gamepad found,
// or null when none is connected. Apply a small dead-zone so resting sticks
// don't drift.
function readGamepad(deadZone = 0.08) {
    const pads = navigator.getGamepads ? navigator.getGamepads() : [];
    for (const pad of pads) {
        if (!pad) continue;
        let steer    =  pad.axes[0] || 0;           // left stick X
        let throttle =  pad.buttons[7]?.value || 0; // right trigger
        let brake    =  pad.buttons[6]?.value || 0; // left trigger
        if (Math.abs(steer) < deadZone) steer = 0;
        // Combine throttle and brake into a single [-1, +1] axis. Brake wins.
        const net = brake > 0.1 ? -brake : throttle;
        return { throttle: net, steer, connected: true, name: pad.id };
    }
    return { throttle: 0, steer: 0, connected: false, name: null };
}
interface PadState {
    throttle: number;
    steer: number;
    connected: boolean;
    name: string | null;
}

function readGamepad(deadZone: number = 0.08): PadState {

🏆 Mini-Project: Hot-Lap with Telemetry

The capstone. A single car on a closed Bézier track. The race state machine drives the lights → green → finish sequence. The follow camera tracks you with velocity lead. The Pacejka integrator gives a smooth slip-to-grip transition. The telemetry HUD shows live speed, slip angle, brake/throttle bars, current lap time, last lap, and best lap. Three laps and the race ends.

🎮 What to try

  • Hold the wheel into a corner: watch the slip-angle gauge climb. Push past ~12° and you'll see the back step out — the curve has peaked.
  • Tap the opposite direction: classic counter-steer save. The slip-angle gauge falls back toward zero as the heading catches up with the velocity vector.
  • Trail-brake: brake while still steering into the corner. Front grip is up, so you can turn-in much sharper than under coast.
  • Compare laps: the best-lap pill turns green when you set a new personal best.
