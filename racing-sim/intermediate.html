🛞 From Hover-Disc to Car

The Beginner tier built a kinematic car that coasts forever. That's fine for proving the integration loop, but it doesn't feel like driving — release throttle and you glide; turn hard and you pirouette without any sense of inertia. This tier fixes both.

📐 The whole tier in one sentence

We split the car's velocity into two components — along the heading (forward / reverse) and perpendicular to it (sideways / lateral) — and apply different rules to each. Forward gets friction (slows you down). Lateral gets grip (the tires try to cancel sideways motion, up to a per-surface limit; beyond that limit you slide).

What changes in the integrator

The Beginner integrator looked like this (paraphrased):

  • Steer → heading.
  • Throttle × heading → acceleration → velocity.
  • Velocity × dt → position. Clamp to MAX_SPEED.

The Intermediate integrator adds two new steps:

  • Project velocity onto forward and lateral basis vectors.
  • Apply rolling drag to forward; apply the grip clamp to lateral. Recombine.
💡 The win: with these two extra steps, every later mechanic in the track falls out for free. Drift = "lateral velocity exceeded grip". Oversteer = "rear grip ran out before front grip did". Ice = "small gripLimit". Same integrator, different surface numbers — that's the whole story. Deeper friction theory in the Fundamentals: Friction & Drag Systems.

🔄 Friction & Drag

"Friction" in a top-down racer means two distinct things that we lump together for the math: rolling friction (tire deformation, bearing drag — proportional to speed) and aerodynamic drag (air resistance — proportional to speed squared). In a sim-leaning Beginner+Intermediate game, a single linear-in-speed term captures both well enough that you don't notice the difference; we'll separate them in the Expert tier.

📐 The math, as one line

v_forward ← v_forward × (1 − dragRate × dt)

This is multiplicative decay — every frame, we keep (1 − dragRate·dt) of the current forward velocity. Two reasons this beats v -= dragRate·dt:

  • Always stops eventually — no risk of overshooting zero into negative speed (you'd see a stopped car suddenly creep backwards otherwise).
  • Scales with speed naturally — at 200 px/s you lose proportionally more than at 20 px/s, which is what real air drag feels like.
📝 Click to see: applying friction along forward only 
// surface.rollingDrag is a per-second decay rate. Tarmac ~ 0.5, gravel ~ 1.5,
// ice ~ 0.2 (almost no rolling drag — the car keeps gliding).
function applyDrag(vForward, surface, dt) {
    return vForward * (1 - surface.rollingDrag * dt);
}
function applyDrag(
    vForward: number,
    surface: { rollingDrag: number },
    dt: number
): number {
    return vForward * (1 - surface.rollingDrag * dt);
}

📐 Lateral vs Longitudinal Velocity

A car's velocity vector lives in 2D, but its physics is best understood in the car's own frame: along the heading (forward), and perpendicular to it (lateral). A gripping car has nearly zero lateral velocity — it's pointed where it's going. A drifting car has lots of lateral velocity — it's pointed somewhere and travelling somewhere else.

🧮 Projection in two lines

The forward basis vector is (cos h, sin h) — same as the Beginner tier. The lateral basis vector is perpendicular to it: (−sin h, cos h) (rotate forward by 90°).

v_forward = v · (cos h, sin h)
v_lateral = v · (−sin h, cos h)

Both are dot products. The result is a single scalar each — how fast the car is going forward, and how fast it's drifting sideways. The Fundamentals refresher on the dot product is at Essential Vector Operations.

📝 Click to see: decompose / recombine 
// Forward + lateral basis vectors. Heading is in radians.
function carBasis(heading) {
    const fx = Math.cos(heading), fy = Math.sin(heading);
    return {
        forward: { x:  fx, y: fy },   // unit vector along the car
        lateral: { x: -fy, y: fx }    // unit vector to the car's left
    };
}

// Project a velocity vector onto the car's frame.
function decompose(velocity, basis) {
    return {
        vForward: velocity.x * basis.forward.x + velocity.y * basis.forward.y,
        vLateral: velocity.x * basis.lateral.x + velocity.y * basis.lateral.y
    };
}

// Recombine (inverse of decompose).
function recombine(vForward, vLateral, basis) {
    return {
        x: vForward * basis.forward.x + vLateral * basis.lateral.x,
        y: vForward * basis.forward.y + vLateral * basis.lateral.y
    };
}
type V = { x: number; y: number };
type Basis = { forward: V; lateral: V };

function carBasis(heading: number): Basis {
    const fx: number = Math.cos(heading);
    const fy: number = Math.sin(heading);
    return {
        forward: { x:  fx, y: fy },
        lateral: { x: -fy, y: fx }
    };
}

function decompose(velocity: V, basis: Basis): { vForward: number; vLateral: number } {
    return {
        vForward: velocity.x * basis.forward.x + velocity.y * basis.forward.y,
        vLateral: velocity.x * basis.lateral.x + velocity.y * basis.lateral.y
    };
}

function recombine(vForward: number, vLateral: number, basis: Basis): V {
    return {
        x: vForward * basis.forward.x + vLateral * basis.lateral.x,
        y: vForward * basis.forward.y + vLateral * basis.lateral.y
    };
}

🛞 The Grip Threshold & Onset of Slip

Real tires can only generate so much sideways force per second before they break loose. We model this with one number per surface: gripLimit, the maximum lateral velocity (in px/s) the tires can cancel per second. Below that, the car grips — lateral velocity goes to zero each frame. Above it, the excess survives as slide.

📐 The grip clamp, in one line

If |v_lateral| ≤ gripLimit × dt, the car grips:

v_lateral ← 0

Otherwise, the tires shave off as much as they can:

v_lateral ← v_lateral − sign(v_lateral) × gripLimit × dt

That's it. No PID controller, no Pacejka curve (we save that for Expert) — just one clamp. The "drift" you feel comes for free.

📝 Click to see: the grip clamp 
function applyGrip(vLateral, surface, dt) {
    const maxKill = surface.gripLimit * dt;
    if (Math.abs(vLateral) <= maxKill) return 0;
    return vLateral - Math.sign(vLateral) * maxKill;
}
function applyGrip(
    vLateral: number,
    surface: { gripLimit: number },
    dt: number
): number {
    const maxKill: number = surface.gripLimit * dt;
    if (Math.abs(vLateral) <= maxKill) return 0;
    return vLateral - Math.sign(vLateral) * maxKill;
}

↩️ Oversteer vs Understeer

Real cars have two sets of tires, front and back. When their grip limits differ, you get the most fundamental tuning knob in motorsport:

  • Understeer — front grip runs out first. The car wants to plow straight when you ask it to turn. Predictable but slow. Most road cars.
  • Oversteer — rear grip runs out first. The back end swings out, the car rotates faster than you steered for. Fast lap times if you can catch it, an instant spin if you can't. Drift cars, classic muscle cars, anything rear-wheel-drive.
  • Neutral — front and rear grip match. The car slides as a whole when you exceed grip. Theoretical ideal; rare in practice.

🎯 How we model it on one car

In a sim-leaning Beginner+Intermediate game we don't simulate two axles separately — that lands in the Expert tier. Instead we approximate the feel with one trick: when the rear loses grip, the car gains a yaw kick (heading turns extra). When the front loses grip, the steering input contributes less (the wheels skate sideways). One grip-balance tunable in [-1, +1] gives the whole spectrum.

🛣️ Surface Presets — Tarmac, Gravel, Ice

Every surface in this track is two numbers: a rollingDrag (how fast you coast to a stop) and a gripLimit (how much sideways velocity the tires can kill per second). Combining them gives wildly different feels from the same integrator.

🎨 The three starter surfaces

These are the numbers used in the mini-project. Tune freely.

Surface rollingDrag gripLimit Feel
Tarmac0.55620Sharp turns, no drift, quick stops
Gravel1.40240Loose, rallies easily, decent feedback
Ice0.2055Glides forever, almost no lateral grip

    gravel: { name: 'Gravel', color: '#6e5a3a', rollingDrag: 1.40, gripLimit: 240 },
    ice:    { name: 'Ice',    color: '#7fbcd4', rollingDrag: 0.20, gripLimit: 55  }
};

    name: string;
    color: string;
    rollingDrag: number;
    gripLimit: number;
}

🤔 Try this yourself

What if you wanted a "wet tarmac" preset? Real wet tarmac has nearly the same rolling drag as dry but only ~60% of the grip. So: { rollingDrag: 0.60, gripLimit: 380 }. Drop that into SURFACES, add a fourth surface button, done.

🏎️ Mini-Project: Drift Pad

Time to feel it. The drift pad is the Beginner parking-lot grown up — same controls (WASD or arrows), same persistent skid layer, but now with friction, the grip clamp, and a surface picker. Switch between tarmac, gravel, and ice and notice how the same throttle + steering input produces a completely different drawing on the floor.

🎮 What to try

  • Tarmac: drive a tight circle. Your skid trail is a clean tyre line.
  • Gravel: the same input now leaves a slightly wider, sliding arc — that's lateral velocity surviving the grip clamp.
  • Ice: try to drive in a straight line. You'll find the car keeps pointing where you want to go, but glides somewhere else entirely. That's lateral velocity overwhelming the (tiny) grip limit.
  • Watch the HUD: the GRIP / SLIDE indicator flips in real time as the grip clamp activates.
