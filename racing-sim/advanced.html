🛣️ From Open Pad to Closed Track

The Intermediate tier left us with a car that handles like a car — friction, grip, slide. What it didn't have is a track. No walls to bounce off, no laps to count, no opponent to chase. This tier closes that gap. By the end, you'll be racing a single AI opponent around a Bézier-shaped oval that you collide with, with proper lap timing.

📐 One data structure, three features

Almost everything in this tier is built on one shared structure: a centerline (an array of sampled points along the middle of the track) plus an inflation step that produces left/right wall segments. From that one structure we get:

  • Walls for collision (inflate the centerline outward and inward).
  • Checkpoints for laps (perpendicular line segments at evenly spaced centerline indices).
  • Racing line for AI (the centerline itself, or an offset version).

That's why this tier teaches more features than the previous ones but doesn't actually have more code. One data structure, many uses.

💡 What we're leaning on from Fundamentals: Bézier Curves & Splines for the track shape, Collision Detection + Bouncing & Reflection for wall bounce, and Steering Behaviours for the AI. We won't re-derive any of those — we'll compose them.

📐 Building a Centerline from Cubic Béziers

A cubic Bézier curve takes four control points P0, P1, P2, P3 and produces a smooth curve from P0 to P3 that bends toward the middle two handles. To build a closed track we chain several together end-to-end and make the last segment's endpoint equal the first's start. An oval needs only four cubics — one per quarter.

🧮 The cubic Bézier formula

B(t) = (1−t)³·P0 + 3(1−t)²t·P1 + 3(1−t)t²·P2 + t³·P3    for t ∈ [0, 1]

Sample B(t) at N evenly spaced t values (say N = 64 per segment) and you get a polyline approximation of the curve. That polyline becomes the centerline samples — the foundation for everything else in this tier. (Strictly speaking, "evenly spaced t" doesn't mean "evenly spaced arc-length" — see the deep-dive in the Simulations tier; for now it's close enough that no one will notice.)

Full refresher and interactive demos: Bézier Curves & Splines.

// One cubic Bézier, sampled at N points (excluding the final endpoint,
// because the next segment will repeat it).
function sampleCubic(p0, p1, p2, p3, n) {
    const out = [];
    for (let i = 0; i < n; i++) {
        const t = i / n;
        const u = 1 - t;
        out.push({
            x: u*u*u*p0.x + 3*u*u*t*p1.x + 3*u*t*t*p2.x + t*t*t*p3.x,
            y: u*u*u*p0.y + 3*u*u*t*p1.y + 3*u*t*t*p2.y + t*t*t*p3.y
        });
    }
    return out;
}

// A closed centerline from an array of cubic segments. Each segment is
// {p0,p1,p2,p3} and the next segment's p0 must equal the previous p3.
function buildCenterline(segments, samplesPerSegment = 64) {
    const points = [];
    for (const s of segments) {
        points.push(...sampleCubic(s.p0, s.p1, s.p2, s.p3, samplesPerSegment));
    }
    return points; // implicitly closed: points[0] follows points[points.length-1]
}
type P = { x: number; y: number };
type CubicSeg = { p0: P; p1: P; p2: P; p3: P };

function sampleCubic(p0: P, p1: P, p2: P, p3: P, n: number): P[] {
    const out: P[] = [];
    for (let i = 0; i < n; i++) {
        const t = i / n;
        const u = 1 - t;
        out.push({
            x: u*u*u*p0.x + 3*u*u*t*p1.x + 3*u*t*t*p2.x + t*t*t*p3.x,
            y: u*u*u*p0.y + 3*u*u*t*p1.y + 3*u*t*t*p2.y + t*t*t*p3.y
        });
    }
    return out;
}

function buildCenterline(segments: CubicSeg[], samplesPerSegment: number = 64): P[] {
    const points: P[] = [];
    for (const s of segments) {
        points.push(...sampleCubic(s.p0, s.p1, s.p2, s.p3, samplesPerSegment));
    }
    return points;
}

Centerline → walls

Walls are just the centerline shoved outward by trackWidth / 2 along the local perpendicular. At sample i, the perpendicular is rotated 90° from the tangent (p[i+1] − p[i]). That gives you parallel left and right polylines, which break into a sequence of wall segments for collision tests.

📝 Click to see: inflate centerline to walls 
// Returns { left: [...], right: [...] } parallel polylines.
function inflateWalls(centerline, halfWidth) {
    const left = [], right = [];
    const n = centerline.length;
    for (let i = 0; i < n; i++) {
        const a = centerline[i];
        const b = centerline[(i + 1) % n];
        // Tangent vector
        const tx = b.x - a.x, ty = b.y - a.y;
        const len = Math.hypot(tx, ty) || 1;
        // Perpendicular (rotated 90° left). Flip the sign for "right".
        const nx = -ty / len, ny = tx / len;
        left.push({  x: a.x + nx * halfWidth, y: a.y + ny * halfWidth });
        right.push({ x: a.x - nx * halfWidth, y: a.y - ny * halfWidth });
    }
    return { left, right };
}
function inflateWalls(
    centerline: P[],
    halfWidth: number
): { left: P[]; right: P[] } {
    const left: P[] = [], right: P[] = [];
    const n = centerline.length;
    for (let i = 0; i < n; i++) {
        const a = centerline[i];
        const b = centerline[(i + 1) % n];
        const tx = b.x - a.x, ty = b.y - a.y;
        const len = Math.hypot(tx, ty) || 1;
        const nx = -ty / len, ny = tx / len;
        left.push({  x: a.x + nx * halfWidth, y: a.y + ny * halfWidth });
        right.push({ x: a.x - nx * halfWidth, y: a.y - ny * halfWidth });
    }
    return { left, right };
}

🧱 Wall Collision & Bounce

A wall is a line segment (two endpoints A and B). The car is a circle (position + radius). Collision is: "is the closest point on AB within radius distance of the car?". If yes, push the car out and reflect its velocity across the wall's normal.

🧮 Closest point on a segment

Project the car's position onto the infinite line through A and B, then clamp the parameter t to [0, 1] so we stay on the segment.

t = clamp(((C − A) · (B − A)) / |B − A|², 0, 1)
closest = A + t · (B − A)

If |C − closest| < radius, we have a collision.

🎱 Bounce via reflection

The wall's outward normal at the collision is the unit vector from closest to C. We push the car out along that normal until its edge no longer overlaps, then reflect the velocity using shared/utils.js's Vector2D.reflect(normal):

v_new = v − 2 · (v · n) · n

Multiply by a bounceDamping factor (e.g. 0.55) to lose energy on impact — pure reflection looks like a billiard ball and is too lively for cars. Refresher: Bouncing & Reflection.

📝 Click to see: car-vs-wall collision & bounce 
// Returns the closest point on segment AB to point C, plus t in [0,1].
function closestPointOnSegment(c, a, b) {
    const dx = b.x - a.x, dy = b.y - a.y;
    const denom = dx * dx + dy * dy;
    if (denom === 0) return { x: a.x, y: a.y, t: 0 };
    let t = ((c.x - a.x) * dx + (c.y - a.y) * dy) / denom;
    t = Math.max(0, Math.min(1, t));
    return { x: a.x + t * dx, y: a.y + t * dy, t };
}

// One wall = one segment. Returns true if we resolved a collision.
function collideCarWithWall(car, a, b, radius, bounceDamping = 0.55) {
    const closest = closestPointOnSegment(car.position, a, b);
    const dx = car.position.x - closest.x;
    const dy = car.position.y - closest.y;
    const dist = Math.hypot(dx, dy);
    if (dist >= radius || dist === 0) return false;
    // Normal — from the wall toward the car (unit length).
    const nx = dx / dist, ny = dy / dist;
    // Push the car out so its edge sits exactly on the wall.
    car.position.x += nx * (radius - dist);
    car.position.y += ny * (radius - dist);
    // Reflect velocity. Vector2D.reflect lives in shared/utils.js.
    const reflected = car.velocity.reflect({ x: nx, y: ny });
    car.velocity.x = reflected.x * bounceDamping;
    car.velocity.y = reflected.y * bounceDamping;
    return true;
}
function closestPointOnSegment(
    c: { x: number; y: number },
    a: { x: number; y: number },
    b: { x: number; y: number }
): { x: number; y: number; t: number } {
    const dx = b.x - a.x, dy = b.y - a.y;
    const denom = dx * dx + dy * dy;
    if (denom === 0) return { x: a.x, y: a.y, t: 0 };
    let t = ((c.x - a.x) * dx + (c.y - a.y) * dy) / denom;
    t = Math.max(0, Math.min(1, t));
    return { x: a.x + t * dx, y: a.y + t * dy, t };
}

function collideCarWithWall(
    car: { position: Vector2D; velocity: Vector2D },
    a: { x: number; y: number },
    b: { x: number; y: number },
    radius: number,
    bounceDamping: number = 0.55
): boolean {
    const closest = closestPointOnSegment(car.position, a, b);
    const dx = car.position.x - closest.x;
    const dy = car.position.y - closest.y;
    const dist = Math.hypot(dx, dy);
    if (dist >= radius || dist === 0) return false;
    const nx = dx / dist, ny = dy / dist;
    car.position.x += nx * (radius - dist);
    car.position.y += ny * (radius - dist);
    const reflected = car.velocity.reflect({ x: nx, y: ny } as Vector2D);
    car.velocity.x = reflected.x * bounceDamping;
    car.velocity.y = reflected.y * bounceDamping;
    return true;
}

🚩 Checkpoints & Lap Counting

A lap is "the car visited every checkpoint in order, then crossed the start line again". To detect that we lay perpendicular line segments across the centerline at evenly spaced indices, then test each frame whether the car's last-step movement (its position segment from previous-frame to this-frame) intersects the next checkpoint segment in order.

🎯 Why "next in order", not "any"?

If we accepted any checkpoint cross, a player could drive backwards across one and "complete" a lap. By tracking a nextCheckpoint index and only advancing it on the right crossing, we enforce going around the track the correct way. Hit the final checkpoint and the start line in order → lap completes.

📝 Click to see: ordered checkpoint progression 
// Returns updated { nextCheckpoint, laps } given that the car moved from
// prevPos to currPos this frame. Uses lineIntersection from shared/utils.js.
function tickCheckpoints(prevPos, currPos, checkpoints, nextCheckpoint, laps) {
    const cp = checkpoints[nextCheckpoint];
    if (lineIntersection(prevPos, currPos, cp.a, cp.b)) {
        nextCheckpoint = (nextCheckpoint + 1) % checkpoints.length;
        if (nextCheckpoint === 0) laps += 1; // wrapped → completed a lap
    }
    return { nextCheckpoint, laps };
}
type Checkpoint = { a: { x: number; y: number }; b: { x: number; y: number } };

function tickCheckpoints(
    prevPos: { x: number; y: number },
    currPos: { x: number; y: number },
    checkpoints: Checkpoint[],
    nextCheckpoint: number,
    laps: number
): { nextCheckpoint: number; laps: number } {
    const cp = checkpoints[nextCheckpoint];
    if (lineIntersection(prevPos, currPos, cp.a, cp.b)) {
        nextCheckpoint = (nextCheckpoint + 1) % checkpoints.length;
        if (nextCheckpoint === 0) laps += 1;
    }
    return { nextCheckpoint, laps };
}

🤖 AI Driving — Path Following as Steering

The Fundamentals tier covers steering behaviours — little vector recipes that move an agent toward a target. We use one of the simplest, seek with look-ahead, to make a car follow our centerline. The AI's "target" isn't where it is now; it's a point a few samples ahead. Look-ahead is why AI cars take corners instead of cutting straight across them.

📐 The AI loop, in three steps

  1. Find the look-ahead target: from the AI's current centerline index, skip forward by LOOKAHEAD samples. Wrap around the end of the array.
  2. Compute desired heading: atan2(target.y − pos.y, target.x − pos.x).
  3. Steer toward it: the shortest angular difference between current heading and desired, fed into the same integrateWithGrip as the player. Throttle eases off when steering hard, like a real driver.

The AI uses the same integrator as the player. It races with the same physics, same grip clamp, same friction. The only difference is where its throttle and steering numbers come from.

📝 Click to see: the AI driver 
// Returns { throttle, steer, currentIndex } for one frame of an AI driver.
function aiDrive(car, centerline, currentIndex, lookahead = 14) {
    // Advance currentIndex if we're close enough to the current waypoint
    // — keeps look-ahead anchored ahead of the car.
    const cur = centerline[currentIndex];
    const distCur = Math.hypot(car.position.x - cur.x, car.position.y - cur.y);
    if (distCur < 36) currentIndex = (currentIndex + 1) % centerline.length;

    // Target is `lookahead` samples ahead of currentIndex.
    const targetIdx = (currentIndex + lookahead) % centerline.length;
    const target = centerline[targetIdx];
    const desired = Math.atan2(target.y - car.position.y, target.x - car.position.x);

    // Shortest angular difference, then clamp to [-1, +1] steering.
    let diff = desired - car.heading;
    while (diff >  Math.PI) diff -= 2 * Math.PI;
    while (diff < -Math.PI) diff += 2 * Math.PI;
    const steer = clamp(diff * 2.2, -1, 1);

    // Throttle eases off when steering hard — basic "lift off in corners".
    const throttle = clamp(0.85 - Math.abs(steer) * 0.35, 0.15, 1);

    return { throttle, steer, currentIndex };
}
function aiDrive(
    car: { position: Vector2D; heading: number },
    centerline: { x: number; y: number }[],
    currentIndex: number,
    lookahead: number = 14
): { throttle: number; steer: number; currentIndex: number } {
    const cur = centerline[currentIndex];
    const distCur = Math.hypot(car.position.x - cur.x, car.position.y - cur.y);
    if (distCur < 36) currentIndex = (currentIndex + 1) % centerline.length;
    const targetIdx = (currentIndex + lookahead) % centerline.length;
    const target = centerline[targetIdx];
    const desired = Math.atan2(target.y - car.position.y, target.x - car.position.x);
    let diff = desired - car.heading;
    while (diff >  Math.PI) diff -= 2 * Math.PI;
    while (diff < -Math.PI) diff += 2 * Math.PI;
    const steer = clamp(diff * 2.2, -1, 1);
    const throttle = clamp(0.85 - Math.abs(steer) * 0.35, 0.15, 1);
    return { throttle, steer, currentIndex };
}

🎯 The Racing Line ≠ The Centerline

A real driver doesn't drive down the middle of the road — they hug the inside of each corner (the apex), set up wide on entry, and unwind wide on exit. That trajectory is the racing line, and on a closed circuit it's shorter and faster than the centerline. The classic geometric racing line is a sequence of arcs that touch the inside of each corner.

🛞 The cheap racing line we'll use

A full racing-line solver is a Simulations-tier topic. In this tier we approximate the racing line with a laterally-offset centerline — at each sample, shift a little toward the inside of the local curve (toward the center of the local turning radius). The result is good enough to look "racing-y" without needing a solver.

Our AI in the mini-project below uses the plain centerline for clarity. Swapping it for an offset racing line is a one-line change in aiDrive — just point centerline at a different array. Try it!

🤔 Try this yourself

Why does the racing line work? At constant grip, the maximum cornering speed is set by the turn's radius. The racing line maximises that radius (you trace the largest possible arc through each corner that still touches all three of: entry edge, apex, exit edge). Bigger radius → higher allowable speed. Same grip, faster lap.
