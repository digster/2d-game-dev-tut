🎮 What is an Isometric Game?

Isometric games show a 3D-looking world using only 2D drawing. Instead of looking at the world from straight above (top-down) or from the side (platformer), the camera is tilted at an angle so you can see the tops and fronts of objects at the same time. You get the spatial feel of 3D without any actual 3D math.

  • Age of Empires II — the inspiration for this track
  • Red Alert 2 / Command & Conquer — classic RTS look
  • Diablo II — same projection, action-RPG
  • SimCity 2000 / RollerCoaster Tycoon — city builders love iso
  • Monument Valley — a modern stylized take

📐 Three Projection Variants Compared

Not all "isometric" games are projected the same way. Three flavors dominate. Knowing the difference saves you from copy-pasting math that's subtly wrong for your style.

1. True isometric (30° / 30°)

The mathematically pure case: each grid axis is rotated 30° from horizontal. A unit step on X moves you (cos 30°, sin 30°) ≈ (0.866, 0.5) pixels. Equal length on all three axes (X, Y, Z) — that's literally what "iso-metric" means.

Used by: Q*bert, classic arcade-era iso games. Looks "purest" but pixel-art tiles end up with awkward step patterns because 0.866 isn't a clean integer.

2. Diamond (2:1) — the pixel-art standard

A pragmatic cheat: pretend the projection is "26.57°" so tile width is exactly twice the tile height. A 64×32 tile is the classic. Pixel-art lines have a clean 2-pixels-right-for-every-1-pixel-down slope — no anti-aliasing fuzz needed.

Used by: Age of Empires II, Red Alert 2, Diablo II, StarCraft. This is what we'll build with.

3. Staggered isometric

Tiles are drawn on a regular rectangular grid, but every other row is offset horizontally by half a tile. Looks identical to diamond visually, but the underlying map data is a normal 2D array — no diagonal math needed for storage.

Used by: Civilization II, many Tiled-editor maps. Easier to store but neighbor-finding gets weird (you have different neighbor offsets for odd vs even rows).

💎 Why We Pick Diamond 2:1

From here on, "isometric" in this track means diamond 2:1. We pick it because:

  • Industry lineage — AoE2, RA2, Diablo II, StarCraft all use it. If you learn it, you can read any classic RTS codebase.
  • Pixel-clean math — tile dimensions are integers (typically 64×32) and slopes are exactly ±0.5. No anti-aliasing artifacts.
  • Simple coordinate transforms — the cart↔iso math is two lines per direction. We'll derive it next.

🧮 Cartesian ↔ Isometric Coordinate Math

Two coordinate systems live side by side: the cartesian tile grid (the logical map — (0,0), (1,0), (2,0), …) and the screen position in pixels. Converting between them is the entire job of the iso projection.

Cartesian → Isometric (drawing)

The formula

screenX = originX + (cartX − cartY) × (tileW / 2)
screenY = originY + (cartX + cartY) × (tileH / 2)

Read it geometrically: moving +1 in cartX takes you right and down (along the front-right axis of the diamond). Moving +1 in cartY takes you left and down. Together they tile the screen with diamonds.

1 Implement cartToIso — the drawing direction:
📝 Click to see: cartToIso 
// Convert tile-grid coordinates to screen pixels.
// originX / originY is where tile (0, 0) lives on screen.
function cartToIso(cx, cy, tileW, tileH, originX = 0, originY = 0) {
    return {
        x: originX + (cx - cy) * (tileW / 2),
        y: originY + (cx + cy) * (tileH / 2)
    };
}

// Example: with a 64x32 tile, tile (3, 1) lands at...
const p = cartToIso(3, 1, 64, 32, 400, 50);
// p = { x: 464, y: 114 }

Isometric → Cartesian (picking)

Going the other way — given a screen pixel (say, a mouse click), what tile is under it? — is just the inverse of the forward transform. Because the projection is a linear 2D transform, its inverse is also linear.

Deriving the inverse

Starting from the two equations:

dx = (cx − cy) × (tileW / 2)
dy = (cx + cy) × (tileH / 2)

Solve for cx and cy:

cartX = dx / tileW + dy / tileH
cartY = dy / tileH − dx / tileW

(where dx = screenX − originX, dy = screenY − originY)

2 Implement isoToCart — the picking direction:
📝 Click to see: isoToCart 
// Inverse of cartToIso. Returns fractional tile coordinates;
// floor() them to get the integer tile.
function isoToCart(sx, sy, tileW, tileH, originX = 0, originY = 0) {
    const dx = sx - originX;
    const dy = sy - originY;
    return {
        x: dx / tileW + dy / tileH,
        y: dy / tileH - dx / tileW
    };
}
// Inverse of cartToIso. Returns fractional tile coordinates;
// floor() them to get the integer tile.
function isoToCart(
    sx: number,
    sy: number,
    tileW: number,
    tileH: number,
    originX: number = 0,
    originY: number = 0
): { x: number; y: number } {
    const dx: number = sx - originX;
    const dy: number = sy - originY;
    return {
        x: dx / tileW + dy / tileH,
        y: dy / tileH - dx / tileW
    };
}

🖌️ Drawing Tiles & Grids

One tile is four points and a polygon. A grid is a nested loop calling drawIsoTile with offsets.

A single diamond

📝 Click to see: drawIsoTile 
// Draws a diamond whose *top* vertex is at (sx, sy).
function drawIsoTile(ctx, sx, sy, tileW, tileH, fillStyle, strokeStyle) {
    const halfW = tileW / 2;
    const halfH = tileH / 2;
    ctx.beginPath();
    ctx.moveTo(sx, sy);                  // top
    ctx.lineTo(sx + halfW, sy + halfH);  // right
    ctx.lineTo(sx, sy + tileH);          // bottom
    ctx.lineTo(sx - halfW, sy + halfH);  // left
    ctx.closePath();
    if (fillStyle) { ctx.fillStyle = fillStyle; ctx.fill(); }
    if (strokeStyle) { ctx.strokeStyle = strokeStyle; ctx.stroke(); }
}

function drawGridIso(ctx, mapW, mapH, tileW, tileH, originX, originY) {
    // Iterate in row-major order. Each (cx, cy) becomes a screen position
    // via cartToIso, and we draw a diamond there.
    for (let cy = 0; cy < mapH; cy++) {
        for (let cx = 0; cx < mapW; cx++) {
            const p = cartToIso(cx, cy, tileW, tileH, originX, originY);
            drawIsoTile(ctx, p.x, p.y, tileW, tileH, '#2a3550', '#4fc3f7');
        }
    }
}

🎯 Picking a Tile From a Mouse Click

In a top-down game, mouse-to-tile is trivial: floor(mouseX / tileW). In an iso game, the diagonal grid means you can't just divide. You run the screen position back through isoToCart, then floor the result.

One subtle gotcha

Because cartToIso places tile (0,0) at its top vertex, a literal inverse picks the tile above where the mouse is. The fix: shift the mouse Y down by half a tile before inverting. This way the mouse sits in the center of the diamond instead of at its top.

📝 Click to see: pickTileFromMouse (with the Y-offset fix) 
function pickTileFromMouse(mouseX, mouseY, originX, originY, tileW, tileH) {
    // Offset Y by tileH/2 so we measure from the diamond's CENTER, not its top vertex.
    const cart = isoToCart(mouseX, mouseY - tileH / 2, tileW, tileH, originX, originY);
    return { x: Math.floor(cart.x), y: Math.floor(cart.y) };
}

// Inside a mousemove handler:
canvas.addEventListener('mousemove', (e) => {
    const rect = canvas.getBoundingClientRect();
    const tile = pickTileFromMouse(
        e.clientX - rect.left,
        e.clientY - rect.top,
        originX, originY, tileW, tileH
    );
    console.log('Hovering tile', tile.x, tile.y);
});

    mouseX: number,
    mouseY: number,

    originY: number,

    tileH: number

🎨 Mini-Project: Paint-the-Grid Editor

Time to put it all together. We have:

  • cartToIso for drawing tiles at the right screen position
  • drawIsoTile for the diamond shape
  • pickTileFromMouse for translating clicks back to tile coords

That's enough to build a tiny tile editor. Click a paint color, then click (or drag) on the grid to paint cells. This is also the seed of the map editor we'll grow in the Advanced tier.

📝 Click to see: the full paint-the-grid logic 
// Map data: a 2D array storing the paint color per tile (or null for "empty").
const MAP_W = 10, MAP_H = 10;
const TILE_W = 64, TILE_H = 32;
const ORIGIN_X = canvas.width / 2;
const ORIGIN_Y = 40;

const tiles = Array.from({ length: MAP_H }, () => Array(MAP_W).fill(null));
let currentColor = '#66bb6a';
let isPainting = false;

function paintAt(mouseX, mouseY) {
    const t = pickTileFromMouse(mouseX, mouseY, ORIGIN_X, ORIGIN_Y, TILE_W, TILE_H, MAP_W, MAP_H);
    if (t) tiles[t.y][t.x] = currentColor;
}

canvas.addEventListener('mousedown', (e) => {
    isPainting = true;
    const r = canvas.getBoundingClientRect();
    paintAt(e.clientX - r.left, e.clientY - r.top);
});
canvas.addEventListener('mousemove', (e) => {
    if (!isPainting) return;
    const r = canvas.getBoundingClientRect();
    paintAt(e.clientX - r.left, e.clientY - r.top);
});
canvas.addEventListener('mouseup',   () => { isPainting = false; });
canvas.addEventListener('mouseleave', () => { isPainting = false; });

function render() {
    clearCanvas(ctx, canvas.width, canvas.height);
    // Painter's algorithm: back-to-front so closer tiles overlap farther ones.
    for (let cy = 0; cy < MAP_H; cy++) {
        for (let cx = 0; cx < MAP_W; cx++) {
            const p = cartToIso(cx, cy, TILE_W, TILE_H, ORIGIN_X, ORIGIN_Y);
            const fill = tiles[cy][cx] || '#1a233a';
            drawIsoTile(ctx, p.x, p.y, TILE_W, TILE_H, fill, '#4fc3f7');
        }
    }
}
