🌊 Flow-Field Pathfinding Visualizer

The Advanced tier introduced flow fields as a black box: "BFS from the goal, produce a vector per cell." Here we open the box. Press Step to advance the BFS one frontier expansion at a time. The cost grid fills outward, the flow vectors materialize cell by cell.

// Each call to step() expands the BFS by one frontier level.
function stepBFS(cost, queue, W, H, blocked) {
    if (queue.length === 0) return false;
    const c = queue.shift();
    const here = cost[c.y * W + c.x];
    for (const [dx, dy] of [[1,0],[-1,0],[0,1],[0,-1]]) {
        const nx = c.x + dx, ny = c.y + dy;
        if (nx < 0 || nx >= W || ny < 0 || ny >= H) continue;
        if (blocked[ny * W + nx]) continue;
        if (cost[ny * W + nx] <= here + 1) continue;
        cost[ny * W + nx] = here + 1;
        queue.push({ x: nx, y: ny });
    }
    return queue.length > 0;
}
function stepBFS(
    cost: Float32Array,
    queue: { x: number; y: number }[],
    W: number, H: number,
    blocked: Uint8Array
): boolean {
    if (queue.length === 0) return false;
    const c = queue.shift()!;
    const here: number = cost[c.y * W + c.x];
    for (const [dx, dy] of [[1,0],[-1,0],[0,1],[0,-1]]) {
        const nx: number = c.x + dx, ny: number = c.y + dy;
        if (nx < 0 || nx >= W || ny < 0 || ny >= H) continue;
        if (blocked[ny * W + nx]) continue;
        if (cost[ny * W + nx] <= here + 1) continue;
        cost[ny * W + nx] = here + 1;
        queue.push({ x: nx, y: ny });
    }
    return queue.length > 0;
}

🌡️ Influence Maps for AI Strategy

AI opponents decide where to attack, defend, and expand by reading influence maps. Each unit projects a Gaussian-decayed value onto every cell — positive for friendly, negative for enemy. The summed map reveals:

  • Strongholds — high positive or negative cells.
  • Frontlines — cells where the absolute total is near zero (forces cancel).
  • Safe zones — far from any enemy.
📝 Click to see: Gaussian influence sum 
function computeInfluence(units, W, H, sigma) {
    const map = new Float32Array(W * H);
    const s2 = sigma * sigma;
    for (const u of units) {
        const sign = u.team === 'friend' ? 1 : -1;
        for (let y = 0; y < H; y++) {
            for (let x = 0; x < W; x++) {
                const dx = x - u.x, dy = y - u.y;
                const w = Math.exp(-(dx * dx + dy * dy) / (2 * s2));
                map[y * W + x] += sign * w;
            }
        }
    }
    return map;
}
interface InfluenceUnit { team: 'friend' | 'enemy'; x: number; y: number; }

function computeInfluence(units: InfluenceUnit[], W: number, H: number, sigma: number): Float32Array {
    const map: Float32Array = new Float32Array(W * H);
    const s2: number = sigma * sigma;
    for (const u of units) {
        const sign: number = u.team === 'friend' ? 1 : -1;
        for (let y: number = 0; y < H; y++) {
            for (let x: number = 0; x < W; x++) {
                const dx: number = x - u.x, dy: number = y - u.y;
                const w: number = Math.exp(-(dx * dx + dy * dy) / (2 * s2));
                map[y * W + x] += sign * w;
            }
        }
    }
    return map;
}

👥 Crowd Movement: Boids vs Continuum Crowds

When 200 units want to reach the same goal, you have a choice. Boids (Reynolds, 1987) treat the crowd as a set of agents applying three forces — separation, alignment, cohesion — toward the goal. Continuum crowds (Treuille / Cooper / Popović, 2006) treat the crowd as a fluid: a precomputed flow field guides motion, and local density slows agents in congested areas.

The qualitative difference

Boids look organic — agents jostle, line up, and occasionally clog at chokepoints. Continuum looks fluid — the swarm flows smoothly around obstacles. Same goal, very different visuals. Both are correct; the right choice depends on the game.

// Boids: each agent computes 3 steering forces vs nearby neighbors,
// adds a goal-seek, and integrates velocity.
function updateBoid(agent, neighbors, goal, dt) {
    let sx = 0, sy = 0, ax = 0, ay = 0, cx = 0, cy = 0, n = 0;
    for (const other of neighbors) {
        if (other === agent) continue;
        const dx = agent.x - other.x, dy = agent.y - other.y;
        const d2 = dx * dx + dy * dy;
        if (d2 > 25 || d2 === 0) continue;       // 5px radius
        const w = 1 / d2;
        sx += dx * w; sy += dy * w;               // separation
        ax += other.vx; ay += other.vy;           // alignment
        cx += other.x;  cy += other.y;            // cohesion
        n++;
    }
    let dvx = 0, dvy = 0;
    if (n > 0) {
        dvx += sx * 0.5;
        dvy += sy * 0.5;
        dvx += (ax / n - agent.vx) * 0.05;
        dvy += (ay / n - agent.vy) * 0.05;
        dvx += (cx / n - agent.x) * 0.0008;
        dvy += (cy / n - agent.y) * 0.0008;
    }
    // Goal seeking
    const gx = goal.x - agent.x, gy = goal.y - agent.y;
    const gd = Math.hypot(gx, gy) || 1;
    dvx += (gx / gd) * 0.3;
    dvy += (gy / gd) * 0.3;

    agent.vx += dvx * dt;
    agent.vy += dvy * dt;
    // Speed cap
    const vMag = Math.hypot(agent.vx, agent.vy);
    if (vMag > 40) { agent.vx *= 40 / vMag; agent.vy *= 40 / vMag; }
    agent.x += agent.vx * dt;
    agent.y += agent.vy * dt;
}

// Continuum: read flow field, scale speed by local density.
function updateContinuum(agent, flowVec, density, dt) {
    if (!flowVec) return;
    const slow = Math.max(0.3, 1 - density * 0.1);
    agent.vx = flowVec.dx * 30 * slow;
    agent.vy = flowVec.dy * 30 * slow;
    agent.x += agent.vx * dt;
    agent.y += agent.vy * dt;
}

🗺️ Procedural Map Generation

Hand-crafted maps don't scale. A two-pass procedural recipe — noise for elevation, biome rules for terrain assignment — generates an infinite supply of usable maps. Click Regenerate to reroll the seed.

The pipeline

  1. Sample value-noise at every tile to get an elevation grid in [0, 1].
  2. Apply biome thresholds: water below sea-level, sand in the shore band, grass in midlands, stone on peaks.
  3. (Optional) Voronoi region partitioning so each biome forms contiguous blobs.
📝 Click to see: value noise + biome assignment 
// Tiny value noise: hash integer points, bilinear smoothstep between.
function makeNoise(seed) {
    function hash(x, y) {
        let h = (x * 374761393 + y * 668265263 + seed * 982451653) | 0;
        h = (h ^ (h >> 13)) * 1274126177;
        return ((h ^ (h >> 16)) >>> 0) / 0xffffffff;
    }
    function smooth(t) { return t * t * (3 - 2 * t); }
    return function(x, y) {
        const ix = Math.floor(x), iy = Math.floor(y);
        const fx = x - ix, fy = y - iy;
        const a = hash(ix, iy),     b = hash(ix + 1, iy);
        const c = hash(ix, iy + 1), d = hash(ix + 1, iy + 1);
        const sx = smooth(fx), sy = smooth(fy);
        return (a * (1 - sx) + b * sx) * (1 - sy) +
               (c * (1 - sx) + d * sx) * sy;
    };
}

function biomeFor(h) {
    if (h < 0.30) return 'water';
    if (h < 0.38) return 'sand';
    if (h < 0.75) return 'grass';
    return 'stone';
}

⚔️ Combat Damage Model Playground

AoE2's combat is rock-paper-scissors: each attack type has a multiplier against each armor type. Tuning these numbers is one of the most consequential balance levers in a strategy game — adjust the matrix and watch the DPS bars react.

The model

effectiveDPS(attacker, defender) = baseDamage × matrix[attacker.type][defender.armor] × attacksPerSecond

Three attack types — pierce, blunt, magic — versus three armor types — light, medium, heavy. Sliders set the multipliers; bars show the resulting DPS per matchup.
