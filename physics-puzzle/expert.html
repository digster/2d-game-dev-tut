🧨 Destruction is built on numbers you already have

Breaking things sounds like a whole new system, but it falls almost entirely out of the rigid solver from the last tier. The solver already computes, at every contact, exactly how hard two bodies hit — the impulse. We add one line to report that (body.impact, the total normal impulse a body received this step) and suddenly we can answer "did that hit break it?" and "which block is bearing the most load?".

  • Impact thresholds — break a body when impact crosses a limit.
  • Voronoi pre-fracture — a shape's shards are just the polygon clipped by the bisectors between random seed points; precompute them, reveal on impact.
  • Pooling — fragments spawn fast, so cap and recycle them.
  • Structural stress — impact on a resting block is its support load; colour by it and the keystone reveals itself.
  • Juice — dust, a trauma-based screen shake, and a beat of hitstop.

📊 Impact thresholds — how hard was that hit?

Fling the heavy ball at the wall. Each block sums the impulse it receives into impact; any block whose impact beats the threshold this frame shatters (and puffs dust). Low threshold → a house of cards; high threshold → a fortress. Same wall, same ball — the only variable is how much punishment a block tolerates.

📝 Click to see: break = read the impulse the solver already summed 
// engine/rigid.js resets body.impact = 0 each step, then pzSolveManifold
// accumulates every normal impulse it applies:  a.impact += jn;  b.impact += jn;
// So after a step, impact ≈ how hard the body was hit.

pzStepWorld(bodies, [], 0, GRAVITY, dt);
for (const b of bodies) {
  if (b.tag === 'block' && b.impact > threshold) {
    spawnDust(b.pos);          // a puff where it broke
    remove(b);                 // (or replace it with fragments — next demo)
  }
}
// in pzSolveManifold, per contact, after clamping the normal impulse jn ≥ 0:
a.impact += jn;
b.impact += jn;
// the threshold test is then trivial — and "how hard" is exactly the impulse
// that the collision response needed anyway. No extra physics.

🪟 Voronoi pre-fracture — latent shards

A shattering pane looks expensive, but the shards are just precomputed geometry. Scatter random seed points across the shape; each one's shard is the shape clipped by the perpendicular bisector between it and every other seed — that's a Voronoi diagram. The faint lines are the latent cracks, computed before any impact. Click the pane to reveal them as real falling fragments.

click the pane to shatter it…
📝 Click to see: a shard is the shape clipped by bisectors 
// For each seed i, the cell is the polygon clipped by the half-plane closer to
// seed i than to every other seed j (the bisector between i and j):
let cell = polygonVerts.slice();
for (const j of otherSeeds) {
  const mx = (s_i.x + s_j.x) / 2, my = (s_i.y + s_j.y) / 2;   // midpoint
  const nx = s_j.x - s_i.x,       ny = s_j.y - s_i.y;          // points AWAY from i
  cell = clipPolyHalfPlane(cell, nx, ny, nx*mx + ny*my);       // keep i's side
}
// cell is now a convex shard → make it a rigid body at its centroid.
function clipPolyHalfPlane(poly: P[], nx: number, ny: number, o: number): P[] {
  const out: P[] = [];
  for (let i = 0; i < poly.length; i++) {
    const a = poly[i], b = poly[(i + 1) % poly.length];
    const da = nx*a.x + ny*a.y - o, db = nx*b.x + ny*b.y - o;
    if (da <= 0) out.push(a);                                  // keep inside vertices
    if (da*db < 0) out.push(lerpEdge(a, b, da / (da - db)));    // + crossing point
  }
  return out;
}

♻️ Debris & pooling — keep it bounded

Every shatter dumps a handful of new bodies into the world; spam it and the body count (and the garbage collector) runs away. So debris lives on a budget: a fixed cap, with the oldest pieces retired as new ones arrive. Click to drop-and-shatter blocks and watch the active count hold steady at the cap no matter how fast you click.

debris cap 70
click to shatter a block…
Why a budget beats "just spawn them all": physics cost is roughly O(n²) in body count without a broad-phase (the Simulations tier's next topic), and unbounded allocation stutters as the GC kicks in. A pool turns "how many can we afford?" into a single tunable number.

🏛️ Structural stress — knock out the keystone

Which block is holding everything up? You already know: a resting block's impact is the impulse the solver applies to support the load above it — so the bottom of a column reads higher than the top. We colour blocks green → red by that load. Click a red, load-bearing block to remove it, and watch the structure above lose its support and collapse.

click a load-bearing block…
📝 Click to see: load is the contact impulse (no new maths) 
// after a step, body.impact already holds the total support impulse. Normalise
// to the busiest block and map to a colour:
let maxImp = 1;
for (const b of blocks) maxImp = Math.max(maxImp, b.impact);
for (const b of blocks) b.color = loadColor(b.impact / maxImp);   // green → red

// "keystone": the click removes whichever block contains the pointer —
// remove a red one and everything it was holding loses support.
// A real stress solver would propagate load through the contact graph; here the
// impulse the constraint solver ALREADY computes is a faithful proxy — the force
// needed to keep a block from accelerating under everything stacked on it.
const t = clamp(b.impact / maxImp, 0, 1);
b.color = loadColor(t);

✨ Juice — dust, screen shake, hitstop

The physics of a big hit is correct without any of this — but it doesn't feel like much. Three cheap tricks fix that: a burst of dust at the contact, a trauma-based screen shake (offset ∝ trauma², so it ramps and fades smoothly), and a few frames of hitstop (freeze the whole sim — the beat that makes an impact land). Toggle each and fling a ball to feel the difference.

drag to fling a ball at the stack…
📝 Click to see: trauma shake + hitstop 
// on a big impact: bump trauma, maybe freeze a few frames
if (b.impact > BIG)        trauma = Math.min(1, trauma + b.impact / 1400);
if (b.impact > BIG * 2.2)  hitstop = Math.max(hitstop, 3);

function update(dt) {
  if (hitstop > 0) { hitstop--; return; }      // FROZEN — the punchy beat
  stepWorld(...);
  trauma = Math.max(0, trauma - dt * 1.6);      // decay
}
function render() {
  const s = trauma * trauma * 16;               // squared → smooth ramp/fade
  ctx.translate(rand(-s, s), rand(-s, s));      // shake the camera
}
// trauma (0..1) is the standard screen-shake model (Squirrel Eiserloh): shake
// amplitude = trauma², so small hits barely wobble and big hits slam, and the
// fade is automatic as trauma decays linearly each frame.
let trauma = 0, hitstop = 0;
