🧩 From a point to a body

Everything so far has been a point — a circle or a rope bead with a position and a velocity. A real object also has orientation: it can spin. The moment you let a box rotate, three new things appear, and this tier is those three things:

  • A second integrator — alongside pos += vel·dt you now run angle += angularVel·dt, and torque changes the angular velocity.
  • Polygon collision — two circles overlap if their centres are close, but two boxes need the Separating Axis Theorem (SAT) to find the normal and the contact points.
  • Impulse at a point — a hit off the centre of mass both pushes and spins the body. That's r × J, and it's what makes stacking, toppling and friction feel real.
💡 This is the same impulse you met in the Beginner tier — generalised. Circle-vs-circle was the special case where every hit is dead-centre (no spin). Here a hit can land anywhere on a convex shape, so we add the rotational term. The Beginner code isn't thrown away; it's the simplest point on this curve.

🧰 The rigid-body engine (all inline in advanced-demos.js)

  • PZRigidBody — a convex polygon with mass, moment of inertia, angle, angular velocity.
  • pzPolyVsPoly — SAT + reference/incident-face clipping → a contact manifold.
  • pzSolveManifold — sequential impulses: normal (+restitution) and Coulomb friction, with r × J.
  • PZJoint — a pivot constraint between two bodies (or a body and a fixed point), with a break threshold.

🌀 The second integrator — angle & torque

Zero gravity, one box. Drag from a point on it and release: the impulse is applied at that point. Pull from the centre and it only slides; pull from a corner and the same impulse also spins it. That difference — the offset from the centre of mass — is the torque, and it feeds a whole second integrator for the angle.

drag from a corner to spin it…
📝 Click to see: position AND angle, and an impulse at a point 
// integrate BOTH: linear and angular
pos.x += vel.x * dt;  pos.y += vel.y * dt;
angle += angularVel * dt;

// an impulse (imx,imy) at world offset (rx,ry) from the centre of mass:
applyImpulse(imx, imy, rx, ry) {
  vel.x += imx * invMass;
  vel.y += imy * invMass;
  angularVel += invInertia * (rx * imy - ry * imx);   // r × J = torque
}
applyImpulse(imx: number, imy: number, rx: number, ry: number): void {
  if (this.isStatic) return;
  this.vel.x += imx * this.invMass;
  this.vel.y += imy * this.invMass;
  this.angularVel += this.invInertia * (rx * imy - ry * imx); // 2D cross
}
Moment of inertia is mass's rotational cousin — how hard the shape is to spin about its centre. A long thin plank resists spin more than a compact box of the same mass. We compute it once from the polygon's vertices and store invInertia = 1/I, exactly like invMass.

📐 SAT — the axis of least overlap

Two convex shapes are apart if you can slip a line between them — and that line is perpendicular to one of their edges. So we test every edge-normal of both shapes: if any axis shows a gap, they're separated. If none do, they overlap, and the axis with the least overlap is the collision normal (the shortest push to separate them). Drag the cyan box into the pentagon to see the normal and contact points SAT finds.

📝 Click to see: the least-penetration probe 
// For each face of A, find B's deepest point along that face's outward normal.
// The face where B pokes in the LEAST is the best separating candidate.
function pzLeastPenetration(A, B) {
  let best = -Infinity, index = 0;
  A.faces.forEach((face, i) => {
    const n = face.normal, v = face.vertex;
    let support = minimize(B.verts, bv => dot(n, bv));   // B's deepest vertex
    const pen = dot(n, sub(support, v));
    if (pen > best) { best = pen; index = i; }            // least negative wins
  });
  return { best, index };       // best >= 0  ⇒  a gap exists, no collision
}
// A full manifold then CLIPS the incident face against the reference face's
// side planes to get up to TWO contact points — what you need for a box to rest
// flat instead of teetering on one corner.
function pzPolyVsPoly(A: PZRigidBody, B: PZRigidBody): Manifold | null {
  const pa = pzLeastPenetration(A, B); if (pa.best >= 0) return null;
  const pb = pzLeastPenetration(B, A); if (pb.best >= 0) return null;
  // ...pick reference face, clip incident face, keep penetrating points...
}

💥 Impulse with rotation — r × J

Now we respond to those collisions. Click to drop boxes onto the floor: when a box lands on a corner, the contact is off its centre of mass, so the impulse both stops it and spins it — it tips, rolls, and settles, exactly like a real block. Crank restitution up for a bouncy rubber feel, down for dead thuds.

The relative velocity is measured at the contact point, not at the centre — it includes the spin (v + ω × r). That single change is what separates a rotating rigid-body solver from the Beginner tier's point solver.

🧱 Friction & correction — boxes that rest & stack

Two finishing touches turn "bouncing boxes" into "a stable tower". Friction — a sideways impulse, clamped by μ times the normal impulse (the Coulomb cone) — stops boxes from sliding off each other. Positional correction — a gentle push that undoes the tiny overlap the solver leaves behind — stops the stack slowly sinking into itself. Drag from the left to fling a ball at the tower; drop friction to watch it slither apart.

📝 Click to see: the friction clamp (Coulomb's cone) 
// after the NORMAL impulse jn, apply friction along the tangent:
let jt = -dot(relVelAtContact, tangent) / invMassTangent;
jt = clamp(jt, -mu * jn, mu * jn);     // friction can't exceed μ·(normal force)
applyFrictionImpulse(tangent * jt);

// positional correction (Baumgarte) folds INTO the normal impulse as a bias:
const bias = (BETA / dt) * Math.max(penetration - SLOP, 0);
let jn = (-(1 + e) * velAlongNormal + bias) / invMassNormal;
jn = Math.max(jn, 0);                  // contacts can only PUSH, never pull
// μ (combined) = sqrt(a.friction * b.friction); e = min(a.e, b.e).
// SLOP lets a sliver of overlap remain so resting boxes don't jitter;
// BETA controls how fast the remaining penetration is pushed out.
const mu = Math.sqrt(a.friction * b.friction);
let jt = -dot(rv, tangent) / invT;
jt = clamp(jt, -mu * jn, mu * jn);

🔗 Joints — pin two bodies, and break the pin

A joint holds a point on one body to a point on another (or to a fixed anchor) — a hinge. Chain a few planks with pivot joints and you get a rigid rope or a bridge. Give each joint a break threshold and you get destructible contraptions: yank the chain hard, or lower the break force, and a joint snaps. Drag the chain to swing it.

// Keep anchorA (on body a) coincident with anchorB (on body b). Solve for the
// impulse P that drives their relative velocity (plus a position-error bias) to
// zero, through the 2×2 effective-mass matrix K:
const C  = sub(pb, pa);                       // position error (want 0)
const rv = relVelAtAnchor(b, rb) - relVelAtAnchor(a, ra);
const P  = solve2x2(K, neg(add(rv, scale(beta/dt, C))));
a.applyImpulse(neg(P), ra);
b.applyImpulse(P, rb);
if (length(P) > breakImpulse) this.broken = true;   // snap!
// K is the standard point-constraint mass matrix (invMass + r⊥·invI·r⊥ terms):
const k11 = ma + mb + ia*ra.y*ra.y + ib*rb.y*rb.y;
const k12 = -ia*ra.x*ra.y - ib*rb.x*rb.y;
const k22 = ma + mb + ia*ra.x*ra.x + ib*rb.x*rb.x;
