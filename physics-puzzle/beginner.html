🧱 What a physics puzzle really is

Strip the slingshot and the smug green pigs away, and an Angry-Birds-style game is three pieces working together, sixty times a second:

  • A world that, every fixed step, pulls each body down by gravity and moves it by its velocity. That's the integrator — and it's all our PZWorld does.
  • Bodies — circles, for now — each with a position, a velocity, a mass and a bounciness.
  • Collision — the rules that stop a ball at a wall and let two balls shove each other. This is the lesson, so we write it ourselves, on top of the world.

That's the whole skeleton. Ropes, rigid bodies, fracture, fluids — everything later in this track is more bodies and cleverer collision on this same world. So we build the world first, then add collision one case at a time, each demo introducing exactly one idea.

💡 Why a fixed timestep? If you moved bodies by velocity × (time since last frame), a launched ball would fly a different arc on a 30 Hz laptop than a 240 Hz monitor, and a frame hitch could glitch it through a wall. Instead we advance physics in fixed 1/60-second steps and render whenever we can. The engine's pzLoop already does this — we just hand it an update(dt) and a render(). It's also why the trajectory preview later can be trusted: the preview is the very same simulation, run forward.

🧰 The engine, loaded on every page in this track

  • PZWorld + PZBody — the integrator and a body (a circle with position, velocity, mass, restitution).
  • pzLoop — the fixed-timestep loop. pzInstallPointer — drag input (where a pull started, where it is now, where it was released).
  • PZ + pzDrawBody / pzDrawArena / pzDrawDots — the shared palette and asset-free drawing.

Open the track index and scroll to the Scaffold Self-Check to see the integrator proven — a ball dropped under gravity rests exactly on the floor.

⚠️ One trap worth knowing up front: our Vector2D (from the shared utils) mutates in place — v.add(...), v.multiply(...) change v itself. Only v.subtract(...) and v.copy() return a new vector. So we always .copy() a shared vector (like gravity) before doing maths on it, or we'd corrupt it for every body at once.

🌍 The World & gravity

The whole world is two lines per body: gravity adds to velocity, velocity moves the body. Watch the ball fall, speed up, and (for now) sail straight off the bottom — there's no collision yet, so it just re-drops from the top. The fading trail shows it accelerating: the gaps between ghosts grow each step. Drag the gravity slider.

// PZWorld.step(dt), once per fixed 1/60 s. For each dynamic body:
for (const b of bodies) {
  b.vel.add(gravity.copy().multiply(dt)); // 1) velocity += g·dt  (COPY g!)
  b.vel.multiply(damping);                 //    bleed a sliver of speed
  b.pos.add(b.vel.copy().multiply(dt));    // 2) position += v·dt  (COPY vel!)
}

// In the demo we just feed that to the loop:
const world = new PZWorld({ gravity: 1400 });
world.add(new PZBody(x, y, 14));
pzLoop((dt) => world.step(dt), render).start();
// semi-implicit (symplectic) Euler — velocity first, THEN position.
for (const b of bodies) {
  b.vel.add(gravity.copy().multiply(dt)); // COPY g — Vector2D.multiply mutates!
  b.vel.multiply(damping);
  b.pos.add(b.vel.copy().multiply(dt));   // COPY vel — same reason
}

const world = new PZWorld({ gravity: 1400 });
world.add(new PZBody(x, y, 14));
pzLoop((dt: number) => world.step(dt), render).start();
Why "semi-implicit" Euler? We update velocity before using it to move. That one ordering choice makes gravity far more stable than the naïve version (move first, then accelerate), and it's what almost every 2D game ships. The damping is an honest cheat — a stand-in for air drag so things eventually settle; real friction arrives in the Advanced tier.

🧊 Walls, ground & restitution — the bounce

Now we add the arena. When the ball crosses a wall we push it back inside and reflect its velocity along the wall's normal, scaled by restitution — the bounciness, 0 (dead clay) to 1 (a perfect superball). Click anywhere to toss a ball; drag restitution and watch the bounces grow or die.

📝 Click to see: a bounce is reflect-and-attenuate along the normal 
// Resolve a circle against the arena box. Axis-aligned walls make this
// simple: the normal is always horizontal or vertical, so "reflect" is
// just "flip that velocity component and scale by restitution e".
function pzResolveStatic(b, arena) {
  const e = b.restitution;
  if (b.left   < arena.x)            { b.pos.x = arena.x + b.radius;          if (b.vel.x < 0) b.vel.x *= -e; }
  if (b.right  > arena.x + arena.w)  { b.pos.x = arena.x + arena.w - b.radius; if (b.vel.x > 0) b.vel.x *= -e; }
  if (b.top    < arena.y)            { b.pos.y = arena.y + b.radius;          if (b.vel.y < 0) b.vel.y *= -e; }
  if (b.bottom > arena.y + arena.h)  { b.pos.y = arena.y + arena.h - b.radius; if (b.vel.y > 0) b.vel.y *= -e; }
}

function update(dt) { world.step(dt); for (const b of world.bodies) pzResolveStatic(b, arena); }
function pzResolveStatic(b: PZBody, arena: Rect): void {
  const e = b.restitution;
  if (b.left   < arena.x)           { b.pos.x = arena.x + b.radius;           if (b.vel.x < 0) b.vel.x *= -e; }
  if (b.right  > arena.x + arena.w) { b.pos.x = arena.x + arena.w - b.radius; if (b.vel.x > 0) b.vel.x *= -e; }
  if (b.top    < arena.y)           { b.pos.y = arena.y + b.radius;           if (b.vel.y < 0) b.vel.y *= -e; }
  if (b.bottom > arena.y + arena.h) { b.pos.y = arena.y + arena.h - b.radius; if (b.vel.y > 0) b.vel.y *= -e; }
}
Restitution is a multiplier, not a force. At e = 1 the ball keeps all its speed and bounces forever; at e = 0 it sticks the moment it lands. Real games live around 0.2–0.6. The walls here are axis-aligned, so the normal is always ±x or ±y — the Advanced tier handles bounces off tilted surfaces with the full reflection vector.

🪃 The slingshot — drag to aim, release to launch

The signature move. Press on the ball and drag backwards — the further you pull, the harder it launches (we clamp the pull so it can't go absurd). Releasing applies an impulse: an instant change in velocity, v += J / m, pointed from the pull back toward the anchor. It's a one-shot kick, not a held force.

const pointer = pzInstallPointer(canvas);

function update(dt) {
  if (pointer.justReleased) {
    // pull vector = anchor − release point; launch the OPPOSITE way.
    const pull = anchor.subtract(pointer.releaseEnd); // subtract → NEW vector
    const launchVel = pull.copy().multiply(POWER).limit(MAX_SPEED);
    ball.vel.set(launchVel.x, launchVel.y);           // impulse on a fresh ball
    flying = true;
  }
  if (flying) { world.step(dt); pzResolveStatic(ball, arena); }
  pointer.endFrame();
}
const pointer = pzInstallPointer(canvas);

function update(dt: number): void {
  if (pointer.justReleased) {
    const pull = anchor.subtract(pointer.releaseEnd!);   // NEW vector
    const launchVel = pull.copy().multiply(POWER).limit(MAX_SPEED);
    ball.vel.set(launchVel.x, launchVel.y);
    flying = true;
  }
  if (flying) { world.step(dt); pzResolveStatic(ball, arena); }
  pointer.endFrame();
}
Impulse vs force. Gravity is a force — it nudges velocity a little every step. A launch is an impulse — it sets velocity all at once. Same units in the end (both change velocity), but one is spread over time and the other is instantaneous. We clamp launch speed with limit() so a fast shot still steps less than its own radius and can't tunnel through a thin wall (high-speed tunneling is a Simulations-tier topic).

🔮 Trajectory prediction — running the sim ahead

Angry Birds shows you a dotted arc before you fire. There's no secret formula behind it — it's the exact same physics, simulated forward on a throwaway copy of the ball. Drag to aim and the dots trace where it would go. Because both use world.step, the preview and the real flight agree to the pixel.

// Same integrator, run on a temporary world that holds one cloned ball.
function predict(launchVel, steps) {
  const ghostWorld = new PZWorld({ gravity: world.gravity.y });
  const ghost = new PZBody(anchor.x, anchor.y, ball.radius, {
    vx: launchVel.x, vy: launchVel.y, restitution: ball.restitution,
  });
  ghostWorld.add(ghost);
  const points = [];
  for (let i = 0; i < steps; i++) {
    ghostWorld.step(1 / 60);
    pzResolveStatic(ghost, arena);          // bounce the preview too
    points.push({ x: ghost.pos.x, y: ghost.pos.y });
  }
  return points;                            // → pzDrawDots(ctx, points)
}
function predict(launchVel: Vector2D, steps: number): Point[] {
  const ghostWorld = new PZWorld({ gravity: world.gravity.y });
  const ghost = new PZBody(anchor.x, anchor.y, ball.radius, {
    vx: launchVel.x, vy: launchVel.y, restitution: ball.restitution,
  });
  ghostWorld.add(ghost);
  const points: Point[] = [];
  for (let i = 0; i < steps; i++) {
    ghostWorld.step(1 / 60);
    pzResolveStatic(ghost, arena);
    points.push({ x: ghost.pos.x, y: ghost.pos.y });
  }
  return points;
}
Determinism is the whole trick. A fixed timestep means "the same inputs produce the same motion, every time" — so a forward simulation is a perfect crystal ball. This is the same property that lets the netcode track replay inputs and the Simulations tier prove its physics is frame-rate independent.

🎱 Circle–circle collision — trading momentum

Until now balls ignored each other. Here a launched ball cracks into a stationary rack like a pool break. When two circles overlap we do two things: push them apart so they stop interpenetrating, then exchange momentum along the line between their centres. Heavier balls shove lighter ones, not the reverse. Drag to launch the cyan cue ball.

function pzCollideCircles(a, b) {
  const delta = b.pos.subtract(a.pos);            // NEW vector, a → b
  const dist = delta.length();
  const minDist = a.radius + b.radius;
  if (dist === 0 || dist >= minDist) return;      // not touching
  const n = delta.copy().divide(dist);            // unit normal (copy → safe)
  const invSum = a.invMass + b.invMass;
  if (invSum === 0) return;

  // 1) positional correction — push apart, split by inverse mass
  const overlap = minDist - dist;
  a.pos.add(n.copy().multiply(-overlap * a.invMass / invSum));
  b.pos.add(n.copy().multiply( overlap * b.invMass / invSum));

  // 2) impulse — only if they're approaching
  const relVel = b.vel.subtract(a.vel);           // NEW vector
  const vn = relVel.dot(n);
  if (vn > 0) return;                             // already separating
  const e = Math.min(a.restitution, b.restitution);
  const j = -(1 + e) * vn / invSum;               // impulse magnitude
  const imp = n.copy().multiply(j);
  a.vel.add(imp.copy().multiply(-a.invMass));     // lighter ball moves more
  b.vel.add(imp.copy().multiply( b.invMass));
}
function pzCollideCircles(a: PZBody, b: PZBody): void {
  const delta = b.pos.subtract(a.pos);
  const dist = delta.length();
  const minDist = a.radius + b.radius;
  if (dist === 0 || dist >= minDist) return;
  const n = delta.copy().divide(dist);
  const invSum = a.invMass + b.invMass;
  if (invSum === 0) return;

  const overlap = minDist - dist;
  a.pos.add(n.copy().multiply(-overlap * a.invMass / invSum));
  b.pos.add(n.copy().multiply( overlap * b.invMass / invSum));

  const relVel = b.vel.subtract(a.vel);
  const vn = relVel.dot(n);
  if (vn > 0) return;
  const e = Math.min(a.restitution, b.restitution);
  const j = -(1 + e) * vn / invSum;
  const imp = n.copy().multiply(j);
  a.vel.add(imp.copy().multiply(-a.invMass));
  b.vel.add(imp.copy().multiply( b.invMass));
}
Inverse mass is the unifier. By dividing impulses by mass (really, multiplying by 1/m), a static body — modelled as infinite mass, so 1/m = 0 — simply never moves. The same two lines handle "ball hits ball" and "ball hits wall" with no special cases. This impulse is exactly what the Advanced tier extends with rotation.

🎮 Mini-Project: "Knock-Down"

Everything at once. A slingshot (drag-to-aim + preview), an arena with restitution, and a stack of red pins balanced on a ledge — all colliding through the same impulse code. Knock every pin off the ledge into the gold zone below, with three shots. Heavy shots scatter the stack; a clever bank shot off a wall can clean up a straggler.

drag the ball to aim — 3 shots…
📝 Click to see: the whole Beginner game in one update() 
function update(dt) {
  // 1. fire on release (if shots remain)
  if (pointer.justReleased && shots > 0 && !ball.flying) {
    const pull = anchor.subtract(pointer.releaseEnd);
    const v = pull.copy().multiply(POWER).limit(MAX);
    ball.vel.set(v.x, v.y); ball.flying = true; shots--;
  }

  // 2. one physics step, then ALL collisions: walls + every body pair
  world.step(dt);
  for (const b of world.bodies) pzResolveStatic(b, arena);
  for (let i = 0; i < world.bodies.length; i++)
    for (let j = i + 1; j < world.bodies.length; j++)
      pzCollideCircles(world.bodies[i], world.bodies[j]);

  // 3. win = every pin has fallen below the ledge into the zone
  won = pins.every((p) => p.pos.y > ledgeY + 30);
  pointer.endFrame();
}
function update(dt: number): void {
  if (pointer.justReleased && shots > 0 && !ball.flying) {
    const pull = anchor.subtract(pointer.releaseEnd!);
    const v = pull.copy().multiply(POWER).limit(MAX);
    ball.vel.set(v.x, v.y); ball.flying = true; shots--;
  }
  world.step(dt);
  for (const b of world.bodies) pzResolveStatic(b, arena);

}
