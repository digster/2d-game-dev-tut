# 2026-06-05 (pass 2) — Fix jittery dragged rigid bodies (Physics Puzzle)

## Request
User (on the Advanced "Contraption" mini-project, with screenshot): "The red block acts very
jittery when picked."

## Root cause (one bug, in 3 pzStepWorld-driven draggable demos)
The drag handler set the body's velocity AND teleported its position to the cursor each step:
```js
grab.vel.set((nx - grab.pos.x) / dt, (ny - grab.pos.y) / dt); // velocity = the move
grab.pos.set(nx, ny);                                          // ALSO teleport there
```
Then `pzStepWorld` integrates `pos += vel*dt` (rigid.js:290) — applying the displacement a
**second time** (overshoot to `2·target − pos`). `pzLoop` runs `update(step)` up to twice per
render frame (fixed-step accumulator) while the async `pointermove` updated `pos` only once, so
on the 2nd sub-step `(target − pos)` flips sign → velocity reverses → the body buzzes between
`2·target − pos` and `pos`. That oscillation is the visible jitter.

## Fix — velocity-only drag + speed clamp (one shared engine helper)
Added `pzDragTo(body, tx, ty, dt, maxSpeed = 2500)` to `engine/rigid.js` (after `pzStepWorld`,
exported on `window`): sets `vel = (target − pos)/dt` ONLY (the world step then integrates it
exactly onto the cursor — no double-count, no sub-step sign-flip), and speed-clamps so a laggy
cursor jump can't tunnel a thin wall and explode the solver. Bonus: collisions now stop a dragged
body at walls instead of teleporting it through.

Replaced the buggy 2-line block at the 3 call sites that integrate via `pzStepWorld`:
- `advanced-demos.js` Contraption (kept `angularVel=0; angle=0` axis-lock) + joint-break demo
- `simulations-demos.js` the rigid-body ragdoll/stack demo (line ~425, has `pzStepWorld`)

**Duck was a false positive — reverted.** The fluid "duck" (simulations-demos.js:~485) matched the
grep but is a plain `{pos,vel}` object integrated MANUALLY, NOT by `pzStepWorld`; its grab branch
and the `else` integration are mutually exclusive, so nothing runs `pos += vel*dt` while grabbed.
`pzDragTo` (velocity-only) would leave it stationary — so it correctly sets `pos` directly. No
double-step there, so it never jittered. Left it as-is (added an explanatory comment).
Also untouched: Verlet drags (set `prev` + `pos`, e.g. intermediate-demos.js, simulations:379) and
the Verlet cloth/vector-label math — none are the double-step bug.
