## What shipped — 15 new lessons

Each lesson is the full **5-file transaction**: a `<section>` + TOC entry in
`shaders/{tier}.html`, an IIFE in `shaders/{tier}-demos.js`, a
`DEMO_HTML`/`DEMO_CODE`(/`_TS`) triplet in `shaders/bundles-{tier}.js`, and a
roadmap-card bullet in `shaders/index.html`. No new `DEPENDENCY_BUNDLES` — all
reuse each tier's existing `data-deps`.

- **Beginner** (`makeShaderToy`): `sh_transformsGL` (2×2 rotation matrix —
  translate/scale/rotate the coordinate), `sh_polarGL` (length/atan → radial
  gradient, cooldown-swipe UI, spiral), `sh_sdfCombineGL` (min/max/smin
  boolean shape combine). Inserted after `#shaping`.
- **Intermediate** (`makeShaderToy`): `sh_perlinGL` (gradient noise + fbm +
  marble + warp), `sh_voronoiGL` (F1/F2 cellular: cells/edges/cracked),
  `sh_flowGL` (conveyor/river/lava/flow-field). After `#distortion`.
- **Expert** (`makeShaderToy` + sprite texture path): `sh_normalmapGL`
  (luminance→heightmap→finite-diff normal→Lambert from `u_mouse`),
  `sh_spritesheetGL` (custom 4×4 sheet painter via `opts.sprite` + frame
  picker), `sh_ditherGL` (recursive 4×4 Bayer: posterize/dither/1-bit/GB
  palette). After `#hitflash`.
- **Advanced** (each = WebGL1 **and** WebGL2 pair): `sh_transitionGL1/GL2`
  (wipe/iris/dissolve/bars), `sh_radialGL1/GL2` (zoom blur / god rays),
  `sh_pixelGL1/GL2` (pixelate/mosaic/crt). Added by extending **both**
  `chainFor`s + the `B`/`kitSrc` body sets + 3 new `DEFS`; added an optional
  `param` field to the DEFS body template (backward compatible). After
  `#grade`.
- **Simulations** (`makeSim`, WebGL2+float): `sh_sand` (conservative
  **Margolus 1×2** gravity — `u_frame&1` flips the pairing; flicker-free),
  `sh_boids` (96² agents, 16 random `texelFetch` samples →
  separation/alignment/cohesion, POINT_VERT display), `sh_wave` (discrete 2D
  wave eq., `.r` height `.g` velocity, Laplacian, CFL-stable, lit display).
  After `#fluid`.

## Scope deviation (intentional, flagged to user)

Plan listed *slime mold (Physarum)* for Simulations; **swapped for the 2D
wave equation**. `makeSim` is single-buffer ping-pong; Physarum needs agents
that *sense* a separately-diffused trail map (two coupled fields the harness
can't feed back). Wave fits the single-state model perfectly (like
reaction-diffusion). Optional 4th lessons per tier were **not** built (kept
to the verified core set).

## Key learnings / gotchas

- **`node -c <file>` every edited bundle/demo BEFORE browser testing.** A
  closing `DEMO_CODE` delimiter was over-escaped (`})();\`;` instead of
  `` })();`; ``) in bundles-intermediate.js → template literal never
  terminated, bled into the next block → "Unexpected token ':'". Caught
  instantly by `node -c`; the inner GLSL backticks ARE escaped, the final
  one is NOT.
- **Test oracle:** `makeShaderToy`/`makeSim` `fail()` flips the info element
  to `#ff7b72` on any GLSL compile/link error. Asserting computed color
  stays grey (`rgb(158,158,158)` / `rgb(139,148,158)` standalone) = "shader
  compiled" without pixel readback.
- **Export verification:** `generateStandaloneHTML(id, deps, lang)` is a
  global; call it in-page, save via Playwright `filename`, decode the
  (double-)JSON-wrapped result, run the standalone, assert 0 console errors.
  Did this for the riskiest demo in every tier. TS export path can't be run
  here (Sucrase CDN blocked in sandbox — pre-existing for ALL repo exports);
  proved TS bundles correct by showing their GLSL is byte-identical to the
  JS twins (which run clean).
- **Advanced DRY contract:** neutral `TEX`/`OUT` bodies in demos.js
  (`g1`/`g2` add PHEAD + resolve dialect) ↔ `kitSrc()` regenerates the same
  via `__VER__/__SDECL__/OUT/TEX` substitution. Inject mode variants with
  `' + x + '` concat (like `gauss`), never `${}`, and keep injected GLSL
  single-line to avoid `\n`→`\\n` escaping in the exported kit.

## Known issue (pre-existing, flagged — NOT introduced as logic bug)

`shaders/advanced.html` eagerly creates a WebGL context per demo canvas. With
6 effects × 2 dialects it was already near the browser cap; +3 effects = 6
more contexts → console warns "Too many active WebGL contexts. Oldest context
will be lost" (fires from the existing `makeFXChainGL1/2`, advanced-demos.js
~216/220). Topmost demos may lose their context on this page. Inherent to the
all-eager-canvas page design; a real fix is a lazy/IntersectionObserver
init refactor of the whole tier — out of scope for this task. All new demos
themselves render correctly.
