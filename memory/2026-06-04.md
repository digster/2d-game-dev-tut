# 2026-06-04 — Tower Defense track: scaffold (engine core + landing page)

## Context / why
User proposed a new **Tower Defense** track (Kingdom Rush / Bloons style) from a
prior-session sketch (two screenshots), explicitly asking to use it as *inspiration*
and make it comprehensive — "add and change whatever is required". The genre is the
natural applied home for the Fundamentals' **flow fields** and **A***
(`advanced.html#flow-fields`/`#pathfinding`), which today exist only as isolated
demos; a TD makes them gameplay-critical.

Planned in plan-mode (plan file: `~/.claude/plans/this-is-a-soft-sundae.md`).
Two decisions confirmed with the user:
1. **Cover BOTH styles** — Kingdom Rush fixed lanes (Beginner/Intermediate) +
   Bloons-style open-field / maze with A* re-routing + flow-field crowds (Advanced).
2. **Scaffold-first, then one tier per commit** (the repo's established cadence).

Tiers were remapped from the screenshot's order into the repo's strict
Beginner→Intermediate→Advanced→Expert→Simulations ramp (the screenshot put flow
fields in Expert and economy in Advanced; here: hard pathfinding → Advanced,
performance/scale → Expert, the whole game + balancing dashboards → Simulations).

## What shipped this pass (the scaffold)
New folder `tower-defense/` mirroring bullet-hell/physics-puzzle structure. Globals
are `td`/`TD`-prefixed, pre-checked vs `shared/utils.js`, no ES modules.

- `engine/loop.js` — `tdLoop` (fixed-timestep accumulator, sibling of `bhLoop`/
  `pzLoop`/`pfLoop`, + a fast-forward `speed` knob the TD genre needs),
  `tdInstallPointer` (the "click a tile to place a tower" verb, getBoundingClientRect
  mapping + pointer capture), light `tdInstallKeys` (hotkeys / 1-9 tower select).
- `engine/render.js` — `TD` palette (frozen) + asset-free draws: `tdDrawGround`,
  `tdDrawGrid` (faint lines + blocked-tint + green/red buildable hover highlight),
  `tdDrawPath` (sandy rounded lane + spawn/goal caps), `tdDrawRange`, `tdDrawTower`
  (rounded base + barrel rotated to `tower.angle`), `tdDrawEnemy` (disc + HP bar +
  frost tint), `tdDrawProjectile`, `tdDrawHUD` (♥ lives / ● gold / wave).
- `engine/world.js` — the **map substrate**: `TDGrid` (tile↔world, `blocked` lane
  mask + `occupied` tower mask, `isBuildable`, `blockAlongPath`) and `TDPath`
  (waypoint or Catmull-Rom-smoothed route, **arc-length parameterized** — cumulative
  distance table → `pointAt(dist)`/`tangentAt(dist)`; a creep follows the lane by
  advancing one scalar `dist += speed*dt`). Pathfinding/targeting/combat are NOT here
  — taught on top in the tiers (the `PZWorld`/`BHField` rule).
- `index.html` — track landing page (What You'll Build, roadmap cards all "Planned",
  Fundamentals prereqs, and a **scaffold self-check** that renders a full demo scene).
- Wired into root `index.html` — a `TOWER` nav-button after bullet-hell + a TOC `<li>`
  (links the track index only; tier links land as tiers ship, matching the track nav).

## Bug the self-check caught (and fixed)
`TDGrid.blockAlongPath` first stamped only the path's polyline **vertices** — fine for
a dense spline but a raw waypoint path only has points at its corners, so cells along a
straight segment stayed buildable (the lane tile at (60,0) was mid-segment). Self-check
test 4 ("lane tile should be un-buildable") failed red. Fixed to **march each segment
in sub-tile steps** (< 1 tile apart, Bresenham-style) so every crossed cell is blocked.
