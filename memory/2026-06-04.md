# 2026-06-04 ‚Äî Tower Defense track: scaffold (engine core + landing page)

## Context / why
User proposed a new **Tower Defense** track (Kingdom Rush / Bloons style) from a
prior-session sketch (two screenshots), explicitly asking to use it as *inspiration*
and make it comprehensive ‚Äî "add and change whatever is required". The genre is the
natural applied home for the Fundamentals' **flow fields** and **A***
(`advanced.html#flow-fields`/`#pathfinding`), which today exist only as isolated
demos; a TD makes them gameplay-critical.

Planned in plan-mode (plan file: `~/.claude/plans/this-is-a-soft-sundae.md`).
Two decisions confirmed with the user:
1. **Cover BOTH styles** ‚Äî Kingdom Rush fixed lanes (Beginner/Intermediate) +
   Bloons-style open-field / maze with A* re-routing + flow-field crowds (Advanced).
2. **Scaffold-first, then one tier per commit** (the repo's established cadence).

Tiers were remapped from the screenshot's order into the repo's strict
Beginner‚ÜíIntermediate‚ÜíAdvanced‚ÜíExpert‚ÜíSimulations ramp (the screenshot put flow
fields in Expert and economy in Advanced; here: hard pathfinding ‚Üí Advanced,
performance/scale ‚Üí Expert, the whole game + balancing dashboards ‚Üí Simulations).

## Bug the self-check caught (and fixed)
`TDGrid.blockAlongPath` first stamped only the path's polyline **vertices** ‚Äî fine for
a dense spline but a raw waypoint path only has points at its corners, so cells along a
straight segment stayed buildable (the lane tile at (60,0) was mid-segment). Self-check
test 4 ("lane tile should be un-buildable") failed red. Fixed to **march each segment
in sub-tile steps** (< 1 tile apart, Bresenham-style) so every crossed cell is blocked.

## Pass 2 ‚Äî Beginner tier "The Path & The Tower" (6 demos) ‚Äî DONE
`tower-defense/beginner.html` + `beginner-demos.js`. Introduced the **inline entity model**
(top-level, console-testable, NOT on `window` ‚Äî promotes to `engine/entities.js` at Intermediate):
- `TDEnemy` ‚Äî path-following as ONE scalar: `dist += speed*dt`, then `path.pointAt(dist)` ‚Üí pos;
  `dist >= length` ‚áí leaked. `pos` is a Vector2D, with `x`/`y` getters so it feeds `tdDrawEnemy`.
- `tdPickTarget(tower, enemies)` ‚Äî the "first" mode: furthest-along (max `dist`) creep IN range
  (squared-distance check, no sqrt).
- `TDTower` ‚Äî range + acquire + aim + `1/fireRate` cooldown; `update()` returns the creep it fired
  at (or null). Deliberately does NOT decide shot type ‚Äî the demo turns the return into a hitscan
  beam OR a `TDProjectile`. Clean seam ‚Üí promotes cleanly.
- `TDProjectile` ‚Äî travels toward target's current pos; hit when within `speed*dt + target.radius`
  (sub-step reach, no tunnelling); fizzles if target dies first. Uses `subtract` (returns NEW) then
  mutates its own `pos` ‚Äî the Vector2D footgun handled as a teaching point.

Demos: (1) map + hover-buildable + straight‚Üîsmooth lane toggle; (2) creep follows lane (speed
slider) + an arc-length SVG explainer; (3) spawner = a countdown emitting on a cadence (count/
interval sliders, Start Wave); (4) hitscan tower (range/fire-rate sliders, click to relocate);
(5) projectile tower (slow shots miss fast orange creeps ‚Üí motivates lead; projSpeed slider);
(6) capstone "First Line of Defense" (place towers on 120 gold / 40 each, Start Wave, kills pay
gold, leaks cost lives, win/lose). Each loop is paused off-screen via an IntersectionObserver
helper (`tdRunWhenVisible`) ‚Äî six rAF loops on one page, only the visible ones run.

## Pass 4 ‚Äî Advanced tier "Mazing, Flow Fields & Sight" (6 demos) ‚Äî DONE
The flagship tier ‚Äî the Fundamentals' A*/flow-field demos become gameplay. Algorithms are
**top-level + console-testable** in `advanced-demos.js`, awaiting promotion to `engine/nav.js` at
Expert (NOT on window yet):
- `tdAStar(grid, start, goal)` ‚Äî 4-connected A*, Manhattan heuristic, linear-min open set (noted:
  heap is the Expert upgrade). Returns [{col,row}] or null.
- `tdBlocksPath(...)` ‚Äî the connectivity guard: tentatively occupy a cell, run A*, restore, return
  whether it sealed the goal.
- `tdFlowField(grid, goal)` ‚Äî one BFS sweep from goal ‚Üí `cost` (integration field) + `flow` (unit
  vector per cell toward lowest-cost 8-neighbour). One field serves any number of creeps O(1) each.
- `tdLineOfSight(grid, ax,ay,bx,by)` ‚Äî sub-cell raycast; fails at first `isBlocked` cell (endpoints
  skipped). (Chose a DDA-style march over reusing lineIntersectsRect ‚Äî cleaner, ties to Bresenham.)
- `tdAStarPath(...)` builds a TDPath from an A* cell list, optionally starting at the creep's exact
  pos so a re-plan doesn't snap to a cell centre.

KEY design: **two creep movement models side by side** (the tier's point). Maze creeps **reuse
engine `TDEnemy`** over an A*-derived `TDPath`, re-planned from current cell on each map change.
Flow creeps are a **separate inline open-field model** (`tdMakeFlowCreep`/`tdStepFlowCreep` ‚Äî no
path, read the field vector at your cell and steer). Both draw via `tdDrawEnemy` (x/y fields).

Demos: (1) A* with drag-to-paint walls (route bends, null when sealed); (2) maze-TD (towers reroute
creeps + the deny-flash connectivity guard); (3) line-of-sight (drag a wall, sight line green/red,
tower only fires on green); (4) flow-field viz (cost heatmap + downhill arrows, re-flows around
painted walls, toggle heatmap); (5) open-field **crowd** (hundreds of flow creeps, spawn-rate
slider, one shared field); (6) capstone **"Build Your Maze"** (towers are gun+wall; A* reroute;
connectivity guard; 'closest' targeting since cross-path `dist` isn't comparable; gold/lives).

Map helpers re-declared tier-local (tdRunWhenVisible/tdPaintWall/tdDrawFlow/tdMarker). Wired into
track index (nav + card‚ÜíReady) and root TOC. Updated README + ARCHITECTURE.

## Pass 5 ‚Äî Expert tier "Ten Thousand Creeps" (6 demos) ‚Äî DONE

Perf systems taught **inline** (terminal consumer, the bullet-hell-Expert pattern ‚Äî NOT an engine
rewrite):
- `TDPool` ‚Äî free-list pool: spawn pops from `free` (allocs++ only when starved); `sweep()`
  recycles `dead` objects via O(1) swap-remove. After warm-up, allocations ‚Üí 0.
- `TDSwarm` ‚Äî Struct-of-Arrays creeps: one Float32Array per field (x/y/spd/hp); `remove(i)` =
  swap-remove (copy last over i, count--); `stepFlow(field,grid,dt)` steers each by the nav field,
  culls goal-reachers/off-grid. Backward iteration + swap-remove = each processed once/frame.
- `TDSpatialHash` ‚Äî uniform grid (string keys, cell‚âàrange); `insert`/`queryInto(x,y,r,out)` reads
  only overlapping cells.

Demos: (1) pool (toggle naive/pooled, "objects ever allocated" climbs-vs-plateaus); (2) SoA (AoS vs
SoA update-ms side by side, agent slider); (3) hash (naive T√óE vs hashed pair-test counts + the
first tower's query cells highlighted); (4) batch (per-dot vs one-path render-ms toggle); (5)
flowScale (thousands of SoA creeps on ONE nav flow field, steer-ms ~flat); (6) capstone **"Swarm"**
(SoA creeps + hash hitscan towers + pooled sparks + batched draw + frame-budget meter; click to add
towers). Towers HITSCAN via hash (no projectile index-tracking ‚Äî robust under swap-remove); damage
applied in the tower loop, dead culled AFTER (candidate indices stay valid mid-loop).

Helpers: `tdMeter` (EMA ms), `tdInfo` (innerHTML metric readout), tdRunWhenVisible re-declared.
