User proposed a new **Tower Defense** track (Kingdom Rush / Bloons style) from a
prior-session sketch (two screenshots), explicitly asking to use it as *inspiration*
and make it comprehensive — "add and change whatever is required". The genre is the
natural applied home for the Fundamentals' **flow fields** and **A***

Planned in plan-mode (plan file: `~/.claude/plans/this-is-a-soft-sundae.md`).

Beginner→Intermediate→Advanced→Expert→Simulations ramp (the screenshot put flow
fields in Expert and economy in Advanced; here: hard pathfinding → Advanced,
performance/scale → Expert, the whole game + balancing dashboards → Simulations).

## Bug the self-check caught (and fixed)
`TDGrid.blockAlongPath` first stamped only the path's polyline **vertices** — fine for
a dense spline but a raw waypoint path only has points at its corners, so cells along a
straight segment stayed buildable (the lane tile at (60,0) was mid-segment). Self-check
test 4 ("lane tile should be un-buildable") failed red. Fixed to **march each segment
in sub-tile steps** (< 1 tile apart, Bresenham-style) so every crossed cell is blocked.

## Pass 2 — Beginner tier "The Path & The Tower" (6 demos) — DONE
`tower-defense/beginner.html` + `beginner-demos.js`. Introduced the **inline entity model**
(top-level, console-testable, NOT on `window` — promotes to `engine/entities.js` at Intermediate):
- `TDEnemy` — path-following as ONE scalar: `dist += speed*dt`, then `path.pointAt(dist)` → pos;
  `dist >= length` ⇒ leaked. `pos` is a Vector2D, with `x`/`y` getters so it feeds `tdDrawEnemy`.
- `tdPickTarget(tower, enemies)` — the "first" mode: furthest-along (max `dist`) creep IN range
  (squared-distance check, no sqrt).
- `TDTower` — range + acquire + aim + `1/fireRate` cooldown; `update()` returns the creep it fired
  at (or null). Deliberately does NOT decide shot type — the demo turns the return into a hitscan
  beam OR a `TDProjectile`. Clean seam → promotes cleanly.
- `TDProjectile` — travels toward target's current pos; hit when within `speed*dt + target.radius`
  (sub-step reach, no tunnelling); fizzles if target dies first. Uses `subtract` (returns NEW) then
  mutates its own `pos` — the Vector2D footgun handled as a teaching point.

Demos: (1) map + hover-buildable + straight↔smooth lane toggle; (2) creep follows lane (speed
slider) + an arc-length SVG explainer; (3) spawner = a countdown emitting on a cadence (count/
interval sliders, Start Wave); (4) hitscan tower (range/fire-rate sliders, click to relocate);
(5) projectile tower (slow shots miss fast orange creeps → motivates lead; projSpeed slider);
(6) capstone "First Line of Defense" (place towers on 120 gold / 40 each, Start Wave, kills pay
gold, leaks cost lives, win/lose). Each loop is paused off-screen via an IntersectionObserver
helper (`tdRunWhenVisible`) — six rAF loops on one page, only the visible ones run.
