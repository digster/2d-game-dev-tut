## Key technical decisions in this tier

- **A* implementation choice**: 4-connected only (no diagonals), Manhattan
  heuristic, sorted-array open list. Keeps the code ~30 lines and the math
  obvious. Will revisit in Advanced when we need 1000-unit-scale performance —
  there's a key-point callout in `expert.html` flagging that future swap.
- **A* visited-set visualization**: `aStarPath` accepts an optional `visitedOut`
  Set; the pathfinding demo passes one in to render the explored cells as a
  heatmap, making the "why A* is better than BFS" lesson visual.
- **Selection happens in screen space**: the drag-box compares each unit's
  current screen position (via `cartToIso`) against the rectangle. Tile-space
  would work too but screen-space matches what the player sees and is what AoE2
  / RA2 do under the hood.
- **Building placement validity** is a single tight loop: for every cell in
  the footprint, check `isWalkable(map, …)` and a `blockedSet` of currently
  occupied cells. Ghost preview uses semi-transparent fills (green = OK,
  red = blocked) so the rule is felt before it's read.
- **Worker state machine**: 4 states (`walking_to_resource` → `gathering` →
  `walking_to_dropoff` → repeat). Drawn in the gathering demo and reused in the
  skirmish mini-project. Workers in the skirmish ignore combat by design — the
  Advanced tier introduces flee/auto-defense behaviors.
- **Combat**: minimal viable model. HP, attackDamage, attackRange,
  attackCooldown (with reset to attackCooldownMax). Ranged units spawn small
  projectile dots as a visual hint. Range > 1.5 is "ranged"; everything else is
  melee. Future tiers refine with armor types / damage tables.
- **Skirmish mini-project** integrates everything: player base (town centre +
  worker auto-harvesting + a soldier), enemy base across a river that
  auto-produces soldiers, drag-box selection + right-click move (A* around the
  water), and combat on contact. Deliberately leaves out fog of war, group
  collision avoidance, and real enemy AI — those are Advanced tier hooks.

## Verification (in-browser, against the static server)

- All 8 canvases paint (pixel-count check: 45k–102k non-bg pixels per canvas).
- Zero console errors, zero failed network requests.
- JS/TS tab swap works on each of the 7 code blocks.
- **Selection demo**: simulated drag-box from (50,50)→(700,350) selects all 7
  units, info display reads "3 soldiers, 2 archers, 2 workers".
- **Pathfinding demo**: click at canvas (478,264) → A* path of 14 tiles to
  (9,6), explored 44 cells. Path avoids the water row except at the single
  sand-ford tile.
- **Building demo**: hovering at canvas (400,100) with 2×2 brush selected →
  "Place 2×2 at (1, 1) — valid". Click placed the building.
- **Gathering demo**: after ~800ms of simulation, wood collected was 60 — proves
  workers ran the full state-machine loop multiple times.
- **Production demo**: clicking "Train Soldier" immediately showed
  "Building soldier (0.1 / 2s)"; after ~2.3s, "Queue empty · spawned 1".
- **Combat demo**: pressing Start triggered "Fight in progress…", combat HUD
  updated with HP counts.
- **Skirmish demo**: rendering and interaction handlers run without console
  errors; clicking "Train soldier" added to the queue.

## Deferred decisions, current status

- **Asset strategy**: still solid-color canvas drawings. Working well at this
  scale even with 8 distinct unit/building/resource visuals.
- **Cross-link vs duplicate**: hit twice in this tier — A* and tilemap concepts
  overlap with `advanced.html#pathfinding` and `expert.html#tilemap` in the
  Fundamentals guide. Resolution: pages have inline key-point callouts linking
  out (e.g., the production-caveat callout in the pathfinding section).
- **TypeScript parity**: maintained for all 7 code blocks. Cost was higher this
  tier (the A* TS pane is 30+ lines with proper types) but still tractable.
  Will reassess for the Advanced tier where 200-line classes are expected.
