  shared assets are referenced as `../shared/...` from inside the subdirectory.

- `isometric-strategy/index.html` — flipped Intermediate tier card from `.locked` to

### Key technical decisions in this tier
- **No new shared/utils additions.** Everything sits on the four iso helpers added in
  Phase 2 plus the existing `Vector2D` / `clamp` / `lerp`. The unit `Unit` class and the
  `Camera` class are taught inside the page (and the demo file) — they're game-specific,
  not shared utilities.
- **Mouse picking through camera** is the punchline of the camera section: we don't
  rewrite picking, we just pass the camera's current `originX/originY` and zoomed
  `tileW/tileH` to `pickTileFromMouse`. Verified end-to-end in `cameraDemo` and
  `miniProject`.
- **Zoom-around-cursor** in the camera/miniProject wheel handler — the origin is adjusted
  so the world-point under the mouse stays under the mouse after zoom. Common UX detail
  that's easy to forget; explicitly implemented.
- **Decorations + entities share the same sort key** `(cy + cx)` in the mini-project
  draw loop, so a tree at (4,3) and a unit at (3.5, 3.5) correctly overlap each other
  per their current fractional position. The depth-sort demo specifically calls out the
  "broken" vs "correct" mode side by side so learners see the bug visually.
