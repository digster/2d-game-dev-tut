## Key decisions / learnings
- **Export scaffold only renders `<button>` controls.** `shared/export-demo.js`
  maps `DEMO_HTML.controls` entries to `<button>` only — sliders/checkboxes/
  radios are not rendered. So only `voxDrawGrid` (button-only) is registered in
  `bundles-beginner.js`; the other 6 demos **omit `data-demo-id`** entirely so
  no broken Export button appears. Honest opt-out beats a broken export.
- **Top-level `const`/`class` don't attach to `window`.** Verification probe
  showed `VOX_COLORS`/`VOX_MATERIALS`/`TileWorld` as `undefined` on `window`
  while `function`s resolved — expected JS scoping, not a bug (same as iso's
  `ISO_COLORS`). Verify demos by **pixel-sampling the canvas**, not by reading
  globals.
- **Path depth is the #1 copy-paste risk** when adapting iso templates: every
  two-deep page needs `../../shared/…`. Verified clean (no 404s).
- The 404 on a missing `bundles-<tier>.js` is harmless (Export buttons just
  don't render) — but we ship the file so the console stays quiet.
