## Key decisions / learnings
- **Export scaffold only renders `<button>` controls.** `shared/export-demo.js`
  maps `DEMO_HTML.controls` entries to `<button>` only — sliders/checkboxes/
  radios are not rendered. So only `voxDrawGrid` (button-only) is registered in
  `bundles-beginner.js`; the other 6 demos **omit `data-demo-id`** entirely so
  no broken Export button appears. Honest opt-out beats a broken export.
- **Top-level `const`/`class` don't attach to `window`.** Verification probe

  globals.
- **Path depth is the #1 copy-paste risk** when adapting iso templates: every
  two-deep page needs `../../shared/…`. Verified clean (no 404s).
- The 404 on a missing `bundles-<tier>.js` is harmless (Export buttons just
  don't render) — but we ship the file so the console stays quiet.

## Key bug found & fixed during verification
**Player jittered ~1px and `onGround` flickered every frame.** Root cause: the
`overlappingTiles` AABB→tile-range helper used `floor((pos + size - 1) / TILE)`
for the far edge. When the player rests *flush* on a tile boundary that formula
reports the row *above* the floor, so gravity sinks the player a sub-pixel
amount undetected, then the collision fires and snaps back — every frame.
Fix: the integer cells a half-open span `[pos, pos+size)` touches are
`floor(pos) .. ceil(pos+size) - 1`. Switching the far edge to `ceil()-1`
detects a sub-pixel overlap the same frame → snap is flush, no jitter, stable
`onGround`. Fixed in all three places (demos.js, the HTML JS+TS code panes,
bundles DEMO_CODE). Verified: y rock-stable at 278 over 10 frames, jitter 0.000.

## Key decisions / learnings
- **Per-axis collision resolution** (move X → resolve → move Y → resolve) is
  what avoids corner-snag; **bottom-up scan** is what stops falling-sand from
  teleporting a whole column in one frame. Both written deliberately as the
  teachable "correct" version.
- **`ceil(end)-1`, not `floor(end-1)`** is the right AABB→cell-range edge — the
  `-1` form silently under-reports on fractional positions. Classic platformer
  off-by-one; cost a real jitter bug here.
- Export scaffold renders only `<button>` controls → demos with sliders/radios
  omit `data-demo-id` (same call as the Beginner tier).
- Bundle TS variants alias `DEMO_CODE_TS = DEMO_CODE` (plain JS is valid TS;
  Sucrase strips nothing) — shaders simulations-tier precedent. The page's
  teaching code blocks still ship hand-written typed TS panes.
- No ARCHITECTURE.md change — the Intermediate tier reuses the existing
  tier-file + bundles pattern exactly; nothing structural is new.

## Verification
- All 5 canvases render real pixels; console clean through every interaction.
- Player physics: walk (x 48→147), full jump (57px / ~3.5 tiles in open air),
  head-bonk correctly caps a jump under a platform, lands back flush, onGround

## Key algorithms / decisions
- **Value noise + fBm**: hash 4 lattice corners, smoothstep-blend, sum octaves
  (double freq, half amplitude). `generateTile(wx,wy,seed)` is a *pure function
  of position* — that purity is exactly what lets chunks generate independently
  and still seam up.
- **Caves = noise iso-band**: carve where `|fbm − 0.5| < width`. An iso-line is
  long and connected → wormy tunnels, not disconnected blobs. A single
  threshold (`fbm > t`) would give blobs instead.
- **Cellular water** = falling-sand + a sideways-spread step. The sideways step
  is what makes it pool instead of pile. Honest limitation noted in the tier
  (binary cells can't equalise levels — a lone droplet wanders).
- **Light flood fill**: BFS/Dijkstra relaxation from sky-exposed + torch tiles,
  subtracting per-tile attenuation (air 0.10, solid 0.24). Flat array used as a
  queue with a `head` pointer to avoid `Array.shift()`'s O(n) cost.
- No ARCHITECTURE.md change — Expert reuses the existing tier-file pattern.

## Bug / polish found during verification
- **seedTorches placed only 1 torch.** It probed 3 fixed columns (14/30/46);
  for seed 21 only one had a cave at a clean air-above-solid spot. Fixed:
  probe ~8 columns (`x += 6`), place into the first 4 that have caves —
  robust to whatever the seed carves. Now 4 torches; verified lighting
  contrast (cave luminance 24 vs lit 563). Fixed in demos.js + bundle.

## Verification

- Noise octave slider, worldgen regenerate, cave-width slider all work.
- Water: poured 28 tiles, conserved through a run, 2800 water-blue pixels

- Lighting: 4 torches seeded, strong dark/lit contrast, glow gradients.
- Chunks: panning grew the generated-chunk count 8 → 14.
- Both export bundles generate valid standalone HTML (vox_water 8.8 KB,

## Key algorithms / decisions
- **RLE compression**: store `[value, count]` run pairs, count capped at 255.
  A generated world compressed 6.1× in testing; holes break runs so a
  hand-mined world compresses less (a nice teaching observation).
- **A* on the tile grid**: g + h with a Manhattan heuristic; the grid *is* the
  nav graph, so "re-plan on edit" is free. Open set is a linear-scanned array
  (simple at demo scale; the tier notes a binary heap is the real fix).
- **Day/night**: just the Expert flood-fill with the skylight *seed value*
  turned into `0.06 + 0.94·dayFactor`, dayFactor a cosine of time. Rain is a
  0.7× multiplier on top.
- **Offscreen-canvas caching**: render the static world once to an off-DOM
  canvas, then one `drawImage` blit per frame. Measured cached ≈0 ms vs
  uncached 1.38 ms/frame.
- No ARCHITECTURE.md change — Advanced reuses the tier-file pattern.

## Bug found during verification
- **voxPathfind opened on "no path".** Two random air cells in a cave-riddled
  world are often stranded in separate pockets. First fix (regenerate until a
  path exists) handled `reset()`, but the auto-roaming enemy still picked a
  fresh *random* goal each time it arrived — frequently unreachable. Real fix:
  a `newGoal()` helper that only returns a goal A* can reach (path length > 6),
  used by both `reset()` and the arrival handler. Verified 6/6 resets now open
  with a valid path. Fixed in demos.js + bundle.
