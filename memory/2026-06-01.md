## Why this track

The **applied home** for the repo's homeless physics content — `simulation.html` /
`simulation-v2.html` teach verlet, springs, rigid bodies + SAT, cloth, SPH, buoyancy as
standalone demos with no game. This track re-teaches them puzzle-first. Distinct from
every prior track in that its core engine *grows tier by tier* (circle bodies →
constraints → rotation/SAT → fracture → fluids). **Through-line baked into the prose:**
two families of 2D physics in one world — position-based **Verlet** (ropes/soft bodies)
vs velocity/**impulse**-based rigid bodies — bridged in the "Rube" capstone.

## Key design decision (changes last iteration's forward plan)

Cut the Rope is built **pure Verlet** (the rope AND the hanging payload are
position-based `PZVerletPoint`s; collision = depenetrate `pos`, no velocity to
reflect). So this tier **does NOT consume the velocity-based `pzResolveStatic`** — its
promotion to `engine/collide.js` correctly slips to **Advanced** (the real 2nd `PZBody`
consumer). Forcing a contrived `PZBody` payload here just to trigger the promotion
would have been less honest. The Verlet `PZVerletPoint`/`PZConstraint` are taught
inline, to promote to `engine/constraints.js` when Simulations (soft bodies/ragdolls)
reuses them. Updated the forward-looking promotion note in README + ARCHITECTURE
accordingly.

# 2026-06-01 (pt.3) — Physics Puzzle: Advanced tier "Rigid Bodies & Joints" (World of Goo)

"Okay, work on the next iteration." → the Advanced tier (heaviest one).

## Design correction (promotion guess was wrong AGAIN — now settled)

Advanced uses **convex-polygon rigid bodies + SAT**, a *different* collision family from
Beginner's circles. So Beginner's `pzResolveStatic`/`pzCollideCircles` are NOT consumed
here — they're **superseded** (the rigid solver is the general case). Net: through
Advanced, the circle family AND the Verlet classes each still have only ONE real consumer
→ both stay inline, and `engine/` stays at **3 modules** (no promotion fired). Real 2nd
consumers: circle family → grand-capstone slingshot; Verlet → Simulations soft-bodies
(→ constraints.js); rigid family → Expert destruction (→ collide.js + rigid module).
Lesson recorded in ARCHITECTURE: **each tier brings its own collision family, so never
pre-commit a promotion to a specific future tier.** Fixed README + ARCHITECTURE.

## Shipped (browser-verified, console clean)

- **`physics-puzzle/advanced.html` + `advanced-demos.js`** — "Rigid Bodies & Joints",
  6 demos: rotation (off-centre impulse spins a box), SAT detection (visualised normal +
  contacts), impulse-with-rotation response, **stable resting stacks** (friction +
  positional correction), breakable joint chains, capstone **"Contraption"** (see-saw
  catapult). Inline rigid engine (standard Box2D-lite approach): `PZRigidBody`
  (mass/inertia/angle/angularVel), `pzPolyVsPoly` (SAT + reference/incident-face
  clipping → ≤2 contacts), `pzSolveManifold` (sequential impulses: normal+restitution+
  Baumgarte bias, Coulomb-clamped friction, all with `r × J`), `PZJoint` (2×2
  effective-mass pivot constraint + break threshold), `pzStepWorld`, plus `pzBoxVerts`/
  `pzRegularVerts`/`pzPolyMassData`/`pzDrawPoly`/`pzArenaBodies`.
- Nav/index sync: landing Advanced card → Ready + nav; intermediate.html nav + recap
  link to Advanced; root index TOC Advanced → real link. README + ARCHITECTURE updated

## Bugs found & fixed this tier

1. **Floor-top mismatch**: demos placed blocks at `H-26` but `pzArenaBodies` floor top is
   `H-6` → blocks spawned 20px high and dropped, jolting marginal structures apart. Fixed
   to `H-6` everywhere.
2. **Resting load vs hit**: stacked blocks have big resting `impact` (bottom of a 3-stack
   ≈ 1300), so an absolute break/juice threshold auto-fires on resting load. Fixes:
   impactDemo uses a STATIC wall (resting impact 0); juiceDemo triggers on the thrown
   BALL's impact (a ball carries no resting load); demolishDemo breaks brittle blocks on
   CONTACT with a fast wrecking ball (ignores resting load).
3. **Unstable structures**: the original stress "temple" (thin columns + heavy lintel) and
   a 6-high demolish column both toppled on their own. Replaced with a stable stacked-post
   **trilithon** (stress) and a stable 2×3 brick wall (demolish), both stability-tested
   (post-settle drift < 3px) and the demolish proven clearable in ≤3 shots (lenient win:
   ≤1 piece above the line).

## Gotchas recorded

- **Heavy rigid bodies + position-based fluid = instability.** Couple only light objects
  (buoyancy); make heavy bodies one-way displacers or remove-on-entry.
- **Preview caching:** the static server / preview browser does NOT revalidate `*.js` on
  reload — a changed demos file keeps serving stale JS. Only a NEW url works: temporarily add
  `?v=N` to the `<script src>` (then revert). Cost me several confusing screenshots. The
  committed files are correct; caching only affects the live preview session.
- `class` decls are lexical globals (not on `window`) — fine in-file; engine files still do
  explicit `window.X = X` for cross-`<script>` use.
