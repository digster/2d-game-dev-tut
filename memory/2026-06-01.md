## Why this track

The **applied home** for the repo's homeless physics content — `simulation.html` /
`simulation-v2.html` teach verlet, springs, rigid bodies + SAT, cloth, SPH, buoyancy as
standalone demos with no game. This track re-teaches them puzzle-first. Distinct from
every prior track in that its core engine *grows tier by tier* (circle bodies →
constraints → rotation/SAT → fracture → fluids). **Through-line baked into the prose:**
two families of 2D physics in one world — position-based **Verlet** (ropes/soft bodies)
vs velocity/**impulse**-based rigid bodies — bridged in the "Rube" capstone.

## Footguns recorded (in code headers + ARCHITECTURE)

- **`Vector2D` mutates in place** (`add`/`multiply`/`divide`/`normalize`/`limit` change
  `this`; only `subtract`/`copy` return new). Every maths line `.copy()`s a shared vector
  first — corrupting the shared `gravity` vector is the bug this prevents.
- **No top-level shadowing** — all engine + inline names `pz`/`PZ`-prefixed (incl. generic
  ones: `PZBody`, not `Body`), pre-checked vs `shared/utils.js`.
- **Tunneling** — launch speed clamped via `limit()` so a fast shot steps < its radius.
