# 2026-06-03 — Beginner page: wire up the 5 missing demo Export buttons

## Context / why
User noticed the Matrix Transformations demo (and others) had no "📋 Export" button.
Question started scoped to the beginner page but clarified to mean the whole
**game-math track** (`beginner → intermediate → advanced → expert → simulation →
simulation-v2`). A track-wide survey (`<canvas id>` count vs `data-demo-id` count)
showed every page except **beginner** was already 100% wired. Beginner had 11
interactive demos but only 6 Export buttons. User chose to wire up all 5 gaps.

## What changed
The Export button is injected by `shared/export-demo.js` only onto
`<details data-demo-id="X" data-deps="…">`, and only *works* if `X` is registered in
`DEMO_CODE` + `DEMO_HTML` (`shared/demo-bundles.js`) with each dep in
`DEPENDENCY_BUNDLES` (`shared/dependency-bundles.js`).

Two tiers of work:
- **Already had bundles, just needed wiring:** `vectorBasics`, `vectorPlayground`.
  - `vectorPlayground` → added attrs to the existing "Complete Code: Vector Playground"
    `<details>` (deps `vector2d,clearCanvas,drawGrid,drawVector,clamp,randomFloat`).
  - `vectorBasics` had **no** code-reveal block (the "Understanding Vectors" demo never
    had one — that's why it had no button). Added a NEW `<details class="code-block"
    data-demo-id="vectorBasics" data-deps="vector2d,clearCanvas,drawGrid,drawVector">`
    after the demo's info div.
- **Needed new bundles authored** (adapted from `beginner-demos.js`, rewriting the
  page-specific `canvas`/`info`/var names to the standalone harness `canvas`/`info`,
  keeping control-button ids): `advancedVector`, `advancedTrig`, `matrix`. Added 3
  `DEMO_CODE` + 3 `DEMO_HTML` entries right after the `homingMissile` beginner group.

## Shared dependency changes (`shared/dependency-bundles.js`)
- Added `project()` + `reflect()` to the `vector2d` bundle (copied from `utils.js`).
- Added a new `matrix2d` bundle (from `utils.js` Matrix2D: rotate/scale/translate/
  transformPoint/identity).

## KEY LEARNING — mirror drift
`DEPENDENCY_BUNDLES` is a **hand-copied mirror of `shared/utils.js`** and had silently
drifted: `Vector2D.project`/`.reflect` existed in `utils.js` but NOT in the `vector2d`
export bundle, and `Matrix2D` wasn't a dependency at all. So `advancedVector`/`matrix`
worked on the page (use live `utils.js`) but their *exports* would have thrown.
→ When registering a demo for export, verify every class/method it calls exists in the
**mirror**, not just in `utils.js`. `matrix2d.transformPoint` returns a `Vector2D`, so
list `vector2d` before `matrix2d` in `data-deps`.

# Part 2 — follow-up: fix the real issues from the edge-case review

After Part 1, reviewed 5 edge cases. Decided (user) **not to add tests** but to **fix real
issues**. Triage: #2 (TS→JS fallback) and #4 (no-mouse NaN) are **non-bugs** (graceful by
design / already guarded). Fixed the other three:

- **Fix 1 — complete the dependency mirror** (`shared/dependency-bundles.js`): the `vector2d`
  bundle still lagged `utils.js` (missing `static subtract`, `distanceSquared`, `setLength`,
  and the `divide` zero-warn); `matrix2d` lacked `copy()`. Added all, verbatim from `utils.js`.
  These weren't called by any current export (so preventive, not an active break), but now the
  bundle classes are **exact mirrors** — verified by a throwaway Node script comparing
  `getOwnPropertyNames(proto/static)` of live vs bundle classes → 0 missing both ways.
- **Fix 2 — clipboard fallback** (`shared/export-demo.js`): new `copyTextToClipboard(text)` —
  async Clipboard API first, then hidden-`<textarea>` + `execCommand('copy')` (works inside the
  click gesture). So Export works on `file://` / insecure contexts instead of "✗ Error". Handler
  now drives ✓/✗ off the boolean return. Shared → benefits every track's Export button.
- **Fix 3 — FOV logic bug** (`beginner-demos.js` + `shared/demo-bundles.js`, kept in sync): the
  `advancedTrig` `fov` branch set `enemy.angle = atan2(toMouse)` (enemy snapped to face cursor)
  → `angleDiff` always 0 → the cone never gated detection. Replaced with an independent sweep
  `enemy.angle += 0.01`, so the rotating cone actually limits the field of view (hide outside it).
  **Behaviour change to an existing demo** (intentional correction).

## Verification (real Chromium, Playwright MCP)

- Mirror: `advancedVector` + `matrix` exports regenerate, run with **zero console errors**, render.
- Clipboard (tested against cache-busted served bytes — beware the project's `.js` browser-cache

## Learning
The hand-mirrored bundle (`DEPENDENCY_BUNDLES` ⟷ `utils.js`) is the recurring hazard: a method
can exist live but not in the mirror, so the *page* works and only the *export* breaks. Keep the
`vector2d`/`matrix2d` bundles exact mirrors; when a demo lives in two places (page + bundle), fix
both together (the FOV bug existed in both copies).
