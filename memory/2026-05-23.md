## Key conventions locked in (inherit for future tiers)

- **Units convention** — position px, velocity px/s, heading rad
  (0 = +X = "right"), angular velocity rad/s, acceleration px/s². Written
  into the top comment of `beginner-demos.js`. The whole track is mass-less
  in Beginner (`acceleration = throttle × MAX_ACCEL`); mass + tire forces
  land in Intermediate/Expert without changing units.
- **`MAX_DT = 0.05`** clamp in every per-frame loop. Tab refocus no longer
  teleports the car. Inherit this in every later tier — global rule,
  not per-demo.
- **Steering at zero speed = no rotation.** Linear `clamp(speed/30, 0, 1)`
  ramp from 0 to full angular response between 0 and ~30 px/s. Sim-leaning
  Ackermann-ish feel without "stuck at standstill" UX.
- **Brake wins** when throttle + brake are held together (net throttle = 0).
- **Offscreen-canvas skid layer** is the cheap-and-correct pattern for
  trail effects — no point arrays, no unbounded growth, easy clear/fade.
  Use it again for the drift-pad demo in Intermediate.
- **Top-level safe names in `beginner-demos.js`:** `RACING_COLORS`,
  `Car`, `MAX_SPEED`, `MAX_ACCEL`, `MAX_STEER_RATE`, `MAX_DT`,
  `integrate`, `drawCarSprite`, `rearWheelWorldPositions`,
  `headingDeg`, `attachKeyInputs`, `startFrameLoop`. None collide with
  `shared/utils.js` exports (`Vector2D`, `Matrix2D`, `lerp`, `clamp`,
  `map`, `clearCanvas`, `drawVector`, `drawGrid`, `createDemoController`,
  etc.) — pre-grep cleared.
- **`<details>` for code blocks intentionally OMITS `data-demo-id`** because
  no `racing-sim/bundles-beginner.js` ships yet. Same opt-out approach the

## Key decisions locked in

- **The Intermediate integrator (`integrateWithGrip`) is a strict superset
  of the Beginner integrator.** Add two steps (decompose → drag+grip →
  recombine) and you can't tell the Beginner-tier behaviour is gone if
  you set `gripLimit = ∞` and `rollingDrag = 0`. Future tiers extend the
  same way: Expert replaces the grip clamp with a Pacejka curve; the rest
  of the loop is unchanged.
- **Units are unchanged from Beginner.** The header comment in
  intermediate-demos.js reaffirms position px, velocity px/s, heading rad
  — and adds the new ones (rollingDrag in 1/s, gripLimit in px/s²).
- **Surface presets are tunables, not enums.** Each surface is just
  `{ name, color, rollingDrag, gripLimit }`. A "wet tarmac" or "snow"
  preset is two numbers, no integrator change.
- **Skid marks now stamp only while sliding** (Intermediate-only refinement
  of the Beginner behaviour). Gripped tyres leave dust, not rubber — the
  visual cue mirrors the GRIP/SLIDE HUD perfectly.
- **`Car`, `RACING_COLORS`, and the helpers are redeclared at top-level
  with the same names as in beginner-demos.js.** Each tier page loads only
  its own demos file (never both), so there's no collision. Using the same
  identifiers across tiers keeps the *reading* of the source uniform.
- **Oversteer/understeer is modelled by two knobs on top of the integrator,
  not by a two-axle model.** Two-axle physics arrives in Expert. The
  one-knob `yawKick` + `steerEff` approximation is the cheapest possible
  thing that produces the qualitative feel — and it composes cleanly with
  the Expert tier's Pacejka upgrade.

## The pivot

The track becomes data:

- **Centerline** = array of sampled points (from `buildCenterline` →
  chained `sampleCubic` calls on cubic Bézier segments).
- **Walls** = `inflateWalls(centerline, halfWidth)` returns parallel
  left/right polylines (each centerline point offset by the local
  perpendicular).
- **Checkpoints** = `buildCheckpoints(centerline, halfWidth, count)`
  returns perpendicular line segments at evenly spaced indices.

That one structure powers all three "new" mechanics. The mini-project
needs ~70 lines on top of it because almost everything is reuse.

## Key decisions locked in

- **AI uses the player's integrator.** No second physics path. The AI
  only differs in where its (throttle, steer) numbers come from.
- **AI uses the centerline, not a true racing line.** A real racing
  line is a Simulations-tier topic. The Advanced tier acknowledges
  this with the "Racing Line ≠ Centerline" section and a "swap
  centerline for offset array" pointer for the curious.
- **Cars don't collide with each other.** Car-vs-car collision lands
  in Expert (it pairs with the race state machine and proper start
  grid). The Advanced race demo plays it as "bump-and-run" — the
  cars pass through each other.
- **Wall bounce uses `bounceDamping = 0.55`** by default. Pure
  reflection (damping 1.0) looks like a billiard ball; cars need
  some energy loss to feel weighted. Tunable per-call.
- **Closed-loop centerline is implicit** — `points[(i+1) % n]`
  everywhere. No "isClosed" flag, no separate handling of the wrap.

## The pivot

The central change is replacing the Intermediate hard grip clamp with a
smooth Pacejka curve:

  F = D · sin(C · atan(B · α))

where α is the slip angle (atan2(v_lateral, v_forward)). The peak sits
at (π/2)/(B·C) — ~6.5° with the defaults (B=8, C=1.5). Past the peak
the force DECREASES, which is what makes losing the back end real (and
what makes counter-steer a meaningful action). Setting B very large
recovers the Intermediate clamp behaviour as a limiting case — strict
superset.

2. **weightTransferDemo** — top-down car with four wheel-load

## Key decisions locked in

- **Pacejka is the only physics change.** Weight transfer is
  visualisation-only at this tier (the integrator does not split front
  vs rear axles). True two-axle physics is a Simulations-tier topic.
- **Brake authority > throttle authority.** `MAX_BRAKE = 360` vs
  `MAX_ACCEL = 300` in the integrator — real cars decelerate faster
  than they accelerate.
- **Camera uses framerate-correct smoothing** (`1 - exp(-k·dt)`), not
  the naive `lerp(a, b, smoothness · dt)`. The naive version overshoots
  past 1 at high dt and produces snapping. Documented inline.
- **Race state machine is the source of truth.** Input layer reads
  `state === 'RACING'` to decide whether to forward keys; HUD reads
  it to decide what overlay to draw; the lap timer only ticks during
  RACING. No flags scattered through the code.
- **Slip-angle threshold for the GRIP/SLIDE flag** uses the curve's
  peak angle, not an arbitrary number. So as you re-tune B/C, the flag
  threshold tracks the actual curve behaviour.
- **Gamepad input is auto-detected.** If any pad axis / trigger is
  past its dead-zone the demo switches to pad input; otherwise it uses
  WASD. The HUD pill shows which mode is live.
