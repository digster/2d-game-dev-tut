## Architecture notes (so future sessions don't relearn)

- **Tier-file convention** (from ARCHITECTURE.md): each `<tier>-demos.js` is
  self-contained — IIFE per demo, early-`return` if its canvas id is absent. A
  top-level `const`/`let`/`class` that shadows a `shared/utils.js` global
  (`lerp`/`clamp`/`map`/`Vector2D`) throws "already declared" at load and silently
  kills the whole file. Keep such helpers inside IIFEs.
- **Engine split:** truly cross-tier stuff (RNG, grid, renderer) lives in
  `roguelike/engine/`; each tier's *taught* algorithm (BSP, FOV, scheduler, ECS) is
  implemented in that tier's demos file and re-declared/assembled in the Simulations
  capstone.
- **Export opt-out:** roguelike demos are keyboard/canvas-driven; `export-demo.js` only
  renders button controls, so demos omit `data-demo-id` (no `bundles-*.js` for now).
  `rl_` namespace reserved for future bundles.

## Cross-track conventions reaffirmed this session (for future work)
- Tier files self-contained; shared helpers promoted to `engine/` on the 2nd/3rd
  consumer. Two promotion styles: *move* (actors/vision — origin tier uses the engine
  copy) vs *lib copy* (dungeon/rpg — origin tier still teaches inline; load the lib
  copy only where the inline copy isn't).
- Make non-trivial algorithms top-level globals so they're unit-testable from the
  console (FOV shadow test, cave connectivity, determinism hash, energy-count).
- Inventory selection = NUMBER keys (1–9); letters are movement. Keyboard demos need
  click-to-focus + preventDefault on movement keys.
- Every commit: verify in-browser with a CLEAN console before showing the commit
  message; docs (README/ARCHITECTURE/PROMPT/memory) finished first.
