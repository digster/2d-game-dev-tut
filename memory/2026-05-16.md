# 2026-05-16 — Shaders track: GLSL readability refactor + Intermediate tier

## Part 1 — Refactor: GLSL string-concat → template literals

The Beginner tier wrote every fragment source as `'…\n' + '…\n' + …`. Converted
to multi-line template literals across `shaders/beginner-demos.js` (plain JS, no
escaping), `shaders/bundles-beginner.js` (nested in `DEMO_CODE` literals → only
delimiter backticks escaped `` \` ``; water demo interpolation `\${...}`), and the
displayed `<details>` panes in `shaders/beginner.html`. Pure readability —
behavior byte-identical (verified: same pixels, exports run, no regressions).

  WebGL helpers are duplicated verbatim into the intermediate twins, matching
  the project-wide `isometric-strategy/{tier}` pattern. Identical
  `window.DEPENDENCY_BUNDLES.sh_*` reassignment is a harmless no-op.
- Kept `precision mediump float;` (consistency/compat); noted the hash-banding
  caveat in a `key-point` rather than switching to `highp`.
- `prism-c` must load before `prism-glsl` (the Beginner load-order fix) — the
  `#random` section has a standalone `language-glsl` hash snippet.

### Key technical decisions

- **Texture source = procedural.** `drawSprite(ctx,256)` paints a faceted
  gem (4 luminance bands, transparent margins) onto an offscreen 2D canvas;
  `makeShaderToy` uploads it. No image assets → exports stay self-contained

  reasoning as the earlier rejected alternatives).
- **`makeShaderToy` extended, not forked.** New optional `opts.sprite`
  uploads `u_tex`/`u_texResolution`; absent → texture path skipped →
  Beginner/Intermediate behaviour byte-identical (not re-verified, untouched).
  `data-deps` unchanged in shape; two new leaf deps `sh_drawSprite`,
  `sh_glslKit` (shared `GLSL_HEAD`/`GLSL_NOISE`) keep demos DRY + export-safe.
- **WebGL1 texture gotchas, fixed once in the helper:** `UNPACK_FLIP_Y_WEBGL`
  (2D canvas Y-down vs `gl_FragCoord` Y-up), 256² POT canvas with
  `NEAREST` + `CLAMP_TO_EDGE` (NPOT-safe, crisp for palette/outline).
- Outline = 8 **unrolled** taps (no loop) → GLSL ES 1.00-safe; `discard`/
  threshold for dissolve; `u_texResolution` for the texel step.
- Sprite fitted into the 800×450 canvas via `toSpriteUV` (aspect-correct
  centred box) + checkerboard `backdrop` — itself §1 content.

### Curriculum (12 demos = 6 effects × {GL1,GL2})

`sh_rttGL1/2` (render-to-texture) · `sh_pingGL1/2` (ping-pong N passes) ·
`sh_blurGL1/2` (separable Gaussian) · `sh_bloomGL1/2` (bright→blur→composite)
· `sh_gradeGL1/2` (colour grade + CRT/vignette/chroma) · `sh_stackGL1/2`
(composed post-stack mini-project).

### Key technical decisions

- **`makeFXChain` is a new sibling, not an extension** of `makeShaderToy`
  (single-pass can't express N FBO passes). Renders scene→FBO, then post
  passes ping-ponging two scratch FBOs, last→screen; every pass gets
  `u_prev` + `u_scene`; RGBA8 FBOs, `checkFramebufferStatus`.
- **Two API variants fully contained:** `sh_fxChainGL1` (webgl + GLSL 1.00
  vertex) / `sh_fxChainGL2` (webgl2 + `#version 300 es` vertex). Page twin
  names them `makeFXChainGL1/GL2` (both load); bundle deps each define plain
  `makeFXChain` (one per export) — page≠bundle, canvas-id precedent.
- **Kit deps** (`sh_kitGL1/GL2`) hold per-dialect scene + post GLSL +
  `chainFor` so each `DEMO_CODE` is ~10 lines. Built from ONE neutral
  template via `.split().join()` of `__VER__`/`__SDECL__`/`OUT`/`TEX`
  (the proven g1/g2 pattern) — minimal escaping.
- **`#version 300 es` is the literal first line** of every GL2 shader
  (vertex + frag) — the #1 WebGL2 footgun, enforced in headers/VERT.
- 8-bit RGBA FBOs → bloom bright-pass is approximate (noted in a key-point;
  no float/HDR chase). bloom/stack use fixed-radius blur so `u_param` stays
  free for bloom strength.
- Page-local `.api-tab` toggle: distinct classes from `.code-tab`, scoped to
  `:scope > .api-pane` per `.section` → JS/TS code tabs keep working inside
  the visible pane; `shared/*` untouched.

### Curriculum (6 demos, WebGL2-only)

`sh_simLoop` (decay/splat — the scaffold) · `sh_life` (Conway, 8-neighbour
`texelFetch`) · `sh_reaction` (Gray-Scott, 12 substeps, feed/kill presets —
the float showcase) · `sh_particles` (256² pos/vel texture, `gl_VertexID`
vertex-fetch points display) · `sh_fluid` (lite non-projected advection) ·
`sh_playground` (fluid + dye presets / viscosity / clear).

### Key technical decisions

- **`makeSim` is a third sibling** (not `makeShaderToy`/`makeFXChain`):
  cross-frame *persistent* double-buffer (FBOs never recreated per frame),
  seed-both-then-ping-pong, `substeps`/frame, mouse (0..1 Y-flipped),
  quad-or-`POINTS` display, `reset()`/`rebuild()`.
- **WebGL2 + RGBA16F / HALF_FLOAT**, `NEAREST` + `texelFetch` (exact texels —
  never linear-filter sim state). `EXT_color_buffer_float` mandatory →
  guarded `fail()` with a clear message (graceful, not blank/crash) if the
  browser/sandbox lacks it.
- `#version 300 es` literal first line of every shader; `precision highp`
  (RD/fluid need it). Particles use a vertex shader that *is* the particle
  (`gl_VertexID`→`texelFetch`) — a genuinely WebGL2/GLSL3 technique.
- Fluid kept honestly *lite* (non-projected, NEAREST-sampled advection) —
  noted in a `key-point`; the technique is the real one, scope is bounded.
- `EXT_color_buffer_float` may be absent in the headless preview →
  verification validates correct behaviour under BOTH capabilities (renders,
  or shows the graceful float-unsupported message — never crash/blank).
