# 2026-05-16 — Shaders track: GLSL readability refactor + Intermediate tier

## Part 1 — Refactor: GLSL string-concat → template literals

The Beginner tier wrote every fragment source as `'…\n' + '…\n' + …`. Converted
to multi-line template literals across `shaders/beginner-demos.js` (plain JS, no
escaping), `shaders/bundles-beginner.js` (nested in `DEMO_CODE` literals → only
delimiter backticks escaped `` \` ``; water demo interpolation `\${...}`), and the
displayed `<details>` panes in `shaders/beginner.html`. Pure readability —
behavior byte-identical (verified: same pixels, exports run, no regressions).

### Key technical decisions

- **GLSL ES 1.00: `for` loop bounds must be constant.** Adjustable tile
  count / fbm octaves / fire turbulence rebuild the shader with the count
  interpolated as a literal via `toy.setFrag(...)` — NOT a uniform loop
  bound. Same state-built pattern as Beginner's water `buildFrag`. Continuous
  knobs (warp strength, distortion strength, fire intensity) use `u_param`.
- Per-tier files are **self-contained** (tier pages never co-load) — the
  WebGL helpers are duplicated verbatim into the intermediate twins, matching
  the project-wide `isometric-strategy/{tier}` pattern. Identical
  `window.DEPENDENCY_BUNDLES.sh_*` reassignment is a harmless no-op.
- Kept `precision mediump float;` (consistency/compat); noted the hash-banding
  caveat in a `key-point` rather than switching to `highp`.
- `prism-c` must load before `prism-glsl` (the Beginner load-order fix) — the
  `#random` section has a standalone `language-glsl` hash snippet.

### Curriculum (sh_-prefixed, all new)

`sh_sampleGL` (texture2D + Y-flip + aspect-fit) · `sh_tintGL` (multiply) ·
`sh_paletteGL` (luminance→ramp) · `sh_dissolveGL` (noise threshold + burn
edge) · `sh_outlineGL` (8-tap alpha dilation) · `sh_hitflashGL` (white pulse
/ alpha blink) · `sh_spriteGL` (capstone: palette + outline + flash).

### Key technical decisions

- **Texture source = procedural.** `drawSprite(ctx,256)` paints a faceted
  gem (4 luminance bands, transparent margins) onto an offscreen 2D canvas;
  `makeShaderToy` uploads it. No image assets → exports stay self-contained
  (external assets would not survive the shared export-demo generator — same
  reasoning as the earlier rejected alternatives).
- **`makeShaderToy` extended, not forked.** New optional `opts.sprite`
  uploads `u_tex`/`u_texResolution`; absent → texture path skipped →
  Beginner/Intermediate behaviour byte-identical (not re-verified, untouched).
  `data-deps` unchanged in shape; two new leaf deps `sh_drawSprite`,
  `sh_glslKit` (shared `GLSL_HEAD`/`GLSL_NOISE`) keep demos DRY + export-safe.
- **WebGL1 texture gotchas, fixed once in the helper:** `UNPACK_FLIP_Y_WEBGL`
  (2D canvas Y-down vs `gl_FragCoord` Y-up), 256² POT canvas with
  `NEAREST` + `CLAMP_TO_EDGE` (NPOT-safe, crisp for palette/outline).
- Outline = 8 **unrolled** taps (no loop) → GLSL ES 1.00-safe; `discard`/
  threshold for dissolve; `u_texResolution` for the texel step.
- Sprite fitted into the 800×450 canvas via `toSpriteUV` (aspect-correct
  centred box) + checkerboard `backdrop` — itself §1 content.
