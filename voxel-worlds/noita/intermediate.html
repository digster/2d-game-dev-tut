💡 One engine, many materials. The Beginner tier had one rule for one material (sand). This tier generalises: each material has a kind — powder, liquid, gas, fire, solid — and a single step function dispatches on kind. Add a material by adding a table row, not a new engine. The update-order trick is a per-cell "moved this frame" flag: mark a cell processed when its content moves, skip already-moved cells — so one scan handles falling sand, rising gas and sideways-flowing liquid at once.

💧 Liquids & Density

A liquid is a powder that also spreads sideways: down → down-diagonal → left/right. The sideways step is what makes water level out into a flat surface instead of piling like sand.

Density: who sinks through whom

Two liquids meet — which goes on top? Each liquid has a density. A liquid can move "down" not only into air but by swapping with a lighter liquid below it. Water (density 3) sinks through oil (density 2), so oil ends up floating — the layering emerges from one comparison. Sand, denser still, sinks through both.

📝 Click to see: the liquid rule 
// Inside the per-cell dispatch, for a liquid of density `dens`:
const dens = MAT[m].density;

// "Down" accepts air OR a lighter liquid — that swap is the density rule.
const canDown = (t) => t === AIR || (MAT[t].kind === 'liquid' && MAT[t].density < dens);
const canSide = (t) => t === AIR;

if (tryMove(i, x, y + 1, canDown)) return;          // straight down
const d = Math.random() < 0.5 ? -1 : 1;             // random tie-break
if (tryMove(i, x + d, y + 1, canDown)) return;      // down-diagonal
if (tryMove(i, x - d, y + 1, canDown)) return;
if (tryMove(i, x + d, y,     canSide)) return;      // sideways — the liquid step
if (tryMove(i, x - d, y,     canSide)) return;

☁️ Gases: Inverted Gravity

A gas is a liquid turned upside down. Where a liquid tries down → down-diagonal → sideways, a gas tries up → up-diagonal → sideways. Smoke and steam rise, pool against ceilings, and slowly dissipate — each frame a small chance to vanish back to air.

Why the "moved" flag earns its keep here

The Beginner tier showed a top-down scan makes falling sand teleport. Rising gas has the mirror bug under a bottom-up scan. The fix that handles both at once: the per-cell moved flag. A gas cell that rises marks its destination moved; when the scan reaches that cell it's skipped — so it rises exactly one row per frame regardless of scan direction.

// A gas: the liquid rule with the vertical direction flipped.
if (Math.random() < 0.02) { grid[i] = AIR; return; }   // dissipate

const canG = (t) => t === AIR;
const d = Math.random() < 0.5 ? -1 : 1;
if (tryMove(i, x,     y - 1, canG)) return;            // up   (vs liquid's y+1)
if (tryMove(i, x + d, y - 1, canG)) return;            // up-diagonal
if (tryMove(i, x - d, y - 1, canG)) return;
if (tryMove(i, x + d, y,     canG)) return;            // sideways
if (tryMove(i, x - d, y,     canG)) return;

🔥 Fire: Propagation & Fuel

Fire isn't a material that moves — it's a material with a lifespan. A fire cell carries a burn timer (we store it in a parallel aux array). Each frame it counts down, tries to ignite flammable neighbours, puffs smoke upward — and when the timer hits zero, it dies.

Ignition is probabilistic

A fire cell doesn't instantly convert every neighbour. Each flammable neighbour catches with a probability per frame — so flame creeps along wood at a believable pace instead of flashing the whole structure at once. When a cell ignites, its burn timer is set from the fuel's energy: oil burns fast and bright, wood burns long.

📝 Click to see: the fire rule 
// `aux[i]` holds this fire cell's remaining burn time.
function stepFire(grid, aux, W, H, i, x, y) {
    aux[i]--;                                       // burn down

    // Try to ignite each flammable neighbour, with a per-frame probability.
    const nb = [[1,0],[-1,0],[0,1],[0,-1]];
    for (const [dx, dy] of nb) {
        const nx = x + dx, ny = y + dy;
        if (nx < 0 || nx >= W || ny < 0 || ny >= H) continue;
        const j = ny * W + nx;
        const nm = grid[j];
        if (MAT[nm].flammable && Math.random() < 0.26) {
            grid[j] = FIRE;
            aux[j] = MAT[nm].fuel;                  // fuel energy → burn time
        }
    }
    // Puff smoke upward into open air.
    if (y > 0 && grid[i - W] === AIR && Math.random() < 0.30) grid[i - W] = SMOKE;
    // Burnt out → become smoke or clear air.
    if (aux[i] <= 0) grid[i] = Math.random() < 0.5 ? SMOKE : AIR;
}

⚗️ Material Reactions

The last ingredient: materials that transform on contact. When water touches lava, both change — the water flashes to steam, the lava chills to stone. Rather than hard-code that into the water rule and the lava rule, we keep a reaction table: a list of "A next to B → A becomes C, B becomes D".

One table, checked before movement

Each frame, before a cell tries to move, it scans its neighbours against the table. A match transforms both cells and the cell is done for the frame. Adding a new interaction — acid eats stone, water freezes near ice — is one more table row, no engine change. This is the data-driven heart of a Noita-style sandbox.

📝 Click to see: the reaction table & check 
// "When A touches B: A → ra, B → rb."  Add an interaction = add a row.
const REACTIONS = [
    { a: WATER, b: LAVA, ra: STEAM, rb: STONE },   // water quenches lava
    { a: WATER, b: FIRE, ra: STEAM, rb: AIR   }    // water puts out fire
];

// Checked per cell, before it moves.
function tryReact(grid, W, H, i, x, y) {
    const m = grid[i];
    const nb = [[1,0],[-1,0],[0,1],[0,-1]];
    for (const rule of REACTIONS) {
        if (rule.a !== m) continue;
        for (const [dx, dy] of nb) {
            const nx = x + dx, ny = y + dy;
            if (nx < 0 || nx >= W || ny < 0 || ny >= H) continue;
            const j = ny * W + nx;
            if (grid[j] === rule.b) {
                grid[i] = rule.ra;
                grid[j] = rule.rb;
                return true;                       // reacted — done this frame
            }
        }
    }
    return false;
}
type Reaction = { a: number; b: number; ra: number; rb: number };
const REACTIONS: Reaction[] = [
    { a: WATER, b: LAVA, ra: STEAM, rb: STONE },
    { a: WATER, b: FIRE, ra: STEAM, rb: AIR   }
];

function tryReact(grid: Uint8Array, W: number, H: number,
                  i: number, x: number, y: number): boolean {
    const m = grid[i];
    const nb = [[1,0],[-1,0],[0,1],[0,-1]];
    for (const rule of REACTIONS) {
        if (rule.a !== m) continue;
        for (const [dx, dy] of nb) {
            const nx = x + dx, ny = y + dy;
            if (nx < 0 || nx >= W || ny < 0 || ny >= H) continue;
            const j = ny * W + nx;
            if (grid[j] === rule.b) {
                grid[i] = rule.ra;
                grid[j] = rule.rb;
                return true;
            }
        }
    }
    return false;
}

🧪 Mini-Project: the Full Sandbox

Every material, every rule, one canvas. Eight paintable materials — powder, three liquids, a flammable solid, fire, an inert wall — all running through the single dispatch step. The fun is in the combinations: oil on water that you set alight, lava you quench, sand poured through steam.

The whole engine is a table + a dispatch

Notice how little special-case code exists. A material is a row in MAT with a kind; the step function switches on kind; interactions are rows in REACTIONS. The richness on screen is emergent — it comes from a handful of small rules composing, not from code that knows about "a burning oil slick on a lake".
