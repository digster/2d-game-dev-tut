🎯 Mouse-to-Tile Picking & Tool Reach

Before the player can mine or place, the game has to answer one question every frame: which tile is under the cursor, and can the player reach it? The first half is the screenToTile function from the Beginner tier. The second half is new: a tool reach limit, so the player can only edit tiles near themselves — exactly like Terraria's pickaxe range.

Chebyshev distance — the "square" reach

Terraria's reach is a square region, not a circle. The distance metric for a square is Chebyshev distance: max(|dx|, |dy|). A tile is in reach when that value is ≤ the reach radius. (Use Euclidean √(dx² + dy²) instead if you want a circular reach.)

inReach = max(|tileX − playerX|, |tileY − playerY|) ≤ reach
📝 Click to see: tileReachOk 
// Is tile (tx, ty) within `reach` tiles of the player's tile (px, py)?
// Chebyshev distance -> a square reach region (Terraria-style).
function tileReachOk(px, py, tx, ty, reach) {
    return Math.max(Math.abs(tx - px), Math.abs(ty - py)) <= reach;
}

⛏️ Mining, Placing & the Hotbar

With picking solved, mining and placing are almost trivially short — that's the payoff of the grid representation. Mining sets a solid tile to air. Placing sets an air tile to the selected material. Both refuse to act on out-of-reach tiles, and placing refuses to overwrite an existing solid tile.

Two guards keep edits sane

  • Reach guard — tileReachOk(...) from the last section.
  • Occupancy guard — mine only solid tiles; place only into air. Without it, a click would "place" dirt on top of dirt (a no-op that still feels buggy) or "mine" empty sky.
📝 Click to see: mineAt & placeAt 
// Mine: clear a solid tile to air (id 0). Returns true if something changed.
function mineAt(world, tx, ty, playerTile, reach) {
    if (!tileReachOk(playerTile.x, playerTile.y, tx, ty, reach)) return false;
    const id = world.get(tx, ty);
    if (id === 0) return false;                 // nothing to mine
    if (!VOX_MATERIALS[id].solid) return false; // skip liquids/air
    world.set(tx, ty, 0);
    return true;
}

// Place: drop the selected material into an air tile. Returns true on success.
function placeAt(world, tx, ty, materialId, playerTile, reach) {
    if (!tileReachOk(playerTile.x, playerTile.y, tx, ty, reach)) return false;
    if (world.get(tx, ty) !== 0) return false;  // occupied — don't overwrite
    world.set(tx, ty, materialId);
    return true;
}

    if (world.get(tx, ty) !== 0) return false;
    world.set(tx, ty, materialId);

}

⏳ Falling Sand & Gravel

Most tiles in Terraria are static — dirt doesn't move. But sand and gravel obey gravity: mine the tile under a sand column and the whole column drops. This is your first cellular automaton — a rule applied to every cell, every frame.

The rule, and why scan order matters

For each falling tile: if the cell below is empty, move down. Else try down-left / down-right so piles form natural slopes.

The subtle part is the scan order. If you scan top-to-bottom, a sand tile moves down one cell, then the scan reaches that same tile again lower down and moves it again — it teleports the whole column in one frame. Scanning bottom-to-top fixes it: each tile is visited before the cell it would move into, so it moves exactly one cell per frame.

📝 Click to see: stepFalling (the cellular automaton) 
// A cell a falling tile can move INTO: only air, in-bounds.
function isOpen(world, x, y) {
    return world.inBounds(x, y) && world.get(x, y) === 0;
}

// Advance every falling tile by one cell. Call once per simulation tick.
function stepFalling(world) {
    // Bottom-up scan: visit each tile BEFORE the cell below it, so a tile
    // moves at most one cell per tick (no single-frame teleporting).
    for (let y = world.height - 2; y >= 0; y--) {
        for (let x = 0; x < world.width; x++) {
            const id = world.get(x, y);
            const m = VOX_MATERIALS[id];
            if (!m || !m.falls) continue;          // dirt/stone never move

            // 1. straight down
            if (isOpen(world, x, y + 1)) {
                world.set(x, y, 0);
                world.set(x, y + 1, id);
                continue;
            }
            // 2. down-left / down-right (randomize so piles aren't lopsided).
            // The extra isOpen(x+dx, y) check stops sand tunnelling diagonally
            // between two solid tiles.
            const dirs = Math.random() < 0.5 ? [-1, 1] : [1, -1];
            for (const dx of dirs) {
                if (isOpen(world, x + dx, y + 1) && isOpen(world, x + dx, y)) {
                    world.set(x, y, 0);
                    world.set(x + dx, y + 1, id);
                    break;
                }
            }
        }
    }
}
function isOpen(world: TileWorld, x: number, y: number): boolean {
    return world.inBounds(x, y) && world.get(x, y) === 0;
}

function stepFalling(world: TileWorld): void {
    for (let y = world.height - 2; y >= 0; y--) {
        for (let x = 0; x < world.width; x++) {
            const id: number = world.get(x, y);
            const m = VOX_MATERIALS[id];
            if (!m || !m.falls) continue;

            if (isOpen(world, x, y + 1)) {
                world.set(x, y, 0);
                world.set(x, y + 1, id);
                continue;
            }
            const dirs: number[] = Math.random() < 0.5 ? [-1, 1] : [1, -1];
            for (const dx of dirs) {
                if (isOpen(world, x + dx, y + 1) && isOpen(world, x + dx, y)) {
                    world.set(x, y, 0);
                    world.set(x + dx, y + 1, id);
                    break;
                }
            }
        }
    }
}

🏃 A Player Avatar: AABB Collision

The player is an axis-aligned bounding box (AABB) — a rectangle with a position and a velocity. Gravity pulls it down each frame; it must not sink through solid tiles or walk through walls. The technique that makes this robust is per-axis resolution.

Move one axis at a time

The naïve approach — move diagonally, then resolve — snags on flush tile seams (the player catches on the crack between two floor tiles). The fix:

  1. Move X only. Find every tile the box now overlaps; if any is solid, snap the box flush to that tile's edge and zero vx.
  2. Then move Y only and resolve the same way. Landing on a floor (vy > 0 hit) also sets onGround.

Resolving axes separately means a horizontal seam can never block horizontal motion — there is no diagonal step to catch.

📝 Click to see: per-axis AABB collision 
// Tile range the player's AABB (px,py,pw,ph in pixels) overlaps.
// A half-open span [pos, pos+size) touches integer cells
// floor(pos) .. ceil(pos+size)-1. The ceil()-1 form detects a
// sub-pixel overlap the same frame, so a player nudged into the
// floor by gravity snaps flush immediately — no jitter, stable onGround.
function overlappingTiles(px, py, pw, ph, TILE) {
    return {
        x0: Math.floor(px / TILE),
        x1: Math.ceil((px + pw) / TILE) - 1,
        y0: Math.floor(py / TILE),
        y1: Math.ceil((py + ph) / TILE) - 1
    };
}

// Move horizontally, then snap out of any solid tile hit.
function moveX(world, p, TILE) {
    p.x += p.vx;
    const r = overlappingTiles(p.x, p.y, p.w, p.h, TILE);
    let hit = null;
    for (let ty = r.y0; ty <= r.y1; ty++) {
        for (let tx = r.x0; tx <= r.x1; tx++) {
            if (!isSolid(world.get(tx, ty))) continue;
            // moving right -> snap to leftmost hit; left -> rightmost hit
            hit = hit === null ? tx : (p.vx > 0 ? Math.min(hit, tx) : Math.max(hit, tx));
        }
    }
    if (hit !== null) {
        p.x = p.vx > 0 ? hit * TILE - p.w : (hit + 1) * TILE;
        p.vx = 0;
    }
}

// Move vertically; landing (vy > 0) also sets onGround.
function moveY(world, p, TILE) {
    p.y += p.vy;
    p.onGround = false;
    const r = overlappingTiles(p.x, p.y, p.w, p.h, TILE);
    let hit = null;
    for (let ty = r.y0; ty <= r.y1; ty++) {
        for (let tx = r.x0; tx <= r.x1; tx++) {
            if (!isSolid(world.get(tx, ty))) continue;
            hit = hit === null ? ty : (p.vy > 0 ? Math.min(hit, ty) : Math.max(hit, ty));
        }
    }
    if (hit !== null) {
        if (p.vy > 0) { p.y = hit * TILE - p.h; p.onGround = true; }
        else          { p.y = (hit + 1) * TILE; }
        p.vy = 0;
    }
}
type Box = { x: number; y: number; w: number; h: number;
             vx: number; vy: number; onGround: boolean };

🛠️ Mini-Project: Side-Scroll Digger

Everything from this tier, composed into one playable scene: a player that walks and jumps through a world bigger than the screen, a camera that follows them, a mining/placing cursor with tool reach, and live falling sand. This is a small but complete Terraria-style core.

How the pieces fit together each frame

  1. Input → player physics — gravity, moveX, moveY (the #player section).
  2. Camera follow — centre the camera on the player, then clamp to world bounds (the Beginner camera).
  3. Mouse edit — mineAt / placeAt, reach measured from the player's tile (the #mine-place section).
  4. Falling sand — stepFalling every few frames so mined sand collapses realistically (the #falling section).
  5. Render — viewport-culled draw, then the player and cursor on top.
