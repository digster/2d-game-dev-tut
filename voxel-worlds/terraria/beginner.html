🧱 What is a 2D Voxel World?

A 2D voxel world represents the playable area as a fixed grid where every cell stores a material (air, dirt, stone, wood, …) instead of a sprite or polygon. The whole game state lives in a 2D array; rendering and physics both read from the same grid. This is the model behind Terraria, Starbound, Noita, Powder Toy, Minecraft 2D — every game where you can "dig anywhere" and "place anywhere".

🕹️ Games you've probably played

  • Terraria — the inspiration for this sub-track
  • Starbound — same model, sci-fi setting
  • Junk Jack, Crea, Forager — tile-mining family
  • Noita, Powder Toy — cellular-pixel cousin (see the Noita sub-track)

Sprite-based vs voxel-based: the same scene, two ways

A sprite-based platformer stores a few objects on top of a backdrop. A voxel game stores every part of the world as a grid cell. The voxel model is more memory-hungry but lets the player change anything.

Left: sprites floating over a backdrop. Right: every cell stores a tile ID.
💡 Key insight: the voxel-grid representation is what makes Terraria-style games universally destructible. Because every visible pixel is just a lookup in a grid, "mining" reduces to writing grid[y][x] = 0. No special-cased "this wall can break, this one can't".
📖 Deeper context: the broader space of "how to make terrain destructible" is covered in Fundamentals → Destructible Terrain. This tier focuses on the tile-grid approach specifically.

📐 Tile Coordinates ↔ Screen Pixels

Two coordinate systems live side by side. Tile coordinates (tx, ty) are integers indexing the grid. Screen coordinates are pixel positions on the canvas. The transform is trivial but writing it down once now saves a tier of confusion later.

The formulas

screenX = tx × TILE − cameraX
screenY = ty × TILE − cameraY

And the inverse (mouse-to-tile picking):

tx = floor((screenX + cameraX) / TILE)
ty = floor((screenY + cameraY) / TILE)

The camera offset shifts the world relative to the screen — when the camera moves right, every tile draws further left. TILE is the per-track constant for tile size in pixels (we'll use 24 by default).

1 Implement the forward transform tileToScreen:
📝 Click to see: tileToScreen / screenToTile 
// Tile coord -> top-left screen pixel of that tile.
function tileToScreen(tx, ty, tile, cameraX = 0, cameraY = 0) {
    return {
        x: tx * tile - cameraX,
        y: ty * tile - cameraY
    };
}

// Screen pixel -> integer tile coord. floor() handles negatives correctly.
function screenToTile(sx, sy, tile, cameraX = 0, cameraY = 0) {
    return {
        x: Math.floor((sx + cameraX) / tile),
        y: Math.floor((sy + cameraY) / tile)
    };
}

🗂️ Tile Data Structures

You have two ways to store the grid:

Option A: array-of-arrays (the readable one)

tiles[y][x] = 2. Easy to write, easy to read in a debugger. Each row is a separate JS array, allocated separately, with pointer indirection on every access. Fine for tiny demos, wasteful at scale.

Option B: flat Uint8Array (the fast one)

tiles[y * W + x] = 2. One contiguous allocation. Each cell is exactly one byte (0–255 — plenty for 256 material types). Cache-friendly, easy to serialize for save/load, and ~1 MB per 1000×1000 world. This is what real voxel engines use.

1 Wrap it in a tiny TileWorld class so the indexing math lives in one place:
📝 Click to see: TileWorld class 
// A 2D grid stored as a single contiguous byte array.
// One byte per cell = 256 possible material IDs.
class TileWorld {
    constructor(width, height) {
        this.width = width;
        this.height = height;
        // Uint8Array zero-initializes by default — that's our "air" tile (id 0).
        this.tiles = new Uint8Array(width * height);
    }

    // Convert (x, y) -> flat array index.
    index(x, y) {
        return y * this.width + x;
    }

    // Bounds-checked read. Returns 0 ("air") for out-of-bounds reads
    // so callers don't have to guard their iteration.
    get(x, y) {
        if (x < 0 || x >= this.width || y < 0 || y >= this.height) return 0;
        return this.tiles[y * this.width + x];
    }

    // Silently ignores out-of-bounds writes (same convention as get()).
    set(x, y, value) {
        if (x < 0 || x >= this.width || y < 0 || y >= this.height) return;
        this.tiles[y * this.width + x] = value;
    }

    inBounds(x, y) {
        return x >= 0 && x < this.width && y >= 0 && y < this.height;
    }
}
// A 2D grid stored as a single contiguous byte array.
class TileWorld {
    width: number;
    height: number;
    tiles: Uint8Array;

    constructor(width: number, height: number) {
        this.width = width;
        this.height = height;
        this.tiles = new Uint8Array(width * height);
    }

    index(x: number, y: number): number {
        return y * this.width + x;
    }

    get(x: number, y: number): number {
        if (x < 0 || x >= this.width || y < 0 || y >= this.height) return 0;
        return this.tiles[y * this.width + x];
    }

    set(x: number, y: number, value: number): void {
        if (x < 0 || x >= this.width || y < 0 || y >= this.height) return;
        this.tiles[y * this.width + x] = value;
    }

    inBounds(x: number, y: number): boolean {
        return x >= 0 && x < this.width && y >= 0 && y < this.height;
    }
}

Visualizing the flat layout

The diagram below shows a 16×8 grid (top) and the corresponding Uint8Array (bottom strip, one square per byte). Drag the slider to scrub through the array — the highlighted cell and byte stay in lock-step. This is why the math is so simple: index = y × W + x just walks the bytes in row-major order.

index=0, x=0, y=0, value=0 (air)

🎮 Why this matters

A 1000×1000 world is one megabyte at one byte per cell. Saving to localStorage is just btoa(String.fromCharCode(...tiles)) (or better, JSON.stringify(Array.from(tiles)) if you want it human-readable). With array-of-arrays you'd pay 8× the memory plus per-row allocation overhead.

🖌️ Drawing a Tile Grid

The render loop is a nested for. Each cell's material ID looks up an entry in a materials table ({ name, color, solid }) and we fillRect a solid color. No images, no sprite atlas — just the canvas API.

🎨 Procedural Textures (No Image Assets)

Solid-color tiles look like programmer art. Two cheap tricks make them look 100× better with zero added assets:

Trick 1 — per-tile color jitter

Hash (x, y) into a small RGB offset and apply it to the base color. Each cell gets a subtly different shade; the world stops looking like solid bars of paint.

Trick 2 — procedural sub-rects

Draw 2–4 darker pebble-like rects inside each tile at hash-driven positions. For stone: small jagged specks. For dirt: scattered roots. For wood: a vertical grain line. Each rule is one or two lines of canvas code.

1 The 2D hash — deterministic noise from (x, y):
📝 Click to see: hash2D & the textured drawTile 
// Tiny deterministic 2D hash. Same (x, y, seed) -> same value, every time.
// This is enough for visual noise — don't use it for crypto.
function hash2D(x, y, seed = 0) {
    let h = (x * 374761393) ^ (y * 668265263) ^ (seed * 2147483647);
    h = (h ^ (h >>> 13)) * 1274126177;
    h = h ^ (h >>> 16);
    return (h >>> 0) / 4294967295; // 0..1
}

// Tint a hex color by ±amount per channel, deterministically per (x, y).
function jitterColor(hex, x, y, amount = 12, seed = 0) {
    const r = parseInt(hex.slice(1, 3), 16);
    const g = parseInt(hex.slice(3, 5), 16);
    const b = parseInt(hex.slice(5, 7), 16);
    const dr = (hash2D(x, y, seed)       * 2 - 1) * amount;
    const dg = (hash2D(x, y, seed + 1)   * 2 - 1) * amount;
    const db = (hash2D(x, y, seed + 2)   * 2 - 1) * amount;
    const clamp = v => Math.max(0, Math.min(255, v | 0));
    return `rgb(${clamp(r + dr)}, ${clamp(g + dg)}, ${clamp(b + db)})`;
}

// Textured drawTile: solid base + jitter + a couple of speckles.
function drawTileTextured(ctx, sx, sy, size, materialId, seed) {
    const m = VOX_MATERIALS[materialId];
    if (!m || !m.color) return;

    // Layer 1: jittered base
    ctx.fillStyle = jitterColor(m.color, sx / size | 0, sy / size | 0, 12, seed);
    ctx.fillRect(sx, sy, size, size);

    // Layer 2: 2-3 small darker speckles for "grain"
    const speckles = 3;
    for (let i = 0; i < speckles; i++) {
        const hx = hash2D(sx / size | 0, sy / size | 0, seed + 10 + i);
        const hy = hash2D(sx / size | 0, sy / size | 0, seed + 20 + i);
        ctx.fillStyle = jitterColor(m.color, sx + i, sy + i, 30, seed + 7);
        ctx.fillRect(
            sx + (hx * (size - 4)) | 0,
            sy + (hy * (size - 4)) | 0,
            2, 2
        );
    }
}
function hash2D(x: number, y: number, seed: number = 0): number {

    h = (h ^ (h >>> 13)) * 1274126177;
    h = h ^ (h >>> 16);
    return (h >>> 0) / 4294967295;
}

function jitterColor(
    hex: string,
    x: number,
    y: number,
    amount: number = 12,
    seed: number = 0
): string {
    const r: number = parseInt(hex.slice(1, 3), 16);
    const g: number = parseInt(hex.slice(3, 5), 16);
    const b: number = parseInt(hex.slice(5, 7), 16);
    const dr: number = (hash2D(x, y, seed)     * 2 - 1) * amount;
    const dg: number = (hash2D(x, y, seed + 1) * 2 - 1) * amount;
    const db: number = (hash2D(x, y, seed + 2) * 2 - 1) * amount;
    const clamp = (v: number): number => Math.max(0, Math.min(255, v | 0));
    return `rgb(${clamp(r + dr)}, ${clamp(g + dg)}, ${clamp(b + db)})`;
}

📷 Camera & Viewport Culling

Real voxel worlds are bigger than the screen — 80×40 is small for a Terraria world, and Terraria worlds go to 8400×2400. The trick that keeps rendering fast: only iterate the tiles that are actually visible.

The viewport-culling pattern

Compute the tile range covered by the visible rectangle and iterate only that range:

tx0 = floor(cameraX / TILE)
tx1 = ceil((cameraX + viewportW) / TILE)
ty0 = floor(cameraY / TILE)
ty1 = ceil((cameraY + viewportH) / TILE)

Result: render time is proportional to the screen, not to the world. An 8400×2400 world renders the same speed as an 80×40 one.

📝 Click to see: renderViewport (the culled render loop) 
// Renders only the tiles that intersect the viewport.
// Returns the count of tiles drawn — handy for the perf counter.
function renderViewport(ctx, world, tile, cameraX, cameraY, viewportW, viewportH) {
    const tx0 = Math.max(0, Math.floor(cameraX / tile));
    const ty0 = Math.max(0, Math.floor(cameraY / tile));
    const tx1 = Math.min(world.width,  Math.ceil((cameraX + viewportW) / tile));
    const ty1 = Math.min(world.height, Math.ceil((cameraY + viewportH) / tile));

    let drawn = 0;
    for (let y = ty0; y < ty1; y++) {
        for (let x = tx0; x < tx1; x++) {
            const id = world.get(x, y);
            if (id === 0) continue; // skip air
            drawTile(ctx, x * tile - cameraX, y * tile - cameraY, tile, id);
            drawn++;
        }
    }
    return drawn;
}

// Per-frame camera update from WASD/arrow keys.
function updateCamera(camera, keys, speed, world, tile, viewportW, viewportH) {
    if (keys.has('a') || keys.has('ArrowLeft'))  camera.x -= speed;
    if (keys.has('d') || keys.has('ArrowRight')) camera.x += speed;
    if (keys.has('w') || keys.has('ArrowUp'))    camera.y -= speed;
    if (keys.has('s') || keys.has('ArrowDown'))  camera.y += speed;
    // Clamp so we can't scroll past the world edges.
    camera.x = Math.max(0, Math.min(world.width  * tile - viewportW, camera.x));
    camera.y = Math.max(0, Math.min(world.height * tile - viewportH, camera.y));
}
function renderViewport(
    ctx: CanvasRenderingContext2D,
    world: TileWorld,
    tile: number,
    cameraX: number,
    cameraY: number,
    viewportW: number,
    viewportH: number
): number {
    const tx0: number = Math.max(0, Math.floor(cameraX / tile));
    const ty0: number = Math.max(0, Math.floor(cameraY / tile));
    const tx1: number = Math.min(world.width,  Math.ceil((cameraX + viewportW) / tile));
    const ty1: number = Math.min(world.height, Math.ceil((cameraY + viewportH) / tile));

    let drawn: number = 0;
    for (let y = ty0; y < ty1; y++) {
        for (let x = tx0; x < tx1; x++) {
            const id: number = world.get(x, y);
            if (id === 0) continue;
            drawTile(ctx, x * tile - cameraX, y * tile - cameraY, tile, id);
            drawn++;
        }
    }
    return drawn;
}

🏞️ Foreground vs Background Walls

Terraria draws two tile grids: a background "wall" layer rendered dim and first, and a foreground "tile" layer rendered bright on top. Walls fill in behind dug-out tunnels; without them, every tunnel would be a slice of empty sky.

The implementation is dumber than it looks

Two Uint8Arrays of the same shape. Render the background first (dimmer palette), render the foreground second (full palette). That's it. The same materials table can be reused, just with a "wall variant" shading multiplier applied at draw time.

📝 Click to see: drawing two layers 
// Background walls render at ~60% brightness — just enough to read as "behind".
const WALL_DIM = 0.6;

function dimColor(hex, factor) {
    const r = parseInt(hex.slice(1, 3), 16) * factor;
    const g = parseInt(hex.slice(3, 5), 16) * factor;
    const b = parseInt(hex.slice(5, 7), 16) * factor;
    return `rgb(${r | 0}, ${g | 0}, ${b | 0})`;
}

function drawWallTile(ctx, sx, sy, size, materialId) {
    const m = VOX_MATERIALS[materialId];
    if (!m || !m.color) return;
    ctx.fillStyle = dimColor(m.color, WALL_DIM);
    ctx.fillRect(sx, sy, size, size);
}

function renderLayered(ctx, fg, bg, tile, cam, vw, vh, mode) {
    // mode: 'fg' | 'bg' | 'both'
    if (mode === 'bg' || mode === 'both') {
        // Background first — dim variant, drawn through ALL cells (no "air" skip).
        renderViewportWith(ctx, bg, tile, cam.x, cam.y, vw, vh, drawWallTile);
    }
    if (mode === 'fg' || mode === 'both') {
        // Foreground next — bright variant, "air" cells skip so bg shows through.
        renderViewport(ctx, fg, tile, cam.x, cam.y, vw, vh);
    }
}
const WALL_DIM: number = 0.6;

function dimColor(hex: string, factor: number): string {
    const r: number = parseInt(hex.slice(1, 3), 16) * factor;
    const g: number = parseInt(hex.slice(3, 5), 16) * factor;
    const b: number = parseInt(hex.slice(5, 7), 16) * factor;
    return `rgb(${r | 0}, ${g | 0}, ${b | 0})`;
}

function drawWallTile(
    ctx: CanvasRenderingContext2D,
    sx: number,
    sy: number,
    size: number,
    materialId: number
): void {
    const m = VOX_MATERIALS[materialId];
    if (!m || !m.color) return;
    ctx.fillStyle = dimColor(m.color, WALL_DIM);
    ctx.fillRect(sx, sy, size, size);
}
