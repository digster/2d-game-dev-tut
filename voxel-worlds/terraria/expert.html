🌊 Noise: the Terrain Primitive

Procedural worlds are built on noise — a function that turns coordinates into smooth, random-looking values. Plain Math.random() is no good here: it has no spatial structure (neighbours are unrelated) and isn't repeatable. We need a function that is smooth, seedable, and a pure function of position — same (x, y) always gives the same value.

Value noise + fBm

Value noise hashes a random value at every integer lattice point, then smoothly interpolates between them — giving soft hills. One layer is too smooth to look natural, so we sum several: fractal Brownian motion (fBm) adds octaves, each at double the frequency and half the amplitude. Big octaves give the broad shape; small octaves add crags and detail.

// Smoothly-interpolated value noise. hash2D returns 0..1 per lattice point.
function smoothNoise2D(x, y, seed) {
    const x0 = Math.floor(x), y0 = Math.floor(y);
    const fx = x - x0, fy = y - y0;
    // smoothstep weights — removes the grid-aligned creasing of linear blend.
    const sx = fx * fx * (3 - 2 * fx);
    const sy = fy * fy * (3 - 2 * fy);
    const n00 = hash2D(x0,     y0,     seed);
    const n10 = hash2D(x0 + 1, y0,     seed);
    const n01 = hash2D(x0,     y0 + 1, seed);
    const n11 = hash2D(x0 + 1, y0 + 1, seed);
    const nx0 = n00 + (n10 - n00) * sx;
    const nx1 = n01 + (n11 - n01) * sx;
    return nx0 + (nx1 - nx0) * sy;
}

// Fractal Brownian motion — sum octaves, doubling frequency, halving amplitude.
function fbm2D(x, y, seed, octaves) {
    let sum = 0, amp = 1, freq = 1, norm = 0;
    for (let o = 0; o < octaves; o++) {
        sum += amp * smoothNoise2D(x * freq, y * freq, seed + o * 1013);
        norm += amp;
        amp *= 0.5;
        freq *= 2;
    }
    return sum / norm;   // back to 0..1
}

🏔️ Surface Heightmaps & Biomes

The simplest world generator is a heightmap: for each column, one fBm sample gives the surface height; everything above is sky, everything below is dirt then stone. A second, very-low-frequency noise carves wide biome zones that pick the surface material — grass for forest, sand for desert, snow for tundra.

generateTile — a pure function of position

The key design choice: the generator is generateTile(wx, wy, seed) — it returns the material at one tile, depending only on coordinates and the seed. No loops, no neighbours. That purity is what lets us generate the world in any order, or in independent chunks (see below) — every call agrees.

📝 Click to see: surfaceHeight & generateTile 
// Surface height at column wx — a 1D slice of fBm around a base level.
function surfaceHeight(wx, seed, baseY, amp) {
    const n = fbm2D(wx * 0.045, 13.7, seed, 4);
    return Math.floor(baseY + (n - 0.5) * 2 * amp);
}

// Biome zones — a very-low-frequency noise makes wide bands.
function biomeAt(wx, seed) {
    const b = smoothNoise2D(wx * 0.012, 4.2, seed + 777);
    return b < 0.36 ? 'desert' : (b > 0.66 ? 'tundra' : 'forest');
}

// The material at one tile — a PURE function of (wx, wy, seed).
function generateTile(wx, wy, seed, opts) {
    const surf = surfaceHeight(wx, seed, opts.baseY, opts.amp);
    if (wy < surf) return 0;                    // sky
    const biome = biomeAt(wx, seed);
    if (wy === surf) {
        return biome === 'desert' ? SAND
             : biome === 'tundra' ? SNOW : GRASS;
    }
    if (wy < surf + 5) return biome === 'desert' ? SAND : DIRT;
    return STONE;
}

}

type Biome = 'desert' | 'forest' | 'tundra';

function generateTile(wx: number, wy: number, seed: number,
                      opts: { baseY: number; amp: number }): number {
    const surf = surfaceHeight(wx, seed, opts.baseY, opts.amp);
    if (wy < surf) return 0;

        return biome === 'desert' ? SAND
             : biome === 'tundra' ? SNOW : GRASS;
    }
    if (wy < surf + 5) return biome === 'desert' ? SAND : DIRT;
    return STONE;
}

🕳️ Caves & Ore Veins

Caves and ore are the same trick — thresholding 2D noise — applied two ways. For caves, carve a tile to air when a 2D fBm sample lands in a thin band around 0.5: |fBm − 0.5| < width. That band is an iso-line of the noise field, and iso-lines are long and connected — so the caves come out as winding tunnels, not disconnected blobs.

Ore: threshold a different noise

For ore, sample a separate high-frequency noise and place ore where it spikes above a threshold. Lower the threshold with depth and the deep rock fills with richer veins — the classic "dig deeper for better loot" gradient.

isCave = |fbm2D(wx·f, wy·f) − 0.5| < caveWidth
isOre  = fbm2D(wx·g, wy·g) > (0.84 − depthFraction · 0.13)

💧 Cellular-Automaton Liquids

Water is the falling-sand automaton from the Intermediate tier with one extra rule. Sand only moves down and down-diagonally, so it piles. Water adds a sideways spread step — so instead of piling, it flows out and pools, filling containers and spilling over walls.

Down → down-diagonal → sideways

Each water tile, scanned bottom-up: fall straight down if it can; else slide down-diagonally; else — and this is the new part — move sideways into an open neighbour. The sideways step is what levels the surface.

Honest limitation: this binary model (a cell is either water or not) can't represent partial fill, so a lone droplet on flat ground wanders rather than settling, and levels don't perfectly equalise. A "water amount per cell" model fixes that — see Fundamentals → Fluid Dynamics and the Noita sub-track.

📝 Click to see: stepWater 
// Advance every water tile one cell. Bottom-up scan = one cell per tick.
function stepWater(world) {
    for (let y = world.height - 2; y >= 0; y--) {
        for (let x = 0; x < world.width; x++) {
            if (world.get(x, y) !== WATER) continue;

            // 1. straight down
            if (world.get(x, y + 1) === 0) {
                world.set(x, y, 0); world.set(x, y + 1, WATER); continue;
            }
            // 2. down-diagonal
            const dl = world.get(x - 1, y + 1) === 0 && world.get(x - 1, y) === 0;
            const dr = world.get(x + 1, y + 1) === 0 && world.get(x + 1, y) === 0;
            if (dl || dr) {
                const dx = (dl && dr) ? (Math.random() < 0.5 ? -1 : 1) : (dl ? -1 : 1);
                world.set(x, y, 0); world.set(x + dx, y + 1, WATER); continue;
            }
            // 3. sideways spread — the rule that makes water POOL, not pile.
            const sl = world.get(x - 1, y) === 0;
            const sr = world.get(x + 1, y) === 0;
            if (sl || sr) {
                const dx = (sl && sr) ? (Math.random() < 0.5 ? -1 : 1) : (sl ? -1 : 1);
                world.set(x, y, 0); world.set(x + dx, y, WATER);
            }
        }
    }
}

💡 Tile Lighting (Flood Fill)

A generated cave is pitch black until we light it. Tile lighting is a flood fill: brightness starts at full at every light source, then spreads outward tile by tile, losing a little each step — more through solid rock, less through open air. It's a breadth-first search, or equivalently a Dijkstra relaxation on the tile grid.

Seeds, then spread

  • Skylight — every tile exposed to the sky (air, with only air above it) starts at brightness 1.
  • Torches — every torch tile starts at brightness 1.
  • Spread — from the queue of lit tiles, push brightness to neighbours minus a per-tile attenuation. A neighbour is only updated (and re-queued) if the new value beats its current one. Because brightness only ever decreases, the flood halts.
📝 Click to see: computeLight (BFS flood fill) 
// Fill `light` (0..1 per tile) by flooding out from sky tiles and torches.
// A flat array used as a queue (head pointer) avoids Array.shift()'s O(n) cost.
function computeLight(world, light) {
    const W = world.width, H = world.height;
    light.fill(0);
    const queue = [];

    // Seed 1: skylight — each column lit from the top until the first solid tile.
    for (let x = 0; x < W; x++) {
        for (let y = 0; y < H; y++) {
            if (isSolid(world.get(x, y))) break;
            light[y * W + x] = 1;
            queue.push(x, y);
        }
    }
    // Seed 2: torches.
    for (let i = 0; i < world.tiles.length; i++) {
        if (world.tiles[i] === TORCH) {
            light[i] = 1;
            queue.push(i % W, (i / W) | 0);
        }
    }
    // Spread: relax brightness outward. Halts because light only decreases.
    let head = 0;
    while (head < queue.length) {
        const x = queue[head++], y = queue[head++];
        const lv = light[y * W + x];
        const nb = [[1,0],[-1,0],[0,1],[0,-1]];
        for (let k = 0; k < 4; k++) {
            const nx = x + nb[k][0], ny = y + nb[k][1];
            if (nx < 0 || nx >= W || ny < 0 || ny >= H) continue;
            const ni = ny * W + nx;
            const atten = isSolid(world.get(nx, ny)) ? 0.24 : 0.10;
            const nl = lv - atten;
            if (nl > light[ni]) {
                light[ni] = nl;
                queue.push(nx, ny);
            }
        }
    }
}

🧩 Chunked Infinite Worlds

A Terraria world is large; a truly infinite world can't fit in memory at all. The answer is chunking: divide the world into fixed blocks (here 16×16 tiles), and generate a chunk only the first time something touches it. Because generateTile is a pure function of position, a chunk generated now is identical to the same chunk generated an hour later — they always seam up.

Lazy generation, keyed by chunk coordinate

A Map keyed by "cx,cy" holds the chunks that exist. getChunk returns the cached chunk, or generates + stores it on first access. The renderer just calls world.get(wx, wy) — chunks materialise as the camera reaches them.

📝 Click to see: ChunkedWorld 
// An infinite world stored as on-demand chunks.
class ChunkedWorld {
    constructor(chunkSize, seed, genOpts) {
        this.cs = chunkSize;
        this.seed = seed;
        this.genOpts = genOpts;
        this.chunks = new Map();   // "cx,cy" -> Uint8Array
    }
    getChunk(cx, cy) {
        const key = cx + ',' + cy;
        let c = this.chunks.get(key);
        if (!c) {                                  // first touch -> generate
            c = new Uint8Array(this.cs * this.cs);
            for (let ly = 0; ly < this.cs; ly++)
                for (let lx = 0; lx < this.cs; lx++)
                    c[ly * this.cs + lx] = generateTile(
                        cx * this.cs + lx, cy * this.cs + ly, this.seed, this.genOpts);
            this.chunks.set(key, c);
        }
        return c;
    }
    get(wx, wy) {
        const cx = Math.floor(wx / this.cs), cy = Math.floor(wy / this.cs);
        const c = this.getChunk(cx, cy);
        return c[(wy - cy * this.cs) * this.cs + (wx - cx * this.cs)];
    }
}
