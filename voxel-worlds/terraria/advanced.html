💾 Save & Load with RLE Compression

A world the player can't keep isn't a world. Saving is just serialising the tile array — but a raw dump is wasteful: a voxel world is mostly enormous stretches of identical tiles (sky, then stone). Run-length encoding exploits exactly that: store each run of identical tiles as a [value, count] pair.

Why RLE wins here

A 1000-tile-wide band of sky is 1000 bytes raw — but just two numbers as a run. Real generated worlds compress 10–50×. The cap of 255 per run keeps every count inside a single byte; longer runs simply split into several pairs.

📝 Click to see: rleEncode & rleDecode 
// Encode a tile array as [value, runLength, value, runLength, ...].
// Runs cap at 255 so each count fits one byte.
function rleEncode(tiles) {
    const out = [];
    let i = 0;
    while (i < tiles.length) {
        const v = tiles[i];
        let run = 1;
        while (i + run < tiles.length && tiles[i + run] === v && run < 255) run++;
        out.push(v, run);
        i += run;
    }
    return out;
}

// Expand the run pairs back into a flat tile array.
function rleDecode(rle, length) {
    const tiles = new Uint8Array(length);
    let pos = 0;
    for (let i = 0; i < rle.length; i += 2) {
        const v = rle[i], run = rle[i + 1];
        for (let k = 0; k < run; k++) tiles[pos++] = v;
    }
    return tiles;
}

// Save / load through localStorage (wrap in try/catch — it can be disabled).
function saveWorld(world) {
    const blob = { w: world.width, h: world.height, rle: rleEncode(world.tiles) };
    try { localStorage.setItem('my-world', JSON.stringify(blob)); } catch (e) {}
    return blob;
}

🧭 A* on a Destructible Grid

An enemy that hunts the player needs to navigate the world. A* finds a shortest path across the tile grid — and because in a voxel game the grid is the world, "re-plan when the terrain changes" is almost free: mining a tile literally edits the navigation graph.

A* in one breath

A* explores tiles cheapest-first, where a tile's cost is g + h: g = steps taken so far, h = a heuristic estimate of steps remaining (here, Manhattan distance to the goal). The heuristic is what makes A* beat a blind flood fill — it pulls the search toward the goal instead of spreading evenly in all directions.

📝 Click to see: aStar 
// Shortest path of non-solid tiles, start -> goal (4-neighbour).
// Returns { path, visited }; path is null if the goal is unreachable.
function aStar(world, start, goal) {
    const W = world.width;
    const key = (x, y) => y * W + x;
    const h = (x, y) => Math.abs(x - goal.x) + Math.abs(y - goal.y);
    const open = [{ x: start.x, y: start.y }];
    const gScore = new Map([[key(start.x, start.y), 0]]);
    const fScore = new Map([[key(start.x, start.y), h(start.x, start.y)]]);
    const cameFrom = new Map();
    const closed = new Set();
    const visited = [];

    while (open.length) {
        // Pop the open tile with the lowest f = g + h.
        let bi = 0;
        for (let i = 1; i < open.length; i++)
            if (fScore.get(key(open[i].x, open[i].y)) < fScore.get(key(open[bi].x, open[bi].y))) bi = i;
        const cur = open.splice(bi, 1)[0];
        const ck = key(cur.x, cur.y);

        if (cur.x === goal.x && cur.y === goal.y) {        // reached — rebuild the path
            const path = [cur];
            let k = ck;
            while (cameFrom.has(k)) { const p = cameFrom.get(k); path.unshift(p); k = key(p.x, p.y); }
            return { path, visited };
        }
        closed.add(ck);
        visited.push(cur);

        for (const [dx, dy] of [[1,0],[-1,0],[0,1],[0,-1]]) {
            const nx = cur.x + dx, ny = cur.y + dy;
            if (nx < 0 || nx >= W || ny < 0 || ny >= world.height) continue;
            if (isSolid(world.get(nx, ny))) continue;       // walls block the path
            const nk = key(nx, ny);
            if (closed.has(nk)) continue;
            const tentative = gScore.get(ck) + 1;
            if (!gScore.has(nk) || tentative < gScore.get(nk)) {
                cameFrom.set(nk, cur);
                gScore.set(nk, tentative);
                fScore.set(nk, tentative + h(nx, ny));
                if (!open.some(o => o.x === nx && o.y === ny)) open.push({ x: nx, y: ny });
            }
        }
    }
    return { path: null, visited };   // no route exists
}
type Pt = { x: number; y: number };

function aStar(world: TileWorld, start: Pt, goal: Pt):
        { path: Pt[] | null; visited: Pt[] } {
    const W = world.width;
    const key = (x: number, y: number): number => y * W + x;
    const h = (x: number, y: number): number => Math.abs(x - goal.x) + Math.abs(y - goal.y);
    const open: Pt[] = [{ x: start.x, y: start.y }];
    const gScore = new Map<number, number>([[key(start.x, start.y), 0]]);
    const fScore = new Map<number, number>([[key(start.x, start.y), h(start.x, start.y)]]);
    const cameFrom = new Map<number, Pt>();
    const closed = new Set<number>();
    const visited: Pt[] = [];

    while (open.length) {
        let bi = 0;
        for (let i = 1; i < open.length; i++)
            if (fScore.get(key(open[i].x, open[i].y))! < fScore.get(key(open[bi].x, open[bi].y))!) bi = i;
        const cur = open.splice(bi, 1)[0];
        const ck = key(cur.x, cur.y);

        if (cur.x === goal.x && cur.y === goal.y) {
            const path: Pt[] = [cur];
            let k = ck;
            while (cameFrom.has(k)) { const p = cameFrom.get(k)!; path.unshift(p); k = key(p.x, p.y); }
            return { path, visited };
        }
        closed.add(ck);
        visited.push(cur);

        for (const [dx, dy] of [[1,0],[-1,0],[0,1],[0,-1]]) {
            const nx = cur.x + dx, ny = cur.y + dy;
            if (nx < 0 || nx >= W || ny < 0 || ny >= world.height) continue;
            if (isSolid(world.get(nx, ny))) continue;
            const nk = key(nx, ny);
            if (closed.has(nk)) continue;
            const tentative = gScore.get(ck)! + 1;
            if (!gScore.has(nk) || tentative < gScore.get(nk)!) {
                cameFrom.set(nk, cur);
                gScore.set(nk, tentative);
                fScore.set(nk, tentative + h(nx, ny));
                if (!open.some(o => o.x === nx && o.y === ny)) open.push({ x: nx, y: ny });
            }
        }
    }
    return { path: null, visited };
}

🌅 Day/Night Cycle & Weather

The Expert tier's flood-fill lighting seeded every sky-exposed tile at full brightness. A day/night cycle changes one number: the skylight seed intensity becomes a function of the time of day. Torches stay at full brightness — so at night the world goes dark except the pools of torchlight.

One cosine drives the whole sky

Time t runs 0..1 over a day. A cosine turns it into a smooth day factor — 0 at midnight, 1 at noon — which scales both the skylight seed and the background colour. Weather is a cheap multiplier on top: rain clouds dim the skylight further.

dayFactor  = 0.5 − 0.5 · cos(2π · t)
skyIntensity = 0.06 + 0.94 · dayFactor  (× 0.7 when raining)

⚡ Offscreen-Canvas Tile Caching

Re-running a textured tile draw for thousands of visible tiles every frame burns time on work that didn't change. The fix is a classic space-for-time trade: render the static world once to an offscreen canvas, then each frame just drawImage the visible slice of that bitmap — one call instead of thousands.

Cache, then blit

Build the cache once into an off-DOM <canvas>. The per-frame cost collapses to a single blit. When a tile does change, you re-render only that tile's rectangle into the cache — never the whole world.

📝 Click to see: building & blitting the cache 
// Build the cache ONCE — the whole world rendered to an off-DOM canvas.
const cache = document.createElement('canvas');
cache.width  = world.width  * TILE;
cache.height = world.height * TILE;
const cctx = cache.getContext('2d');
for (let y = 0; y < world.height; y++)
    for (let x = 0; x < world.width; x++)
        drawTileTextured(cctx, x * TILE, y * TILE, TILE, world.get(x, y), x, y);

// Per frame: one blit of the visible region — no per-tile work at all.
function render() {
    ctx.drawImage(cache, camera.x, camera.y, viewW, viewH,
                         0, 0, viewW, viewH);
}

// When a tile changes, patch just its rectangle back into the cache.
function setTile(x, y, id) {
    world.set(x, y, id);
    cctx.clearRect(x * TILE, y * TILE, TILE, TILE);
    drawTileTextured(cctx, x * TILE, y * TILE, TILE, id, x, y);
}

🏕️ Mini-Project: A Living World

Four tiers of systems, composed into one scene: a generated, chunk-free world you can mine; a player with real physics; a ghost that A*-hunts you and re-routes when you reshape the terrain; a day/night cycle; and full flood-fill lighting — all on a camera that follows you.

Every frame, in order

  1. Player physics — gravity + per-axis AABB collision (Intermediate).
  2. Enemy AI — A* toward the player's tile, re-planned a few times a second so mining instantly changes its route (this tier).
  3. Day/night — advance time, recompute the skylight (this tier).
  4. Lighting — BFS flood fill from sky + torches (Expert).
  5. Camera + render — follow the player, viewport-cull the draw (Beginner).
