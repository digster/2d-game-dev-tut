🕳️ Cave-Carving Visualizer

The Expert tier carved caves from a noise iso-band. Here you can see the noise field behind the caves, and compare it to a completely different carver — the worm-walk, where random walkers tunnel discs along a wandering path.

Two carvers, two characters

  • Noise iso-band — caves are |fBm − 0.5| < width. Toggle the noise overlay: the cyan tiles are exactly the band, and they are the caves. Caves connect because iso-lines of a smooth field are long and continuous.
  • Worm-walk — caves are the union of many random-walk tunnels. More controllable (you choose where worms start), but connectivity is down to luck.
📝 Click to see: the worm-walk carver 
// Random walkers carve discs along a wandering path.
function carveWorms(world, count, length) {
    for (let w = 0; w < count; w++) {
        let x = 4 + Math.random() * (world.width - 8);
        let y = world.height * 0.45 + Math.random() * (world.height * 0.45);
        let angle = Math.random() * Math.PI * 2;
        for (let s = 0; s < length; s++) {
            // carve a disc of radius 2 around (x, y)
            for (let dy = -2; dy <= 2; dy++)
                for (let dx = -2; dx <= 2; dx++)
                    if (dx * dx + dy * dy <= 4)
                        world.set((x + dx) | 0, (y + dy) | 0, 0);
            angle += (Math.random() - 0.5) * 0.7;   // wander the heading
            x += Math.cos(angle);
            y += Math.sin(angle) * 0.7;             // bias slightly horizontal
        }
    }
}

🗺️ Biome Blend Playground

Biomes come from a single very-low-frequency noise: where it crosses a threshold, the surface material changes. The top panel graphs that noise; the thresholds split it into desert, forest, and tundra. The problem with raw thresholds is the hard seam — an abrupt material change. Blending smooths it.

From hard step to smooth ramp

A hard threshold is a step function: b < 0.36 ? desert : forest. Replace the step with a smoothstep ramp of half-width blend around the threshold and the surface colour interpolates across the border instead of snapping.

📝 Click to see: biomeColor with smoothstep blending 
const DESERT = [215,200,120], FOREST = [74,138,58], TUNDRA = [223,233,240];
const T1 = 0.36, T2 = 0.66;   // biome thresholds

// smoothstep — an S-curve ramp from 0 to 1 between edges a and b.
function smoothstep(a, b, x) {
    const t = Math.max(0, Math.min(1, (x - a) / (b - a)));
    return t * t * (3 - 2 * t);
}

// blend = 0 -> hard biome borders; blend > 0 -> colour ramps across them.
function biomeColor(b, blend) {
    if (blend <= 0) return b < T1 ? DESERT : (b > T2 ? TUNDRA : FOREST);
    const k1 = smoothstep(T1 - blend, T1 + blend, b);   // desert -> forest
    const k2 = smoothstep(T2 - blend, T2 + blend, b);   // forest -> tundra
    return lerp3(lerp3(DESERT, FOREST, k1), TUNDRA, k2);
}

function biomeColor(b: number, blend: number): number[] {

    const k1 = smoothstep(T1 - blend, T1 + blend, b);
    const k2 = smoothstep(T2 - blend, T2 + blend, b);
    return lerp3(lerp3(DESERT, FOREST, k1), TUNDRA, k2);
}

💡 Light-Propagation Stepper

The Expert tier computed lighting with a flood fill — but it ran instantly, so the spreading was invisible. Here it's slowed to one BFS ring per Step: you watch brightness radiate out from the torch, one tile of distance at a time, the orange outline marking the frontier about to be processed.

A flood fill is breadth-first search

Keep a frontier — the tiles at the current distance. Each Step relaxes light into every unseen, non-solid neighbour of the frontier, and those neighbours become the next frontier. Walls aren't in the frontier, so light flows around them, never through — the cave fills, the rock stays dark.

📝 Click to see: one BFS ring 
// Advance the flood fill by ONE ring. `frontier` holds the current ring;
// `seen` marks tiles already lit; brightness fades with ring distance.
function step() {
    if (!frontier.length) return;
    const next = [];
    const brightness = Math.max(0, 1 - (ring + 1) * FALLOFF);
    for (const c of frontier) {
        for (const [dx, dy] of [[1,0],[-1,0],[0,1],[0,-1]]) {
            const nx = c.x + dx, ny = c.y + dy;
            if (nx < 0 || nx >= W || ny < 0 || ny >= H) continue;
            const ni = ny * W + nx;
            if (seen[ni] || isSolid(world.get(nx, ny))) continue;  // walls block
            seen[ni] = 1;
            light[ni] = brightness;
            next.push({ x: nx, y: ny });
        }
    }
    frontier = next;   // the neighbours become the next ring
    ring++;
}

💧 Liquid-Flow Stepper

The water automaton from the Expert tier, with its decision-making made visible. Every water cell, every frame, asks the same ordered question — can I go down? down-diagonally? sideways? — and an arrow drawn on each cell shows the answer it's about to act on. Step, and the arrows become moves.

Rule precedence, colour-coded

Yellow ↓ = falling straight down · orange = sliding down-diagonally · red = spreading sideways · a white dot = settled. The colours are the rule precedence: a cell only considers the next rule when the one before it fails.

📝 Click to see: the decision function 
// Which rule WOULD this water cell apply this tick? Pure — no mutation,
// so we can draw the decision as an arrow before Step executes it.
function decision(world, x, y) {
    if (world.get(x, y + 1) === 0) return 'down';

    const dl = world.get(x - 1, y + 1) === 0 && world.get(x - 1, y) === 0;
    const dr = world.get(x + 1, y + 1) === 0 && world.get(x + 1, y) === 0;
    if (dl || dr) return 'diag';

    const sl = world.get(x - 1, y) === 0;
    const sr = world.get(x + 1, y) === 0;
    if (sl || sr) return 'side';

    return 'settled';
}
type WaterMove = 'down' | 'diag' | 'side' | 'settled';

💎 Ore Distribution Generator

Ore is noise-thresholding with a twist: the threshold drops with depth, so deep rock spawns ore more readily. This demo pairs the world cross-section with a histogram of how much ore landed at each depth — and the cyan line is the threshold itself, so you can see cause and effect side by side.

Threshold as a function of depth

oreThreshold = 0.84 − depthFraction × 0.13 − richness
place ore where  fbm2D(x·g, y·g) > oreThreshold

As depth increases the threshold falls, so more of the noise field clears the bar — the histogram skews toward the bottom. The richness slider shifts the whole threshold down at once.

The Simulations tier turned five algorithms inside out:
