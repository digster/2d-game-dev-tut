🎯 What You'll Build

This track teaches you how to make a 2D multiplayer game feel good across the Internet's hostile reality of latency, jitter, packet loss, and out-of-order delivery. Every demo runs against a simulated network with sliders for RTT, jitter, loss, and reorder probability — so you can crank conditions to nightmare levels and watch each technique earn its keep.

🎬 The final mini-game (by the end of the Simulations tier)

  • A two-player top-down arena playable inside one browser tab with two canvases acting as "server" and "client"
  • Toggleable techniques — client-side prediction, server reconciliation, entity interpolation, lag compensation, area-of-interest filtering
  • Live network weather sliders — RTT 0–500 ms, jitter 0–100 ms, loss 0–50%, reorder 0–20%
  • Lockstep mini-RTS and rollback duelling cars (reusing the racing-sim car) showing the two extreme determinism strategies
  • A bandwidth-budget calculator — tweak tick rate × players × quantization → bytes/sec
  • A replay scrubber — every tick recorded, scrub backward and forward
💡 Why netcode is its own track: multiplayer isn't a genre — it's a cross-cutting concern. RTS wants lockstep; fighting and racing want rollback; shooters want lag compensation; survival games want chunk streaming. This track is the reference the other tracks will eventually point back to.

📚 Recommended Prerequisites

This track assumes you've already worked through these topics from the Fundamentals guide. The further into the track you go, the more they matter.

Strongly recommended before BEGINNER

  • Delta Time & Fixed Timestep — every netcode demo runs on a fixed-step simulation; this explains why
  • Network Interpolation (primer) — the existing Fundamentals primer; this track is the deep dive

Strongly recommended before INTERMEDIATE

  • Linear Interpolation (Lerp) — entity interpolation is just lerp between snapshot buffers
  • Vectors: Position & Direction — every entity's state is a vector

Strongly recommended before ADVANCED

  • State Machines — the connection lifecycle is a state machine
  • Collision Detection — lag-comp hit validation needs collision against historical positions

Strongly recommended before EXPERT

  • Spatial Partitioning & QuadTree — area-of-interest filtering uses a spatial grid
  • Top-Down Racing Sim track — the rollback demo reuses the racing-sim car

💡 How to Use This Track

👶 If you've never built multiplayer

Start at the Beginner tier (once it ships). Drag the FakeNetwork sliders to their worst settings early and often — every later tier exists to fix one specific symptom you'll see in the Beginner mini-project.

🛠️ If you've shipped a multiplayer game before

Skim Beginner and Intermediate, jump to Advanced for reconciliation + lag-comp, and use the Expert tier as a reference for lockstep vs rollback tradeoffs. The Simulations tier's bandwidth calculator is the part you'll bookmark.

🎮 If you're building a specific genre

RTS → focus on the lockstep section (Expert). Fighting/racing → the rollback section (Expert). Shooter → reconciliation + lag-comp (Advanced). MMO/survival → area-of-interest (Expert) + delta compression (Advanced).
