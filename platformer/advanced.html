🏗️ A controller worth promoting

The Intermediate tier left us with a controller (PlayerBody) that already feels great. This tier is its second consumer, which in this repo means it graduates out of the tier file and into the shared engine — it now lives in engine/player.js, loaded by both the Intermediate and Advanced pages. (Same rule the roguelike track used to promote its actors.js.)

Two kinds of new thing land here. The abilities (wall-slide, wall-jump, dash) belong to the character, so they go into PlayerBody behind the same zeroable-knob pattern as the feel kit. The collision extensions (one-way platforms, slopes, moving platforms) are about the world, so they're taught inline here as a richer resolver that PlayerBody calls through a swappable resolve() hook.

💡 The extension seam: PlayerBody never calls moveAndCollide for its main moves — it calls this.resolve(box, dx, dy), which defaults to plain solid-tile collision. Swap that one function for pfResolveWorld and the same character suddenly understands one-way platforms and slopes, with zero changes to the controller.

🧱 Wall-slide & wall-jump

Press toward a wall in mid-air to cling and slide down at a capped speed; jump to leap up and away from it. Chain left-right wall-jumps to climb the shaft and reach the ledge. ← → move, ↑/Space jump.

// wallDir(): is there a SOLID column just outside my left/right edge?
const onWall = airborne ? wallDir() : 0;           // -1 left, +1 right, 0 none

// sliding = pressing INTO the wall while descending → cap the fall speed
if (onWall === inputX && vy > 0) vy = Math.min(vy, WALL_SLIDE_SPEED);

// wall jump = up + AWAY from the wall, then briefly ignore input so the
// impulse actually carries you off (otherwise "hold toward wall" cancels it)
if (jumpPressed && onWall !== 0) {
  vy = -WALL_JUMP_Y;
  vx = -onWall * WALL_JUMP_X;
  inputLock = WALL_JUMP_LOCK;     // ~0.16 s
}
const onWall: number = airborne ? wallDir() : 0;

if (onWall === inputX && vy > 0) vy = Math.min(vy, WALL_SLIDE_SPEED);

if (jumpPressed && onWall !== 0) {
  vy = -WALL_JUMP_Y;
  vx = -onWall * WALL_JUMP_X;
  inputLock = WALL_JUMP_LOCK;
}

💨 Dash

Tap X / K / L to dash in the aimed direction (or the way you face). The dash is a fixed-duration burst that ignores gravity, has a short cooldown, and refreshes when you touch ground or a wall — so you get exactly one mid-air dash, the Celeste rule. The bar shows when it's ready.

dash speed 560 px/s
click, then dash across the gaps…
📝 Click to see: a dash is a short, gravity-free, refillable burst 
// trigger: a press, charge available, not already dashing
if (dashPressed && canDash && cooldown <= 0) {
  const [ax, ay] = aimedDir() || [facing, 0];   // 8-way, normalised
  dashVX = ax * DASH_SPEED; dashVY = ay * DASH_SPEED;
  dashLeft = DASH_TIME; cooldown = DASH_COOLDOWN; canDash = false;
}

if (dashLeft > 0) {                 // during the dash:
  vx = dashVX; vy = dashVY;         //   fixed velocity, NO gravity
  dashLeft -= dt;
}
if (onGround || wallSliding) canDash = true;   // refill the charge
if (dashPressed && canDash && cooldown <= 0) {
  const [ax, ay] = aimedDir() ?? [facing, 0];
  dashVX = ax * DASH_SPEED; dashVY = ay * DASH_SPEED;
  dashLeft = DASH_TIME; cooldown = DASH_COOLDOWN; canDash = false;
}

if (dashLeft > 0) { vx = dashVX; vy = dashVY; dashLeft -= dt; }
if (onGround || wallSliding) canDash = true;

🪜 One-way platforms

A one-way platform is solid only to something landing on it from above. Jump up through the wooden platforms from below; land on top of them coming down; hold ↓ and press jump to drop through the one you're standing on.

⛰️ Slopes

Flat tiles can't make a hill. Slope tiles are 45° triangles, and the trick is to stop treating the foot collision as "is this cell solid?" and start asking "how high is the ramp surface under my centre?". Walk ← → over the hill — your feet stay glued to the ramp going up and down.

click, then walk over the hill…
📝 Click to see: sample the ramp surface under the centre 
// localX = how far the player's centre is into the slope tile (0..TILE)
const localX = clamp(centreX - col * TILE, 0, TILE);

// SLOPE_NE rises to the right; SLOPE_NW rises to the left
let surfaceY = null;
if (tile === SLOPE_NE) surfaceY = row * TILE + (TILE - localX);
if (tile === SLOPE_NW) surfaceY = row * TILE + localX;

if (surfaceY !== null) {
  const climbing = box.bottom >= surfaceY - 1;            // feet under the ramp
  const sticking = wasGround && (surfaceY - box.bottom) <= TILE*0.6; // glue downhill
  if (climbing || sticking) { box.y = surfaceY - box.h; hit.down = true; }
}
const localX = clamp(centreX - col * TILE, 0, TILE);

let surfaceY: number | null = null;
if (tile === Tile.SLOPE_NE) surfaceY = row * TILE + (TILE - localX);
if (tile === Tile.SLOPE_NW) surfaceY = row * TILE + localX;

if (surfaceY !== null) {
  const climbing = box.bottom >= surfaceY - 1;
  const sticking = wasGround && (surfaceY - box.bottom) <= TILE * 0.6;
  if (climbing || sticking) { box.y = surfaceY - box.h; hit.down = true; }
}
The "sticking" term matters: walking down a ramp, the per-step horizontal move would otherwise launch you off the edge into a little hop. Gluing the feet to the surface when you're already grounded and within ~half a tile keeps the descent smooth.

🛗 Moving platforms & conveyors

When the ground moves, the rider must move with it. The fix is relative motion: each step, move the platform first, then add the platform's delta to anyone standing on it before resolving collisions. Ride the orange platform across, the green lift up, and the blue conveyor belt (which drags you sideways) to the goal ledge.

click, then ride to the right ledge…
📝 Click to see: carry the rider by the platform's delta 
const riding = feetOnTop(player, platform) && player.vy >= 0;

const before = { x: platform.x, y: platform.y };
movePlatform(platform, dt);                 // advance its path/lerp
const ddx = platform.x - before.x, ddy = platform.y - before.y;

if (riding) {
  player.x += ddx; player.y += ddy;         // inherit the platform's motion
  if (platform.conveyor) player.x += platform.conveyor * dt;  // belt drag
}
const riding = feetOnTop(player, platform) && player.vy >= 0;

const before = { x: platform.x, y: platform.y };
movePlatform(platform, dt);
const ddx = platform.x - before.x, ddy = platform.y - before.y;

if (riding) {
  player.x += ddx; player.y += ddy;
  if (platform.conveyor) player.x += platform.conveyor * dt;
}
