⚙️ Do less work, not faster work

Every technique in this tier follows one rule: the cheapest work is the work you never do. Don't draw tiles off-screen (culling). Don't allocate objects you could reuse (pooling). Don't test collision pairs that are nowhere near each other (broad phase). Don't redraw pixels that didn't change (render caching). None of it alters how the game looks or plays — it changes how much the CPU touches each frame, which is what lets a big, juicy level hold 60 fps.

✂️ Viewport culling

A 200-column map auto-pans past the view. With culling off the renderer touches every tile in the whole map, every frame; on, it iterates only the tiles inside the camera window. ♻️ Object pooling

A fountain spews particles forever. Pooled reuses a fixed array of objects — after the initial allocation, nothing is ever created, so the garbage collector stays asleep. Naive news up an object per spawn and filters out the dead each frame, feeding the GC and risking a hitch. Same fountain; watch the "objects allocated" counter.

📝 Click to see: a swap-remove object pool 
// preallocate once; `n` objects are live at the FRONT of the array
constructor(cap) { this.arr = Array.from({length: cap}, makeBlank); this.n = 0; }

spawn(p) { if (this.n >= cap) return;          // full → drop, never allocate
  const o = this.arr[this.n++];                // reuse a dead slot's memory
  o.x = p.x; o.y = p.y; /* …copy fields… */ o.t = 0; }

update(dt) {
  for (let i = 0; i < this.n; i++) {
    const o = this.arr[i]; o.t += dt; /* …integrate… */
    if (o.t >= o.life) { this.arr[i] = this.arr[--this.n]; this.arr[this.n] = o; i--; } // swap-remove
  }
}
constructor(cap: number) { this.arr = Array.from({length: cap}, makeBlank); this.n = 0; }

spawn(p: Particle) { if (this.n >= cap) return;
  const o = this.arr[this.n++];
  o.x = p.x; o.y = p.y; o.t = 0; }

update(dt: number) {
  for (let i = 0; i < this.n; i++) {
    const o = this.arr[i]; o.t += dt;
    if (o.t >= o.life) { this.arr[i] = this.arr[--this.n]; this.arr[this.n] = o; i--; }
  }
}

🧮 Broad-phase collision (spatial grid)

Checking every entity against every other is n(n−1)/2 pair tests — 120 balls is 7,140 checks per frame. A spatial grid buckets entities by cell, so each only checks its own and neighbouring cells. Slide the count up and compare the pair-check numbers; the grid stays roughly linear. (Cross-link: Fundamentals → Spatial Partitioning.)

// 1) bucket every entity into a cell ≈ its own size
for (const e of entities)
  bucket(`${Math.floor(e.x/cell)},${Math.floor(e.y/cell)}`).push(e);

// 2) each entity only tests its own + 8 neighbouring cells
for (const e of entities) {
  const cx = Math.floor(e.x/cell), cy = Math.floor(e.y/cell);
  for (let dy = -1; dy <= 1; dy++) for (let dx = -1; dx <= 1; dx++)
    for (const o of bucket(`${cx+dx},${cy+dy}`) || [])
      if (o.id > e.id) collide(e, o);   // id ordering tests each pair once
}
// far-apart pairs are never even considered → ~O(n) instead of O(n²)

🧱 Chunked render caching

A static tile layer never changes, so why redraw it every frame? Render each chunk (here 8×8 tiles) to its own offscreen canvas once, then just blit those images. Only a chunk whose tiles changed gets re-rendered. Click a tile to toggle it and watch the "re-renders this frame" counter tick exactly once — for the one chunk you touched.

draw(ctx, originX, originY, viewW, viewH) {
  for (const [cx, cy] of visibleChunks(originX, originY, viewW, viewH)) {
    const cv = this.canvasFor(cx, cy);
    if (this.dirty.has(key(cx, cy))) {         // render ONCE, or after an edit
      drawTileMap(cv.getContext('2d'), this.map, { originX: cx*px, originY: cy*px });
      this.dirty.delete(key(cx, cy));
    }
    ctx.drawImage(cv, cx*px - originX, cy*py - originY);   // otherwise just blit
  }
}
markDirtyTile(col, row) { this.dirty.add(key(col/ct | 0, row/ct | 0)); }

      this.dirty.delete(key(cx, cy));
    }
    ctx.drawImage(cv, cx*px - originX, cy*py - originY);
  }
}

✅ Recap — what you built

  • Viewport culling — iterate only the tiles in the camera window (O(view), not O(world)).
  • Object pooling — reuse a fixed array; zero per-frame allocation, no GC hitches.
  • Broad-phase collision — a spatial grid turns O(n²) pair checks into ~O(n).
  • Chunked render caching — blit cached chunk images; re-render only dirty chunks.
