✨ Juice: the feedback layer

"Juice" is the layer of feedback that doesn't change the rules but makes the game feel responsive and alive: a camera that frames the action, backgrounds with depth, a screen that kicks on impact, dust off your heels, a body that squashes and stretches. None of it touches the physics from the earlier tiers — it sits on top and amplifies what already happened.

💡 Why the camera comes first: every juice technique here is expressed in terms of the camera — parallax scrolls relative to it, screen shake offsets it, particles and the character are drawn through it. So we build a real Camera first (taught inline; it graduates to engine/camera.js when the Simulations capstone reuses it), then layer everything else on.

🎥 The camera-space convention used by every demo here

Clear the canvas → draw parallax in screen space → ctx.translate(-cam.originX, -cam.originY) → draw the world (tiles + entities) in world coordinates → restore() → draw the HUD in screen space. One translate, and everything in the world just works.

🎥 The follow camera

Run around the level (it's wider than the view — see the minimap, top-right). With smoothing at 0 the camera is welded to the player (rigid, nauseating); raise it and the camera eases. The deadzone (dashed box) lets you move within it without the camera budging; look-ahead biases the view toward the way you face so you see what's coming. ← → run, ↑ jump.

follow(tx, ty, dt, o) {
  const focus = tx + o.lookAhead * o.lookDir;   // bias toward where you're heading
  let goalX;
  if (o.deadzoneW) {                              // only move when the target leaves the box
    const sx = focus - this.x, mid = this.viewW / 2, half = o.deadzoneW / 2;
    if      (sx < mid - half) goalX = focus - (mid - half);
    else if (sx > mid + half) goalX = focus - (mid + half);
    else                       goalX = this.x;    // inside → hold still
  } else goalX = focus - this.viewW / 2;          // else keep centred

  // frame-rate-correct easing: 99% of the gap closed in smoothMs (0 = rigid)
  const a = o.smoothMs > 0 ? 1 - Math.exp((Math.log(0.01) / (o.smoothMs/1000)) * dt) : 1;
  this.x += (goalX - this.x) * a;
  this.x = clamp(this.x, 0, this.worldW - this.viewW);   // never show past the edges
}
follow(tx: number, ty: number, dt: number, o: FollowOpts): void {
  const focus = tx + o.lookAhead * o.lookDir;
  let goalX: number;
  if (o.deadzoneW) {
    const sx = focus - this.x, mid = this.viewW / 2, half = o.deadzoneW / 2;
    if      (sx < mid - half) goalX = focus - (mid - half);
    else if (sx > mid + half) goalX = focus - (mid + half);
    else                       goalX = this.x;
  } else goalX = focus - this.viewW / 2;

  const a = o.smoothMs > 0 ? 1 - Math.exp((Math.log(0.01) / (o.smoothMs/1000)) * dt) : 1;
  this.x += (goalX - this.x) * a;
  this.x = clamp(this.x, 0, this.worldW - this.viewW);
}

💥 Screen shake (the trauma model)

Add some trauma (a 0–1 value) on impact; the actual shake is trauma², so a little trauma barely wobbles while a lot really kicks — and trauma decays linearly so it always settles. Hit the buttons (or land a hard fall) and watch the trauma bar and the shake.

📝 Click to see: trauma in, trauma² out, decay over time 
addTrauma(t) { this.trauma = clamp(this.trauma + t, 0, 1); }   // events ADD trauma

updateShake(dt, maxOffset = 12, decay = 1.5) {
  const s = this.trauma * this.trauma;          // ← the key: shake = trauma²
  this.shakeTime += dt;
  // high-frequency sines → a fast, smooth jitter (not a per-frame random flicker)
  this.shakeX = maxOffset * s * (Math.sin(this.shakeTime*47) + 0.5*Math.sin(this.shakeTime*111));
  this.shakeY = maxOffset * s * (Math.sin(this.shakeTime*57 + 1.7) + 0.5*Math.sin(this.shakeTime*93));
  this.trauma = Math.max(0, this.trauma - decay * dt);   // always settles back to calm
}
// the renderer translates by (x + shakeX, y + shakeY)
addTrauma(t: number) { this.trauma = clamp(this.trauma + t, 0, 1); }

updateShake(dt: number, maxOffset = 12, decay = 1.5): void {
  const s = this.trauma * this.trauma;
  this.shakeTime += dt;
  this.shakeX = maxOffset * s * (Math.sin(this.shakeTime*47) + 0.5*Math.sin(this.shakeTime*111));
  this.shakeY = maxOffset * s * (Math.sin(this.shakeTime*57 + 1.7) + 0.5*Math.sin(this.shakeTime*93));
  this.trauma = Math.max(0, this.trauma - decay * dt);
}

🏔️ Parallax scrolling

Depth on the cheap: draw background layers that scroll at fractions of the camera's speed. Far layers barely move (small factor), near layers move almost with the world. Toggle it off and the same art moves all together — flat and painted-on. Run to feel the difference.

click, then run…
📝 Click to see: each layer offset by camera × its depth factor 
const LAYERS = [
  { factor: 0.15, ... },   // far ridge — barely moves
  { factor: 0.38, ... },   // mid hills
  { factor: 0.66, ... },   // near hills — moves almost with the world
];

for (const L of LAYERS) {
  const offset = -(camX * L.factor);   // ← the whole trick
  drawHillsTiled(offset, L);           // repeat the shapes across the view, wrapped
}
// foreground tiles use the full camera offset (factor 1, drawn under the translate)
const LAYERS = [
  { factor: 0.15 }, { factor: 0.38 }, { factor: 0.66 },
];

for (const L of LAYERS) {
  const offset = -(camX * L.factor);
  drawHillsTiled(offset, L);
}

🤸 Animation from the state machine

We never need to ask "which animation should play?" — the controller's state machine (idle / run / jump / fall / land, from the Intermediate tier) already knows. Each state maps to a pose, and a phase advances limbs over time (faster the faster you run). Move and jump and watch the procedural figure react.

📝 Click to see: state → pose, phase advanced by dt 
// the controller already computed body.state this frame — just READ it
const animSpeed = body.state === 'run' ? 6 + Math.abs(body.vx) * 0.03 : 3;
phase += animSpeed * dt;            // loops the limb cycle

function drawCharacter(state, phase, facing) {
  let legSwing = 0;
  if (state === 'run')  legSwing = Math.sin(phase) * 8;   // alternating stride
  else if (state === 'jump') legSwing = -5;               // tucked
  else if (state === 'fall') legSwing = 5;                // spread
  // ...draw head + torso + legs(legSwing) + arms(opposite swing)
}

🌫️ Particles — dust & landing puffs

A handful of fading rectangles sells weight and speed. Kick up dust behind your feet while running; burst a puff on landing, scaled by how hard you hit. Run and jump around.

update(dt) {
  for (const p of this.ps) { p.t += dt; p.vy += p.g*dt; p.x += p.vx*dt; p.y += p.vy*dt; }
  this.ps = this.ps.filter(p => p.t < p.life);     // reap the dead
}
draw(ctx) {
  for (const p of this.ps) { ctx.globalAlpha = 1 - p.t/p.life; ctx.fillRect(p.x, p.y, p.size, p.size); }
  ctx.globalAlpha = 1;
}
